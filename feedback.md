# Ebi Showcase 完全フィードバックレポート（新鮮視点）

**目的**: 中学生のプログラミング初心者と、実務経験のあるプロプログラマーの両方の立場から、現在の教材 HTML と Go コードの説明・実装を徹底的に点検し、**伝わりにくい点・意味が通らない点** を箇条書きで洗い出す。

## 1. Core Lessons (LEVEL 01‑12)
### LEVEL 01 – Update / Draw / Layout
- **中学生視点**: 「Update は何をするの？」と説明が抽象的。実際に変数をインクリメントする例が無く、**「ゲームループの流れ」** が頭に浮かばない。
- **プロ視点**: `layout.Draw` がコードに出てこないのに、HTML で「描画は Layout が管理」 と書いている点が**実装と文言の不整合**。
- **中学生**: ラジアンと度の変換式が記載されていない。`math.Sin` を度で読んでしまう危険がある。
- **プロ**: `rand.Seed(time.Now().UnixNano())` が毎フレーム呼ばれると**テスト再現性が崩れる**。シードは一次だけにすべき。

### LEVEL 02 – Movement & Velocity
- **中学生**: `dx / distance` の 0 除算防止が全く書かれていない。**「速さが無限大になる」** と混乱する。
- **プロ**: `math.Hypot` が紹介されていない。代わりに `math.Sqrt(dx*dx+dy*dy)` がコードに残るのは**可読性が低い**。
- **中学生**: キーボードとタッチ操作の違いがページに記載されていない。**「スマホでどう操作する？」** が不明。

### LEVEL 03 – Collision Detection
- **中学生**: 図は AABB（矩形）なのにコードは円形衝突。**「図とコードが食い違う」** で理解が止まる。
- **プロ**: `iota` を使った列挙が登場するが、**「iota が何をするのか」** の説明が全く無い。

### LEVEL 04 – Gravity & Jump
- **中学生**: 重力定数 `g := 0.4` の単位が **ピクセル/フレーム²** とだけ書かれ、現実感がない。**「1 フレームがどれだけ時間か」** が不明。
- **プロ**: ジャンプ速度が負の値でハードコードされているが、**「上方向が負」** の座標系説明が欠如。

### LEVEL 05 – Tile Map
- **中学生**: タイルサイズが 32 とだけ書かれ、**「タイルは何ピクセル？」** が不明。マップデータが JSON 配列だけで、**「JSON の配列とオブジェクトの違い」** が未説明。
- **プロ**: `for _, tile := range levelData` の `_` が未使用警告になるが、コメントが無い。

### LEVEL 06 – Animation
- **中学生**: `fps := 8` が **「1 秒あたり 8 フレーム」** とだけ書かれ、**「アニメはどれくらい速くなるか」** がイメージしにくい。
- **プロ**: `anim.Tick()` の内部ロジックが隠蔽され、**「フレームインデックスのロジック」** が読めない。

### LEVEL 07 – Input Handling
- **中学生**: `ebiten.IsKeyPressed(ebiten.KeySpace)` が **「スペースキーが押されたか」** とだけ記載。**「キーコードはどこで定義されている」** が不明。
- **プロ**: タッチ入力の終了判定がなく、**「タッチが離れたらどうするか」** が抜けている。

### LEVEL 08 – Particle System
- **中学生**: `life` がフレーム数で管理されているが、**「1 フレームは何秒か」** が説明されていない。
- **プロ**: ランダム速度 `rand.Float64()*2-1` の範囲説明が無く、**「-1 から 1」** が直感的に分からない。

### LEVEL 09 – Camera Follow
- **中学生**: `camX := player.X - screenW/2` が整数除算になると**「カメラが 1 ピクセルずれる」** と混乱する。
- **プロ**: カメラ平滑化の `*0.1` が **「イージング」** とだけ書かれ、**アルゴリズムの根拠** が不明。

### LEVEL 10 – State Machine
- **中学生**: `iota` で列挙された `GameState` が **「enum のようなもの」** とだけ説明。**「各状態は何を意味するか」** が抜けている。
- **プロ**: 状態遷移図が画像だけで、コードの `switch` とリンクが無い。

### LEVEL 11 – Audio Playback
- **中学生**: `audio.NewContext(44100)` が **「サンプリングレート」** とだけ書かれ、**「44.1kHz が何を意味するか」** が不明。
- **プロ**: 音量スケールが 0‑1 とだけ書かれ、**「0.5 が実際にどれくらいの音量か」** の感覚が欠如。

### LEVEL 12 – Save / Load
- **中学生**: `json.Marshal` が **「JSON データに変換」** とだけ書かれ、**「JSON の構造はどうなる」** が全く描かれていない。
- **プロ**: エラーハンドリングが `panic` だけで、**「ユーザーにエラーメッセージを表示」** する方法が抜けている。

## 2. Visual Effects Lab
### vfx‑transform
- **中学生**: 「原点へ平行移動 → 回転 → 再平行移動」と書かれているが、実装は **`Translate(dx,dy); Rotate(theta); Translate(-dx,-dy)`** で順序が逆。**「実際の動きと説明が食い違う」**。
- **プロ**: `theta` がラジアンなのに、説明は度単位。**単位変換が欠如**。
- **中学生**: `Scale(2,2)` が **「拡大」** とだけ書かれ、**「拡大の中心が左上」** と説明しないと予想がつかない。

### vfx‑color‑scale
- **中学生**: `ColorM.Scale(1,0.5,0.5,1)` が **「半透明」** とだけ書かれ、**「アルファは 0‑1」** が抜けている。
- **プロ**: sRGB ↔ linear 変換が無視され、**「色が暗く見える」** 原因が説明されていない。

### vfx‑particle‑spray
- **中学生**: スプライトシート座標が `Rect(0,0,16,16)` だけで、**「スプライトは左上から順に」** が不明。
- **プロ**: `rand.NormFloat64()` が正規分布である旨が無く、**「出力が負になると画面外へ」** というバグの原因が不透明。

### vfx‑magic‑fire
- **中学生**: `math.Pow(2, intensity)` が **「2 の何乗」** とだけ書かれ、**「指数関数の意味」** が全く説明されていない。
- **プロ**: `intensity` を `int()` にキャストすると切り捨てが起き、**「炎の強さが段階的に変わらない」** バグが潜む。

## 3. Track 系列（代表的なページ）
### clicker / growth‑curves
- **中学生**: `math.Pow(1.18, float64(n))` が **「1.18 の n 乗」** とだけ書かれ、**「指数関数のイメージ」** が不足。
- **プロ**: 浮動小数点 → 整数のキャストで **丸め誤差** が起き、**期待値と実際がずれる**。
- **中学生**: 成長曲線の **「飽和点」** が UI に表示されず、**「いつ止まるか」** が分からない。

### deckbuilder / deck‑cycle
- **中学生**: `rand.Shuffle` の背後にある **Fisher‑Yates アルゴリズム** が全く説明されていない。**「なぜランダムになるのか」** が謎。
- **プロ**: デッキが空になったときの **`discard = nil`** がメモリ解放に不十分で、**GC が走らない可能性** がある。
- **中学生**: `append([]Card(nil), deck...)` が「コピー」なのか「参照」なのか混乱する。

### merge‑physics / falling‑circles
- **中学生**: `dx / distance` の 0 除算防止が無く、**「円が重なったまま止まらない」** バグが起きやすい。
- **プロ**: 反射ベクトル計算に `dot` の定義が抜けていて、**「物理的に正しいか」** が不透明。
- **中学生**: バウンド係数のスライダーが無いので、**「跳ね返りの強さが変えられない」**。

### fighting / hit‑reaction
- **中学生**: `why-grid` の見出しが `<h3>` で他ページは `<p class="eyebrow">`。**「見出し階層が途切れ」** で読みにくい。
- **プロ**: ダメージ計算 `int(math.Floor(base * (1 - armor/100)))` が **整数除算** の罠を抱えている。`armor` が `float64` でないと誤差が出る。
- **中学生**: ヒットエフェクトが PNG だが透過が無く、**「背景が白で見えにくい」**。

### maze‑chase / buffered‑turn
- **中学生**: `bufferedTurn := true` がコードにあるがコメントが無く、**「何のためのフラグか」** が不明。
- **プロ**: 迷路生成が深さ優先探索だけで、**「なぜ再帰的分割を選ばないのか」** が説明不足。
- **中学生**: プレイヤーとゴーストの速度が同じで、**「追いかけられない」** と感じる。

### rpg / command‑battle
- **中学生**: `iota` で列挙したステートが **「自動インクリメント」** とだけ書かれ、**「実際にどのステートが何を意味するか」** が不明。
- **プロ**: 敵探索で `maxFloat := math.MaxFloat64` を使用しているが、**`math.Inf(1)` の方が意図が明確**。
- **中学生**: 被ダメージがタイマー終了時にまとめて計算され、**「攻撃が遅延」** と不自然。

### visual‑effects (advanced) / vfx‑fx‑A01‑A12
- **中学生**: FX とゲームロジックが混在しているコードが多く、**「どこで何が描画されるか」** が読めない。
- **プロ**: `BlendLighter` の合成モード説明が無く、**「背景が白いと見えにくい」** という実装上の落とし穴がある。

## 4. インフラ・共通コンポーネント
- **HTML メタ情報**: 多くのページで `<meta charset="UTF-8">` が抜けている。文字化けリスク。
- **相対パス**: `href="../../../../play/<slug>/"` が深すぎて保守が困難。`<base href="/EbiShowcase/">` を `head` に入れ、**ルート相対** に統一すべき。
- **アクセシビリティ**: フィードバックフォームの `<input>` に `aria-label` が無く、**スクリーンリーダー不可**。
- **スクリプトデバッグ**: `scripts/gen-visual-effects.mjs` の `console.log` が本番に残る。ビルドフラグで除去。
- **画像最適化**: `web/assets/hero.png` が 1.2 MB。WebP へ変換し **30 % 以下** に削減すべき。
- **OG タグ**: 各ページに OG/Twitter カードが不完全。`og:image` がデフォルト画像のまま。
- **README**: ビルド手順に Go バージョンが明記されていない。`go 1.24` が必須である旨を追記。
- **CSRF 対策**: `scripts/insert-feedback-form.mjs` の `<form>` に CSRF トークンが無い。
- **Prism テーマ**: 暗すぎるテーマが多く、可読性が低い。明るめテーマへ変更。
- **タイトルプレフィックス**: `Ebi Showcase - ` が無いページが散在。SEO 向上のため統一。

## 5. まとめと優先度リスト
| 優先度 | カテゴリ | 問題点 | 推奨アクション |
|---|---|---|---|
| ★★★★★ | Core → LEVEL 01 | Update/Draw の説明が実装と不整合 | HTML と Go コードを完全同期、ラジアン変換式を追記 |
| ★★★★★ | Visual Effects → vfx‑transform | 実装順序が逆、単位が不一致 | 実装順序を修正、ラジアン⇔度変換表記を追加 |
| ★★★★ | All → iota の説明不足 | 中学生・プロ共に理解不能 | `iota` の概念と具体例を全ページに統一追加 |
| ★★★★ | Track → deck‑cycle | デッキリサイクルのメモリ管理が不安定 | `discard = make([]Card,0)` に変更、コメント追加 |
| ★★★ | インフラ → メタ情報・相対パス | 文字化け・保守性低下 | `<meta charset>` 追加、`<base>` タグ導入 |
| ★★★ | アクセシビリティ | フォームに aria-label が無い | `aria-label` を全入力欄へ付与 |
| ★★ | ビジュアル → 画像サイズ | hero.png が大きい | WebP 変換、圧縮率 30 % 以下 |
| ★★ | SEO → OG/Twitter | デフォルト画像が使われている | 各ページに固有 OG 画像生成スクリプトを拡張 |
| ★ | カラーマッピング | sRGB ↔ linear が無視されている | `op.ColorM` の使用例に変換式を追加 |
| ★ | コメント・ガイドライン | 多くのコードに説明不足 | 各関数・重要ロジックに行コメントを挿入 |
