# Ebi Showcase 全体フィードバックレポート

**対象:** `web/ja/` と `web/en/` の全教材ページ (HTML) と対応する Go ソースコード。中学生の読者とプロのプログラマーの両方が **『意味が通らない』** と感じる箇所を中心に指摘しています。

---

## 共通指摘 (全ページ)
- **タイトルタグが一貫していない** – 日本語ページは `Ebi Showcase | ...`、英語ページは `Ebi Showcase – ...` のように表記が混在。全ページで同一フォーマット (`Ebi Showcase – <Lesson Name>`) に統一すべき。
- **メタディスクリプションが短すぎる** – 80 文字未満が多く、検索エンジン最適化 (SEO) が不十分。各ページで 150‑160 文字程度に要点を入れる。
- **`<meta charset="utf-8">` が欠落** のページが散見。ブラウザの文字化け防止のため必ず最初に記述する。
- **ARIA ラベルが不足** – クリック可能なカードやボタンに `aria-label` が付いていない。キーボード・スクリーンリーダー利用者が操作できるよう、必ず付与する。
- **画像の代替テキストが空** – `alt=""` が多く、画像の意味が伝わらない。特にゲームのサムネイルは「<Lesson> デモ画面」など具体的に記述。
- **相対パスの深さが不統一** – `../../play/` と `../../../play/` が混在し、ページ移動時に 404 が起きやすい。`<base href="/">` を導入し、全リンクをルート相対に統一する。
- **`<canvas>` のサイズが固定** – `width="480" height="720"` がハードコーディングされているページが多数。レスポンシブ対応の CSS (`max-width: 100%; height: auto;`) を追加し、モバイルでも正しく表示させる。
- **日本語と英語の文体が不揃い** – 中学生向けにやさしい語彙を使うべきが、技術用語が無造作に混ざっている。**日本語は 2 年生程度の語彙、英語は A2 レベル** を目安に書き直す。
- **コードブロックのハイライトが欠けている** – `pre` にクラスが付いていないため、Go のシンタックスが見えにくい。Prism.js などのハイライトライブラリを全ページで有効化。
- **ページ内ナビゲーションが省略** – 前後レッスンへのリンクが無い、または `rel="prev"/"next"` が欠如。学習フローが途切れやすくなるので必ず設置する。
- **フォントサイズが小さすぎる** – スマートフォンで `12px` 以下になる箇所が多数。最小 `16px` に統一し、`rem` 単位で可変にする。
- **`<button>` が `type="button"` を明示していない** – フォーム送信時に意図しない送信が起こるリスクがある。
- **リンクが未翻訳の `README.md` へ直リンク** – 日本語ページから英語の GitHub リポジトリへ直接リンクすると、読者が言語切替に困惑する。言語別のガイドページへリダイレクトする。
- **OGP 画像が同一** – 全ページで同じ OG 画像を使用しており、SNS でのシェア時に情報量が低い。各レッスンのキャプチャ画像を自動生成し、`og:image` に差し替えるスクリプトを導入。
- **`<section>` の入れ子が不正** – HTML5 のアウトラインが崩れ、アクセシビリティ支援ツールが正しく解釈できない。`<article>`、`<section>` の階層を見直す。

## Core カリキュラム (LEVEL 01‑12) 代表的指摘
### LEVEL 01 – 「Update と Draw の基礎」
- **概念説明が抽象的** – 「`Update` はゲームの状態を変える」だけで、**「なぜ毎フレーム呼ばれるのか」** が未説明。フレームレートと時間管理の簡単な例を追加。
- **`main.go` のインポートが省略** – `ebiten` が暗黙的にインポートされているが、初心者は `import "github.com/hajimehoshi/ebiten/v2"` が必要だと混乱。コードブロックにインポート文を必ず掲載。
- **キーボード入力の説明が抜け** – `ebiten.IsKeyPressed` の使い方がなく、**「スペースキーでジャンプします」** とだけ書かれている。実装例とキーコードの意味を追記。
- **`Layout` の役割が省略** – PC とモバイルでキャンバスサイズが変わることを説明しないと、**「画面がずれる」** と不具合認識が遅れる。
- **デバッグ表示が無い** – 変数 `score` が増えても画面に表示されない。`ebitenutil.DebugPrint` の使用例を入れると学習効果が上がる。

### LEVEL 04 – 「衝突判定」
- **「矩形同士の交差」** とだけ書かれているが、**「座標系は左上が原点」** が抜けている。図解で左上 = (0,0) を示す。
- **`Rect` 構造体の定義が省略** – `type Rect struct { X, Y, W, H float64 }` がコードに出てこないと、**「Rect が何？」** と混乱。
- **衝突が起きた時のフィードバックが無い** – 音やエフェクトがないと、**「本当に当たったか？」** という疑問が残る。

### LEVEL 07 – 「アセットロード」
- **画像のパスがハードコード** – `"assets/hero.png"` が相対パスで記載。ビルド時に `go:embed` が必要だが触れていないため、**「画像が表示されない」** エラーが出る。
- **画像サイズが固定** – 96×96 のスプライトサイズについて説明がなく、**「スプライトがはみ出す」** と読者が試行錯誤する。
- **`ebiten.NewImageFromFile` が非推奨** – 最新 `ebiten.NewImageFromReader` の使用を推奨しないと、**「古い API でビルドが失敗」** になる。

### LEVEL 10 – 「スコア管理」
- **永続化の説明が抜け** – スコアをローカルストレージに保存しないので、リロードでリセットされる点が不自然。`localStorage` の代替案 (Go からは `js.Global().Get("localStorage")`) を簡単に示す。
- **整数オーバーフローの注意がない** – 大きなスコアで `int` が溢れる可能性を示すと、**「スコアが負になる」** バグ防止になる。

### LEVEL 12 – 「ゲームの終了とリスタート」
- **`ebiten.Game` の `Layout` が再度呼ばれる理由が不明** – ループが止まらない原因が説明不足。**「`ebiten.RunGame` がブロッキング」** と明記。
- **リスタート処理が `os.Exit(0)`** で実装されている場合、**「ブラウザがクラッシュ」** になる。`NewGame` の再生成手順を示す。

## Visual Effects Lab (VFX) 代表的指摘
- **エフェクト名が英語だけ** – `vfx-fire`、`vfx-ice` など、和訳が無く中学生が意味を把握できない。日本語ラベル（炎、氷）を併記。
- **スライダーの単位が不明** – `speed` スライダーに単位が無い。**「0.0–2.0（倍速）」** と明示する。
- **パラメータ変更が即時反映しない** – `requestAnimationFrame` のタイミングが遅く、**「スライダーを動かしたのに変わらない」** と混乱。デバッグログを残す。
- **タッチ操作が未対応** – スマホでスライダーが操作できない。`pointerdown` と `pointermove` の併用例を追加。
- **エフェクトの組み合わせが不可能** – `vfx-fire` と `vfx-ice` 同時適用ができない設計。コンポジットパイプラインの説明を入れると上級者向けになる。

## ジャンルトラック（例示）
### Platformer – 「ジャンプと重力」
- **重力定数がハードコード** – `const gravity = 0.5` が何を意味するか説明が無い。**「0.5 ≈ 0.5px/frame²」** と書く。
- **踏み台の当たり判定が不正確** – 頂点が少し浮く。**「`player.Y + player.Height == platform.Y`** が必要** とコメントで示す。
- **ステージデータが JSON で埋め込まれているが、フォーマットが不明** – 例として `{ "x":0, "y":100 }` だけで、**「どのプロパティが必須か」** がわからない。

### Puzzle – 「ブロックの回転」
- **回転角度がラジアンで記述** – 中学生は **「90°」** が直感的。`ebiten.GeoM.Rotate(angle * math.Pi / 180)` の変換式を必ず示す。
- **回転中心が左上** – デフォルトが左上であることが不明。**「`GeoM.Translate(-originX, -originY)` が必要」** と明記。

### Shooter – 「弾丸プール」
- **プールサイズが固定** – `maxBullets = 10` が少なすぎるケースがある。**「画面サイズと同倍」** に調整すべき根拠を示す。
- **弾丸が画面外に出ても削除されない** – メモリリークになる。`if bullet.Y < 0 { bullet.Active = false }` のように境界チェックを必ず入れる。

### Rhythm – 「ビート同期」
- **BPM がハードコード** – 120 がデフォルトだが、**「BPM の計算式は `60 / interval`**」を解説。
- **オーディオコンテキストの初期化が遅延** – ユーザー操作前に `AudioContext` を作成しようとするとブラウザがブロック。**「タップイベントで開始」** と指示。

## その他の重要改善点
- **用語集へのリンクが不足** – `Glossary.md` があるが、本文中に `[用語集](/docs/Glossary.md)` のリンクがほぼ無い。全ページの重要キーワード（`Update`, `Draw`, `GeoM`, `SpriteSheet` など）にフットノートでリンク付与。
- **コードスニペットが古い Go バージョン向け** – `go.mod` は 1.24 だが、 `ioutil.ReadFile` が残っている。`os.ReadFile` に置換し、**「Go 1.24 で非推奨」** と注記。
- **エラーハンドリングが省略** – `err != nil` のチェックがコメントアウトされているページがある。実装例として `if err != nil { log.Fatal(err) }` を必ず記載。
- **コメントが日本語と英語混在** – 同じコメント行に両言語が入ると読者が混乱。**日本語ページは日本語のみ、英語ページは英語のみ** に統一。
- **ビルドエラーが出やすいパス** – `go run ./cmd/gen-atlas` が `internal/heroatlas` に書き込み権限を要求。README に **「実行前に `chmod -R u+w internal/heroatlas`」** を追記すべき。
- **テストコードが不十分** – `*_test.go` が 5 行未満の簡易テストだけ。**「境界テスト」** と **「入力乱数固定」** を追加し、テストカバレッジを 80 % 以上に保つ。
- **Open Graph の URL が絶対パス** – `og:url` が `/games/...` となっており、GitHub Pages で正しく解決しない。`https://kumagi.github.io/EbiShowcase/games/...` のように `SITE_ORIGIN` を利用。
- **HTML のインデントが崩れている** – `</section>` の直前に余計な空白が入り、ビューソースが読みにくい。自動整形ツール (prettier) を導入。
- **`<script>` がインラインで長大** – `web/learn.js` の一部がページごとに埋め込まれ、重複コードが増える。**外部ファイルに分離** し、`defer` 属性で読み込み。
- **`<meta name="robots" content="noindex">` が残っている** – デモページが検索エンジンにインデックスされていない。**`index,follow`** に変更。

## まとめ
- 全ページで **「中学生が『何が起きているか分からない』** と感じる抽象的記述** を具体例・図解・数式で補完。
- **プロのプログラマーが『型の不一致や非推奨 API』** と怒る箇所を最新 Go 1.24 のベストプラクティスに合わせて修正。
- アクセシビリティ（ARIA、代替テキスト、キーボード操作）の欠如を全頁で統一的に対策。
- SEO・OGP・リンク構造の統一で、外部からの流入と SNS 共有を最適化。
- 用語集・コードスニペット・テスト・ビルド手順を更新し、**学習体験の質を総合的に向上** させる。
