GRADUATION 01

赤いテストを、60秒ゲームに変える。

空の Go 作業フォルダで始めます。スターターを3ファイルだけブラウザから保存し、失敗するテストを TODO の順に緑へ変えます。

0. DOWNLOAD

clone せずに始める

空のフォルダで下のコマンドを実行し、starter の3ファイルを Raw 表示から同じフォルダへ保存します。

mkdir arcade-60 && cd arcade-60; go mod init example.com/arcade-60; go get github.com/hajimehoshi/ebiten/v2; go test

最初の失敗は正解です。 まだ reference は開きません。

1. SCORE

最初の赤を緑にする

TestActionAddsOneStar が失敗したら、rules.go の TODO 4 です。Input{Action: true} を受け取ったときだけ Score に10を足します。キーやクリックを読むのは main.go、点数の規則は Round.Step です。

go test -run TestActionAddsOneStar

2. TIMER

60秒で終わらせる

TestTimeLimitEndsRound は TODO 5 に対応します。毎回の Step で Frames を一つ増やし、RoundFrames に届いたら Over を true にします。Draw は Over を見て結果を表示するだけです。

go test -run TestTimeLimitEndsRound

3. RESTART

もう一回を作る

TestRestartMakesFreshRound は TODO 3 に対応します。Rキーで NewRound() の状態へ戻します。古い Score と Frames が残らないことをテストが守ります。

go test; go run .

ADD ONE RULE

コンボ、ミスの減点、残り10秒の演出などを一つ選び、先にテストを書きます。

KEEP IT PURE

得点と時間の規則は rules.go、入力と絵は main.go。窓を毎回開かずに規則を確かめられます。

COMPARE LAST

全テストが緑になってからだけ reference を開き、構成の違いを観察します。写しません。
