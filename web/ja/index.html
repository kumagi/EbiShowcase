はじめて読む人へ

ここを先に見よう

ゲームループは、Updateが約60回/秒の時計として数字を進め、Drawがその最新状態を絵にする仕組みです。Drawが遅くてもゲーム進行は止まりません。海老・天次郎（えび・てんじろう）が的を追う最小例から始めます。

このページの1本
説明と次のコードを対応させてから、ゲームで変化を確かめよう。

func (g *Game) Update() error { g.x++; return nil }
func (g *Game) Draw(screen *ebiten.Image) { drawTarget(screen, g.x) }

BASIC / 基礎編

簡単な一歩から、
本格的なゲームへ。

最初にゲームループを覚え、あとは1レベルにつき新しい考え方を1〜2個だけ足します。前のゲームの部品が、次でもそのまま使えます。

01 / 60 FRAMES

ゲームは、1秒に約60枚のパラパラ漫画。

1コマごとに Update が位置や点数の数字を進め、続けて Draw がその数字どおりの絵を描きます。テレビやアニメが短い時間に何枚もの絵を切り替えるのと同じです。

02 / STATE FIRST

ルールは Update、Draw は写すだけ。

入力、得点、衝突、乱数は Update で決めます。Draw はその結果を画面へ写し、数字や状態を変えません。

03 / COLLISION

当たり判定は「2つが重なった？」の質問。

丸と指、カゴと星、主人公と壁。2つの形が重なったかを数字で調べ、重なっていれば得点や停止などのルールを動かします。
