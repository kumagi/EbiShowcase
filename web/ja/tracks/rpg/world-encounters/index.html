シーン変数が
マップと戦闘を分ける

シーン変数 は状態機械の兄弟です。scene == mapScene ならフィールドの移動とエンカウント判定を動かし、scene == battleScene なら戦闘UIを表示します。シーンが変わるだけで入力もグラフィックも丸ごと切り替わります。敵テーブル は地域ごとの敵スライスで、プレイヤーのX座標から地域を計算してテーブルを選びます。

1

シーン変数で画面を切り替える

scene に mapScene=0 か battleScene=1 を入れるだけで、Update と Draw が完全に別のコードを実行します。戦闘が終わったら scene = mapScene に戻すだけです。

g.scene = battleScene

歩数で出現確率を制御する

毎歩 steps++ し、5の倍数のときだけ乱数でエンカウント判定します。確率を 65（65%）に設定しています。数字を変えるだけでエンカウント率を調整できます。

if g.steps%5 == 0 && rng.Intn(100) < 65

バイオーム判定は biome() 関数が担います。X座標が4未満なら草原（0）、7未満なら砂漠（1）、以降は雪山（2）です。座標を見るだけでテーブル番号が決まります。

IN THE REAL GAME

エンカウント判定とシーン切り替え

移動するたび歩数を増やし、条件を満たしたときにバイオームから敵を選んでシーンを切り替えます。戦闘が終わると scene = mapScene に戻り、マップの操作が再開されます。

if dx != 0 || dy != 0 {
    g.steps++
    if g.steps%5 == 0 && g.rng.Intn(100) < 65 {
        biome := g.biome()
        list := tables[biome]
        g.foe = list[g.rng.Intn(len(list))]
        g.scene = battleScene
    }
}
func (g *game) biome() int {
    if g.x < 4 { return 0 }  // 草原
    if g.x < 7 { return 1 }  // 砂漠
    return 2                    // 雪山
}

WITHOUT SCENE VARIABLE

シーンなしだと？

マップ移動と戦闘をフラグで切り替えると、条件分岐がどんどん増えます。シーン変数1つで「今どこにいるか」を明確にすることで、コードが整理されます。

WITH ENEMY TABLES

テーブルで地域感を演出する

テーブルを変えるだけで「この森にはゴブリンが多い」「砂漠にはサソリが出る」という世界観を作れます。コードを増やさず、データを追加するだけです。

YOUR FIRST TUNING

バイオームの境界を変えてみよう

biome() の g.x < 4 を g.x < 6 に変えると草原が広くなります。敵テーブルに新しい敵構造体を加えれば、その地域の出現ラインナップも変わります。
