ショップは
3つの判定だけ

ショップ処理のルールはシンプルです。①すでに装備済みか → 断る、②お金が足りるか → 足りなければ断る、③スロットの種類で分岐 → 武器はフラグ、盾はフラグ、消耗品は個数を増やす。この3ステップを守れば、どんな複雑なショップも書けます。

1

商品をスライスで定義する

商品情報を構造体のスライスにまとめます。名前・スロット・価格・効果値をセットで持つと、ループで画面描画とロジック両方に使えます。

shop = []item{{"Coral Sword", "weapon", 60, ...}}
2

装備スロットをboolで管理

武器と盾はそれぞれ bool で持ちます。sword = true にしたら以後は「すでに装備済み」と判定できます。個数制限がある消耗品はカウンターで管理します。

g.sword = true
3

ゴールドを引いて購入する

購入可能と判定したら gold -= price し、スロットを更新します。この順序が重要です。先にスロットを更新してからゴールドを引くと、お金チェックのバグで二重購入が起きます。

g.gold -= it.price

THE PURCHASE RULE

owned := (choice == 0 && g.sword) || (choice == 1 && g.shield) すでに持っている？ if owned { message = "already equipped" } お金が足りる？ else if g.gold < it.price { message = "not enough gold" } 購入成立 else { g.gold -= it.price; ... }

スロット別の分岐は switch choice で書きます。武器スロットは g.sword = true、消耗品は g.potions++ と、スロットの性質に合わせて処理を変えます。

IN THE REAL GAME

購入判定とスロット更新

プレイヤーが商品を選ぶと、所持チェック・ゴールドチェックを順に行い、どちらも通過したら購入します。装備は bool、消耗品はカウンターで管理しているため、スロットごとに違う処理が必要です。

owned := (choice == 0 && g.sword) || (choice == 1 && g.shield)
if owned {
    g.message = it.name + " is already equipped."
} else if g.gold < it.price {
    g.message = "Not enough gold."
} else {
    g.gold -= it.price
    switch choice {
    case 0: g.sword = true
    case 1: g.shield = true
    case 2: g.potions++
    }
    g.message = "Bought " + it.name + "!"
}

WITHOUT SLOT CHECK

チェックなしだと二重購入される

所持チェックがないと、同じ装備を何度でも買えてしまいます。ゴールドが無限に減り、ゲームが壊れます。

WITH SLOTS

スロット種別で柔軟に拡張できる

武器・防具は bool、消耗品はカウンターという区別を最初から設計しておくと、後で「装備を外して売る」機能を追加するときも構造が崩れません。
