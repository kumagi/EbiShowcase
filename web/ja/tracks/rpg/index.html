ドラゴンクエスト型ゲームを作る

コマンドRPG

ドラゴンクエストのような、コマンドを選んで進めるRPGを作ります。町の探索、会話から始め、順番に「たたかう」などのコマンドを選ぶ戦闘（ターン制戦闘）、成長、装備、世界地図を順に統合します。

共通基礎の続きです。LEVEL 01 の Update / Draw の上に、このジャンルの部品を1ステップずつ足します。主人公はオリジナルの 海老・天次郎（えび・てんじろう） です。

はじめて読む人へ

このコースの組み立て図

村→会話→戦闘→持ち物の順に、海老・天次郎（えび・てんじろう）と一部品ずつ作ります。 このページの1本はコース全体で共通するUpdate→Drawの骨格です。下の01から順に、その中へ各仕組みを実装していきます。

このページの1本
説明と次のコードを対応させてから、ゲームで変化を確かめよう。

func (g *Game) Update() error { return nil }

LEARNING PATH

小さな完成を
積み重ねる。

各ステップは単独で遊べるミニゲームです。前の仕組みを再利用しながら、最後に1本の作品へ統合します。

01

小さな村を歩く

マス目の村を歩き、壁を通れないようにし、向きを記録します。

タイル衝突と向き
→ 02

村人との会話

話しかけた相手や出来事によって、会話の内容を変えます。

文章送りとイベントフラグ
→ 03

1対1コマンドバトル

「たたかう」「ぼうぎょ」「どうぐ」などのコマンドを選び、味方の行動、敵の行動、勝敗判定の順番で進めます（この決まった手順を状態機械と呼びます）。

順番にコマンドを選ぶ戦闘の流れ（ターン制・状態機械）
→ 04

能力値と状態変化

攻撃力、防御力、素早さ、毒や強化を計算へ組み込みます。

ダメージ式と一時効果
→ 05

装備屋と道具袋

道具の個数、装備スロット、お金、店の価格を管理します。

所持品・装備・売買
→ 06

世界地図とエンカウント

町、フィールド、戦闘の画面を切り替え、地域ごとに決まった敵の一覧からランダムに1体を選びます。

画面の切り替えと、敵の種類・強さの一覧（敵テーブル）
→ 07

敵の次の行動を読む

敵の攻撃予告を見て、攻撃するか防御するかを選ぶ戦闘を作ります。

予告がコマンド選択を作戦に変える
→ 08

攻撃を3つの時間に分ける

踏み込み、命中、戻りの中間フレームに、点滅・粒子・揺れを合わせます。

予備動作・接触・余韻
→ 09

Ebi Quest

仲間、目的、進行状況、セーブデータを統合して短編RPGを完成させます。

仲間・クエスト・セーブ
