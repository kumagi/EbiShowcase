DEEP DIVE / STATE MACHINE

状態機械が
ターンを整理する

状態機械 とは「今どの状態にいるか」を1つの変数で管理し、状態ごとに異なる処理をする仕組みです。この戦闘は command → playerTurn → enemyTurn → result の4状態を順番に移り変わります。どの状態にいるかさえ分かれば、入力の受け付けと演出タイマーをシンプルに書けます。

1

状態を定数で定義する

iota は定数ブロックで「0から1ずつ増える番号」を自動で振る機能です。command = iota の次の行は自動で 1、その次は 2…となります。自分で 0,1,2,3 と書かなくても、状態の名前だけ並べれば連番になります。

state = command
2

状態ごとに入力と演出を分ける

switch g.state で分岐し、command 状態だけキー入力を受け付けます。他の状態ではタイマーを減らし、アニメーション中であることを示します。

switch g.state { case command: ... }
3

タイマーで演出を作る

行動後に timer = 35 をセットし、毎フレーム1ずつ減らします。タイマーがゼロになったとき次の状態へ移ります。待ち時間の数字を変えるだけで演出の長さを調整できます。

g.timer--; if g.timer <= 0 { ... }

ガードを選ぶと guard = true フラグが立ち、敵ターンのダメージを半分にします。前のステップで学んだフラグがそのまま使えています。

IN THE REAL GAME

ダメージは演出の「始まり」で入れる

Update() は毎フレーム呼ばれます。コマンドを選んだ瞬間に味方の効果を入れ、playerTurn のタイマーは演出用です。敵のダメージも enemyTurn に入った瞬間（攻撃モーション開始）に入れ、残りタイマーは余韻だけにします。タイマーが0になってから初めて HP を減らすと、モーションと数字のタイミングがずれて不自然になります。

const (
    command = iota // 0
    playerTurn     // 1（自動で+1）
    enemyTurn      // 2
    result         // 3
)

switch g.state {
case command:
    if c == 0 {  // 攻撃を選んだ＝味方ターン開始
        d := 8 + g.rng.Intn(7)
        g.enemyHP -= d          // ダメージはここで
        g.state = playerTurn
        g.timer = 35            // あとは演出
    }
case playerTurn:
    g.timer--
    if g.timer <= 0 {
        if g.enemyHP <= 0 { g.clear = true } else {
            d := 7 + g.rng.Intn(8)
            g.hp -= d           // 敵ターン開始＝ヒットの瞬間
            g.state = enemyTurn
            g.timer = 40
        }
    }
case enemyTurn:
    g.timer--                   // ダメージ済み。余韻だけ待つ
    if g.timer <= 0 { g.state = command }
}

WITHOUT STATE MACHINE

状態なしだと？

if 文を組み合わせるだけでは、「コマンド選択中に敵が動いてしまう」「演出中に入力できてしまう」など、タイミングのバグが生まれます。

WITH STATE MACHINE

状態変数1つで整理できる

現在の状態だけを見れば、何が起きていいか・悪いかが明確です。アニメーション中は入力を無視し、コマンド中は敵が動かない、が自然に実現します。

YOUR FIRST TUNING

タイマーの長さを変えてみよう

g.timer = 35 や g.timer = 40 の数字を大きくすると演出がゆっくりに、小さくすると素早い戦闘になります。テンポを調整してみましょう。
