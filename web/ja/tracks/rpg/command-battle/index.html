DEEP DIVE / STATE MACHINE

状態機械が
ターンを整理する

状態機械 とは「今どの状態にいるか」を1つの変数で管理し、状態ごとに異なる処理をする仕組みです。この戦闘は command → playerTurn → enemyTurn → result の4状態を順番に移り変わります。どの状態にいるかさえ分かれば、入力の受け付けと演出タイマーをシンプルに書けます。

1

状態を定数で定義する

iota で連番の定数を作ります。command=0, playerTurn=1, enemyTurn=2, result=3 の4つです。変数1つで「今どの段階か」を管理できます。

state = command
2

状態ごとに入力と演出を分ける

switch g.state で分岐し、command 状態だけキー入力を受け付けます。他の状態ではタイマーを減らし、アニメーション中であることを示します。

switch g.state { case command: ... }
3

タイマーで演出を作る

行動後に timer = 35 をセットし、毎フレーム1ずつ減らします。タイマーがゼロになったとき次の状態へ移ります。待ち時間の数字を変えるだけで演出の長さを調整できます。

g.timer--; if g.timer <= 0 { ... }

THE STATE FLOW

command → (コマンド選択) → playerTurn timer 消化後 playerTurn → (HPチェック) → enemyTurn or result timer 消化後 enemyTurn → (HPチェック) → command or result

ガードを選ぶと guard = true フラグが立ち、敵ターンのダメージを半分にします。前のステップで学んだフラグがそのまま使えています。

IN THE REAL GAME

switch で状態を切り替える

Update() は毎フレーム呼ばれます。command 状態でキーを押すとダメージ計算をして playerTurn へ移行し、タイマーをセットします。タイマーが尽きると敵ターンへ移り、敵がダメージを与えて次のコマンド入力に戻ります。

switch g.state {
case command:
    if c == 0 {  // 攻撃
        d := 8 + g.rng.Intn(7)
        g.enemyHP -= d
        g.state = playerTurn
        g.timer = 35
    }
case playerTurn:
    g.timer--
    if g.timer <= 0 {
        if g.enemyHP <= 0 { g.clear = true }
        else { g.state = enemyTurn; g.timer = 40 }
    }
case enemyTurn:
    g.timer--
    if g.timer <= 0 {
        d := 7 + g.rng.Intn(8)
        if g.guard { d = (d + 1) / 2 }
        g.hp -= d
        g.state = command
    }
}

WITHOUT STATE MACHINE

状態なしだと？

if 文を組み合わせるだけでは、「コマンド選択中に敵が動いてしまう」「演出中に入力できてしまう」など、タイミングのバグが生まれます。

WITH STATE MACHINE

状態変数1つで整理できる

現在の状態だけを見れば、何が起きていいか・悪いかが明確です。アニメーション中は入力を無視し、コマンド中は敵が動かない、が自然に実現します。

YOUR FIRST TUNING

タイマーの長さを変えてみよう

g.timer = 35 や g.timer = 40 の数字を大きくすると演出がゆっくりに、小さくすると素早い戦闘になります。テンポを調整してみましょう。
