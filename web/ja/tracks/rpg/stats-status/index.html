DEEP DIVE / DAMAGE FORMULA

ダメージは
引き算で決まる

RPGの基本公式は ダメージ = 攻撃力 − 防御力 です。これに「強化中は攻撃+8」「毒でターン毎に3ダメージ」という 一時効果カウンター を加えると、状態変化のある戦闘が完成します。カウンターを毎ターン1ずつ減らし、ゼロになったら効果が消えます。

1

ダメージ式

攻撃から防御を引き、乱数で幅を持たせます。結果が1未満になっても最低1ダメージは保証します。max(1, ...) を使うと「防御が高すぎてノーダメ」を防げます。

d := max(1, atk - def + rand)
2

強化カウンター

パワーアップを選ぶと buff = 3 をセット。攻撃時に buff > 0 なら攻撃力を加算し、ターン末に buff-- します。3ターンで自然消滅します。

if g.buff > 0 { atk += 8 }
3

毒カウンター

3ターンごとに毒が付きます。毒中は毎ターン3ダメージを受け、poison-- します。解毒剤を使うと即座に poison = 0 にリセットします。

if g.poison > 0 { g.hp -= 3; g.poison-- }

TRY IT / LAB

攻撃−防御（強化込み）

基本は攻撃−防御。一時強化は攻撃側に足してから計算します。

本物のゲームの公式を、手でゆっくり回す模型です。

max(1, atk+buff-def)
攻撃10
防御3
ダメージ7

THE DAMAGE FORMULA

atk := g.attack // 基本値 15 強化中なら加算 if g.buff > 0 { atk += 8 } 防御を引いて乱数で幅を付ける d := max(1, atk - 6 + g.rng.Intn(5))

このゲームでは敵の防御力が固定値 6 です。プレイヤーの攻撃力 15 − 防御 6 = 最小9ダメージ。強化すると最小17ダメージになります。数字を変えるだけでバランスが変わります。

IN THE REAL GAME

攻撃、強化、毒の処理を一度に見る

コマンド選択後、プレイヤーの行動を処理し、次に敵の攻撃と毒・強化カウンターの更新を行います。カウンターは整数なので、値を見れば「あと何ターン効果が続くか」が一目で分かります。

// プレイヤーが攻撃を選んだとき
atk := g.attack
if g.buff > 0 { atk += 8 }
d := max(1, atk-6+g.rng.Intn(5))
g.enemyHP -= d

// 敵の反撃と状態更新
enemyAttack := 13 + g.rng.Intn(5)
g.hp -= max(1, enemyAttack-g.defense)

if g.turn%3 == 0 { g.poison = 4 }
if g.poison > 0 { g.hp -= 3; g.poison-- }
if g.buff > 0 { g.buff-- }

WITHOUT FORMULA

固定ダメージだと？

毎回同じダメージでは戦略が生まれません。攻撃力・防御力の差で結果が変わることで、装備や強化を選ぶ意味が出てきます。

WITH COUNTERS

カウンターで効果に期限をつける

永続する強化はゲームを壊します。カウンターを使えば「あと2ターン」という有限の効果が自然に実現し、タイミングを考える戦略が生まれます。

YOUR FIRST TUNING

強化の期間と威力を変えてみよう

buff = 3 を buff = 5 に、atk += 8 を atk += 12 に変えると、強化がより強く長続きします。バランスを崩すほど変えて、RPGの数字設計を体感しましょう。
