DEEP DIVE / TILE COLLISION

動く前に、
次のマスを調べる

マップは 文字列の配列 です。'#' が入ったマスは壁、それ以外は通路です。プレイヤーが動くとき、まず「移動先のマスが壁か？」を確かめ、通路のときだけ座標を変えます。向きは壁の手前でも更新するので、壁の向こうのNPCにも後から話しかけられます。

1

マップを数字の表で持つ

文字列スライスをそのままマップに使います。'#' は壁、スペースは通路。インデックスが座標になるので変換が要りません。

rows[y][x] == '#'
2

移動前に壁チェック

入力から向きベクトル d を作り、現在地に足して次のマス n を得ます。壁でなければ座標を更新します。

n := point{p.x+d.x, p.y+d.y}
3

向きは常に更新する

壁に当たっても向きだけは変えます。向きを保存しておくと、後のステップでキャラが「どちらを向いているか」を使った話しかけが書けます。

g.facing = d

THE MOVEMENT RULE

g.facing = d 次のマスを計算 n := point{g.p.x + d.x, g.p.y + d.y} 壁でなければ移動 if !g.walls[n] { g.p = n }

向きを先に更新してから壁を調べることで、「壁の向こうへ話しかける」とき、プレイヤーの座標ではなく facing を使うだけで自然に実現できます。

IN THE REAL GAME

入力 → 向き更新 → 移動判定

実際のコードは入力を向きベクトルに変換し、向きを記録してから移動できるか判定します。星を踏んだら収集数を増やし、ほこらに到着して3つ集まっていたらクリアです。

if d.x != 0 || d.y != 0 {
    g.facing = d
    n := point{g.p.x + d.x, g.p.y + d.y}
    if !g.walls[n] {
        g.p = n
        if g.stars[n] {
            delete(g.stars, n)
            g.collected++
        }
    }
}
if g.p == (point{8, 1}) && g.collected == 3 {
    g.clear = true
}

WITHOUT TILE CHECK

壁チェックなしだと？

座標をそのまま増減すると、壁の中に入ったり、マップ外にはみ出したりします。ゲームが壊れる原因になります。

WITH FACING

向きを保存する利点

後のステップでNPCに話しかけるとき、facing を足すだけで「正面のマス」が分かります。複雑な計算は不要です。

YOUR FIRST TUNING

rows の文字列を書き換えると、通路の形がそのままゲームに反映されます。# で壁を増やし、迷路を作ってみましょう。
