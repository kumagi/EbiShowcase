はじめてのお店

お菓子を貯めてオーブンを買う。買うには「持っているか？」という条件を調べ、足りなければ灰色にして弾きます。クリッカーの「お店」の基本形です。

このステップで学ぶこと購入条件とコスト上昇

このコースも LEVEL 01 で学んだ Update（数字）→ Draw（絵） のくり返しの上に、新しい部品を1つ足していきます。

PLAYABLE / WEBASSEMBLY

オーブンを5台集めよう。

上半分をタップでお菓子を焼き、下のボタンで買います。Bキーでも購入できます。

● LIVE / GO + WASMSTEP 02

DEEP DIVE / SHOP

なぜ「買えない」ときは
ボタンが暗くなるの？

ボタンの色は sweets >= cost という条件で決まります。お菓子が足りないと暗い色を塗り、足りると明るい色を塗る。これだけで「買える／買えない」の視覚フィードバックが完成します。

1

購入条件チェック

押したときに「今のお菓子の数がコスト以上か」を調べます。条件を満たしていなければ購入処理をスキップし、代わりにエラーフラッシュをセットします。

if g.sweets >= g.cost { … }
2

タッチ位置で領域を分ける

マウスやタッチの y 座標が 470 より上ならベイク、下ならショップ。同じ「押した」イベントを座標で振り分けることで、ボタンなしで2つの操作を分けられます。

if y < 470 { bake } else { buy }
3

コストの段階的上昇

買うたびに cost += 5 します。初期値10、1台目は15、2台目は20…と増えるので、同じペースで押し続けるだけでは全台買えなくなります。これがクリッカーの「進む感」の仕組みです。

g.cost += 5

TRY IT / LAB

足りるときだけ買う

足りるときだけ購入し、値段を上げます。足りなければ何もしません。

本物のゲームの公式を、手でゆっくり回す模型です。

if gold >= cost

THE SHOP RULE

sweets >= cost → 購入OK：sweets -= cost; ovens++否sweets < cost → 購入NG：エラーフラッシュ

フラッシュ値 flash は正なら成功の光、負なら失敗の赤テキストを出します。毎フレーム絶対値を1減らしてフェードアウトします。

IN THE REAL GAME

条件確認 → 購入 → コスト更新

y 座標で操作を分岐し、ショップ領域なら sweets がコストを上回るか確認。成功したら sweets を減らしてオーブンを増やし、次の cost を加算します。

if ok {
    if y < 470 {
        g.sweets++
        g.flash = 8
    } else if g.sweets >= g.cost {
        g.sweets -= g.cost
        g.ovens++
        g.cost += 5
        g.flash = 12
    } else {
        g.flash = -15   // 失敗フラッシュ
    }
}

WITHOUT COST CHECK

お金がなくてもタダで買える

条件なしに ovens++ するだけだと、無限に増やせます。ゲームとして成立しません。

WITH RISING COST

後半が自然に難しくなる

固定コストなら全台買ったら終わりですが、段階的に上がると最後の1台が最も大変になります。プレイ時間が伸びます。

YOUR FIRST TUNING

コスト増加量を変えよう

g.cost += 5 の 5 を大きくすると難度が増し、小さくすると簡単になります。クリア条件 5 台も変えてみましょう。
