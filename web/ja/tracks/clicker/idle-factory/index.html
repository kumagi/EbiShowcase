DEEP DIVE / DELTA TIME

なぜ放置してても
お菓子が増えるの？

毎フレーム「前のフレームからの経過秒数 dt」を求め、machines × dt をお菓子に足します。60fps でも30fps でも同じ速度で増えるのは、フレーム数ではなく秒数を基準にしているからです。

1

デルタタイム（dt）

前フレームの時刻を last に保存し、今フレームとの差を秒で計算します。60fps なら約0.0167秒、重い処理で30fpsに落ちても約0.033秒になります。

dt := now.Sub(g.last).Seconds()
2

毎秒生産量（CPS）

「1台のオーブンが1秒に1個作る」という定義にすると、machines × dt 個/フレームで足し算できます。台数を増やすほど1秒あたりの増加量が増えます。

g.sweets += float64(g.machines) * dt
3

dt の上限キャップ

ブラウザのタブを隠すと長時間 Update が呼ばれず、復帰した瞬間に巨大な dt になることがあります。0.25 秒でキャップして、一気に何百個も増えないようにします。

if dt > .25 { dt = .25 }

TRY IT / LAB

時間差分で自動生産

経過時間 dt に機械数をかけて自動生産します。

本物のゲームの公式を、手でゆっくり回す模型です。

points += rate * dt
ポイント0.0
生産2/s
dt0.25

THE IDLE FORMULA

増加量 = machines × dt例：台数3、dt＝0.0167秒0.0167 × 3 ≈ 0.050 個/フレーム

1秒間に60フレームあるとすると、0.050 × 60 ≈ 3.0 個/秒。台数＝CPS になります。

IN THE REAL GAME

時刻を記録 → dt を計算 → 生産する

Update の先頭で now を取得し、g.last と差分を計算。dt を 0.25秒でキャップしてから sweets に足します。その後 g.last を now に更新して次フレームに備えます。

now := time.Now()
dt := now.Sub(g.last).Seconds()
g.last = now
if dt > .25 {
    dt = .25
}
g.sweets += float64(g.machines) * dt

WITHOUT DELTA TIME

fps が変わると速度が変わる

毎フレーム固定値を足す方法だと、60fps と30fps で生産速度が2倍変わります。重いPCでは遅く、軽いPCでは速くなってしまいます。

WITH DELTA TIME

どの環境でも同じ速度

経過秒数をベースにすることで、fps がどう変動しても1秒あたりの増加量が一定になります。モバイルでも一致します。

YOUR FIRST TUNING

コスト増加量と初期コストを変えよう

コスト上昇幅 += 10 や初期値 15 を調整すると、難度のバランスが変わります。CPS が machines × 1 ではなく machines × 2 にしてみましょう。
