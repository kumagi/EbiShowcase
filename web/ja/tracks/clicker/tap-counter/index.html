DEEP DIVE / INPUT

なぜタップした瞬間だけ、
数が増えるの？

Ebitengine の inpututil.IsMouseButtonJustPressed は、そのフレームで初めてボタンが押されたときだけ true を返します。長押しでもtickごとに増えない理由はここにあります。

1

「押した瞬間」を見つける

IsPressed は押し続けている間ずっと true ですが、JustPressed は押し始めた1 tickだけ true です。クリッカーでは JustPressed を使わないと、ボタンを押しっぱなしにするだけで数が増え続けてしまいます。

inpututil.IsMouseButtonJustPressed(...)
2

カウンター変数

「今何個か」を表す整数変数 count に 1 を足すだけです。表示のときは fmt.Sprintf("%03d", g.count) で 3桁ゼロ埋めにすると、数字の位置がずれず読みやすくなります。

g.count++
3

パルスで視覚フィードバック

押した瞬間に pulse = 10 をセットし、tickごとに 1 ずつ減らします。その値で円の半径を少し大きくすると「ぷにっ」と膨らむアニメになります。

scale := 1 + float64(g.pulse)*.012

THE INPUT RULE

IsPressed → 押している間ずっと truevsIsJustPressed → 押し始めた1 tickだけ true

タッチ対応は inpututil.AppendJustPressedTouchIDs(nil) で取得します。スマホでも同じ「瞬間」ルールが使えます。

IN THE REAL GAME

押した → 数える → 光らせる

pressed() 関数がキー・マウス・タッチをまとめて調べます。true なら count を増やし、pulse をセット。30個になったら clear フラグを立てます。

func pressed() bool {
    return inpututil.IsKeyJustPressed(ebiten.KeySpace) ||
        inpututil.IsMouseButtonJustPressed(ebiten.MouseButtonLeft) ||
        len(inpututil.AppendJustPressedTouchIDs(nil)) > 0
}

if pressed() {
    g.count++
    g.pulse = 10
    if g.count >= 30 {
        g.clear = true
    }
}

WITHOUT JUSTPRESSED

押しっぱなしで∞カウント

IsPressed を使ってしまうと、ボタンを1秒握るだけで60回カウントされ、ゲームにならなくなります。

WITH JUSTPRESSED

1タップ = 1個の保証

JustPressed は同じtickには2度目が来ません。どんな入力デバイスでも「1操作 = 1イベント」が成立します。
