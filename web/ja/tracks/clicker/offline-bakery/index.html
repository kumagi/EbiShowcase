DEEP DIVE / SAVE & OFFLINE

閉じている間も
お菓子が増えるのはなぜ？

ゲームを終了する瞬間、お菓子の数・オーブン台数・現在時刻を localStorage に書き込みます。次に起動したとき、保存した時刻と現在時刻の差（留守時間）に生産速度を掛けて、不在中の生産分を一括で加算します。

1

localStorage でセーブ

WebAssembly からは syscall/js でブラウザのJSを呼べます。js.Global().Get("localStorage") で localStorage を取得し、setItem / getItem でキーと値を読み書きします。

s.Call("setItem", prefix+"sweets", value)
2

オフライン経過時間の計算

保存時の Unix タイムスタンプを読み出し、現在時刻との差を秒で求めます。上限は8時間（28800秒）。あまり長くしすぎると、久しぶりに戻った瞬間の「ボーナス」が大きすぎてバランスを壊します。

away := math.Min(8*3600, float64(now-stamp))
3

120tickごとの自動セーブ

tickごとにセーブすると localStorage の書き込みが多すぎます。フレームカウンターが 120 の倍数のときだけ save() を呼ぶことで、約2秒ごとに定期セーブします。

if g.frame%120 == 0 { g.save() }

THE OFFLINE RULE

offline = min(留守秒数, 28800) × rate()ロード時g.sweets += offline

rate() はオーブン台数 × 5 [個/秒] です。留守中の生産量 = 留守秒数 × 5 × 台数 を起動時に一括加算します。

IN THE REAL GAME

ロード → オフライン計算 → 加算

load() は起動直後に1回呼ばれます。localStorage からデータを読み取り、保存時刻と現在時刻の差を計算。8時間を上限として留守中の生産分を sweets に足します。

func (g *game) load() {
    store := js.Global().Get("localStorage")
    raw := store.Call("getItem", prefix+"sweets")
    if raw.Type() != js.TypeString { return }
    g.sweets, _ = strconv.ParseFloat(raw.String(), 64)
    g.ovens, _ = strconv.Atoi(
        store.Call("getItem", prefix+"ovens").String())
    g.cost = 20 * math.Pow(1.25, float64(g.ovens))
    stamp, _ := strconv.ParseInt(
        store.Call("getItem", prefix+"time").String(), 10, 64)
    away := math.Min(8*3600,
        float64(time.Now().Unix()-stamp))
    if away > 0 {
        g.offline = away * g.rate()
        g.sweets += g.offline
    }
}

BONUS / DRAW ONLY

ゲームは同じ。絵だけを、別の作り方にする。

これは上の完成ゲームと同じパッケージから作ったWASMです。UpdateとLayoutには手を加えず、そのまま呼びます。1回のDrawが作った同じ瞬間を、通常、輪郭線、矩形モザイクへ同時に投影します。位置、得点、敵、当たり判定、時間は1つしかありません。

01

UPDATE

入力とゲームの事実を進める。3画面で共通。

02

LAYOUT

元ゲームの大きさをそのまま返す。

03

DRAW

同じsnapshotの見せ方だけを替える。

THE ONLY BOUNDARY

UpdateとLayoutは、元のゲームへそのまま渡す。

矩形版はsnapshotを小さな格子へ縮め、nearestで戻します。ワイヤ版は隣の画素との差をKageシェーダーで線にします。どちらもゲーム状態を読み書きしません。

func (g *gallery) Update() error {
    return g.inner.Update()
}

func (g *gallery) Layout(w, h int) (int, int) {
    return g.inner.Layout(w, h)
}

func (g *gallery) Draw(screen *ebiten.Image) {
    snapshot := ebiten.NewImage(width, height)
    g.inner.Draw(snapshot)
    drawPolished(screen, snapshot)
    drawWireframe(screen, snapshot)
    drawRectangles(screen, snapshot)
}
