DEEP DIVE / EXPONENTIAL

なぜ後半のラボは
こんなに高いの？

コストは 10 × 1.18n で計算します。ラボを買うたびに前のコストの1.18倍になるため、最初は手ごろでも後半は急激に高くなります。これが「指数関数的成長」です。

1

指数関数的コスト

math.Pow(a, n) は「a を n 回かける」計算です（2の3乗＝8）。float64(g.labs) は整数を実数に変えるだけで、Pow が実数専用だから書きます。n=0 で10、n=5 で約23、n=20 で約270 と後半ほど急に増えます。

10 * math.Pow(1.18, float64(g.labs))
2

二乗の生産量

生産速度は labs × labs × 8 です。台数が増えるほど1台あたりの貢献が大きくなる「スケーリング」が楽しさの核心です。1台で8/秒、3台で72/秒、10台で800/秒。

float64(g.labs*g.labs) * 8
3

K・M・B表記

1e6 は 1×10の6乗＝100万、1e9 は10億の短い書き方です。fmt.Sprintf("%.2fM", v) の %.2f は「小数点以下2桁の実数」。M は百万の印です。

fmt.Sprintf("%.2fM", v/1e6)

THE GROWTH FORMULA

cost(n) = 10 × 1.18ⁿ生産速度rate(n) = n² × 8 [/秒]

コストは指数的に増えますが、生産速度は二乗で増えます。ちょうどいいタイミングで買い続けると、後半でも快適に進めます。

IN THE REAL GAME

コスト計算 → 生産 → 数値表示

cost() と rate() は毎フレーム計算します。math.Pow は累乗、float64(...) は型合わせ、1e9 は10億、%.2f は小数2桁表示の指定です。short() がこれらをまとめて短い文字にします。

func (g *game) cost() float64 {
    return 10 * math.Pow(1.18, float64(g.labs))
}
func (g *game) rate() float64 {
    return float64(g.labs*g.labs) * 8
}

func short(v float64) string {
    switch {
    case v >= 1e9: return fmt.Sprintf("%.2fB", v/1e9)
    case v >= 1e6: return fmt.Sprintf("%.2fM", v/1e6)
    case v >= 1e3: return fmt.Sprintf("%.2fK", v/1e3)
    default:      return fmt.Sprintf("%.1f", v)
    }
}

WITHOUT EXPONENTIAL

コストが固定だとすぐ買い切れる

線形コストだと10台買えば終わりになりがちです。指数的上昇があることで、後半の「もう少し！」感が続きます。

WITH BIG NUMBER FORMAT

画面が数字で埋まらない

生の数値「1234567.89」を表示すると枠からはみ出します。short() があれば「1.23M」に収まり、すっきり読めます。

YOUR FIRST TUNING

指数の底を変えよう

1.18 を 1.10 にすると緩やかに、1.30 にすると急激に高くなります。rate の係数 8 も変えて全体のバランスを確かめましょう。
