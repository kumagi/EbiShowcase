台詞から複数章と結末収集へ

選択肢で分かれる会話ゲーム

一文字ずつ読める会話窓から始め、立ち絵演出、選択フラグ、章データを一つずつ加えて、4つの結末を集める物語へ進みます。

共通基礎の Update / Draw の上に、文章を再生する状態と演出を一段ずつ足します。

はじめて読む人へ

このコースの組み立て図

会話→立ち絵→選択肢→フラグ→結末の順に、海老・天次郎（えび・てんじろう）と一部品ずつ作ります。

海老・天次郎の見方
Updateが天次郎の数字を変え、Drawが次の一コマを描きます。

func (g *Game) Update() error { return nil }

func (g *Game)の(g *Game)は、この仕事がGameの値を読むための名札（レシーバー）です。

LEARNING PATH

台詞から複数章と結末収集へ

一文字ずつ読める会話窓から始め、立ち絵演出、選択フラグ、章データを一つずつ加えて、4つの結末を集める物語へ進みます。

01

一文字ずつ話す会話窓

文章を一度に出さず、表示済みの文字数を増やして、タップで全文表示と次の台詞を切り替えます。

Updateでshownを少しずつ増やす
→ 02

立ち絵の登場を演出する

フェード、横移動、拡大、跳ねる動きを0〜1の進み具合から作り、4つすべて試す小さな演出ゲームです。

座標・大きさ・透明度を補間する
→ 03

選択を物語のフラグにする

3つの質問で勇気と優しさの値を増やし、選んだ履歴から異なる結末を作ります。

選んだ瞬間に物語状態へ記録する
→ 04

3章の物語をデータで進める

港・市場・天文台の章ごとに場所、場面数、役割をまとめ、章をまたぐ一回の物語として再生します。

scene終了時にchapterを一つ進める
→ 05

Moon Lantern Stories

3章の月灯り事件を選択で進み、4つのエンディングを集める会話アドベンチャーです。

立ち絵の表情・まばたき・登退場を動かす
