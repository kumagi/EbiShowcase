一文字ずつ読める会話窓から始め、立ち絵演出、選択フラグ、章データを一つずつ加えて、4つの結末を集める物語へ進みます。

はじめて読む人へ

このコースの組み立て図

会話→立ち絵→選択肢→フラグ→結末の順に、海老・天次郎（えび・てんじろう）と一部品ずつ作ります。 このページの1本はコース全体で共通するgame状態の境界です。ルールはUpdateへ足し、Drawの順番や見た目は自由に差し替えられます。下の01から順に、その中へ各仕組みを実装していきます。

このページの1本
説明と次のコードを対応させてから、ゲームで変化を確かめよう。

func (g *Game) Update() error { return nil }
01 会話→02 立ち絵→03 選択肢→04 フラグ→05 結末

同じ STATE、4つの DRAW

描き方を替えても、ゲームのルールは替わらない。

Updateが作った同じ game を、ASCII、ワイヤーフレーム、ドット絵、美麗なスプライトの4通りで描けます。4分割画面へ同時に描いても、Update・入力・物理は一つのままです。DrawはUpdateの直後に必ず呼ばれる相棒ではなく、状態を読む差し替え可能な投影です。

╲ ○ ╱PIXEL ART▪▪●▪▪
▪▪▪▪▪SPRITES✦ EBI ✦

共通基礎の Updateが進める状態の上に、Drawは自由な投影として、文章を再生する状態と演出を一段ずつ足します。

LEARNING PATH

台詞から複数章と結末収集へ

一文字ずつ読める会話窓から始め、立ち絵演出、選択フラグ、章データを一つずつ加えて、4つの結末を集める物語へ進みます。

01

一文字ずつ話す会話窓

文章を一度に出さず、表示済みの文字数を増やして、タップで全文表示と次の台詞を切り替えます。

Updateでshownを少しずつ増やす
→ 02

立ち絵の登場を演出する

フェード、横移動、拡大、跳ねる動きを0〜1の進み具合から作り、4つすべて試す小さな演出ゲームです。

座標・大きさ・透明度を補間する
→ 03

選択を物語のフラグにする

3つの質問で勇気と優しさの値を増やし、選んだ履歴から異なる結末を作ります。

選んだ瞬間に物語状態へ記録する
→ 04

3章の物語をデータで進める

港・市場・天文台の章ごとに場所、場面数、役割をまとめ、章をまたぐ一回の物語として再生します。

scene終了時にchapterを一つ進める
→ 05

Moon Lantern Stories

3章の月灯り事件を選択で進み、4つのエンディングを集める会話アドベンチャーです。

立ち絵の表情・まばたき・登退場を動かす
