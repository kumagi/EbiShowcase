矢印/WASDまたは画面下の方向ボタンで探索します。黄色い太線がチャンクの境目。上には現在座標、キャッシュ数、今描いている数が出ます。

DEEP DIVE / CHUNK CACHE

大きな世界を、小さな地図カードに分ける。

世界の全マスを一度に持つ代わりに、8×8マスを一つのチャンクとして扱います。プレイヤーが新しい区画へ近づいたときだけ生成してmapへ保存し、画面外のチャンクは保存したまま描きません。

1

世界のマスをチャンクへ割る

世界タイルxを8で割った商がチャンクx、余りがチャンク内xです。負の座標では床方向に割る関数を使います。

chunkX = floorDiv(tileX, 8)
2

なければ生成して保存

カメラ中央の周囲5×5を調べ、mapにないキーだけ同じシードで生成します。戻ってきたときは同じデータを再利用します。

chunks[key] = generateChunk(key)
3

画面と重なる区画だけ描く

カメラの左上と右下をチャンク番号へ直し、その間だけDrawします。キャッシュ数が増えても描画数はほぼ一定です。

for cy := minCY; cy <= maxCY; cy++

IN THE REAL GAME

位置が変わるたび周囲を保証する

Updateは現在チャンクの周囲をキャッシュへ用意します。Drawはカメラと重なるキーだけ取り出し、生成はしません。

center := playerChunk()
for cy := center.y-2; cy <= center.y+2; cy++ {
    for cx := center.x-2; cx <= center.x+2; cx++ {
        key := chunkPos{cx, cy}
        if _, ok := chunks[key]; !ok {
            chunks[key] = generateChunk(key)
        }
    }
}

WHY CACHE?

変更した世界を覚える

拾ったビーコンはチャンク内で草へ変わります。再生成せずキャッシュを使うので、戻っても拾ったままです。

WHY CULL?

広さと描画量を分ける

100区画を訪れても、今見える数枚だけ描けば1フレームの仕事は急に増えません。
