DEEP DIVE / COMPOSED WORLD STATE

別々に作った仕組みを、同じ世界データでつなぐ。

地形、チャンク、在庫、ランタン、生き物、昼夜は別のルールです。でも全部が一つのgame状態を読み書きします。「採取したのでタイルが空き、在庫が増え、その材料で明かりを置き、生き物が逃げる」という因果がゲームになります。

1

世界を小さな状態へ分ける

tilesは地形、inventoryは所持数、lanternsはチャンク目標、creatureは生き物を担当します。役割は別でもUpdateで同じ順に進めます。

world → inventory → light → creatures
2

ルールを受け渡しでつなぐ

採取はtilesを0にして在庫を増やします。クラフトは在庫を減らしてtilesへランタンを書き、ランタンは近い生き物を消します。

tile → item → recipe → light → AI
3

必要な状態をまとめて保存

タイルだけ保存しても、所持品や位置が戻らなければ同じ世界ではありません。再開に必要な値をsnapshotへ一緒にコピーします。

snapshot{tiles, player, inventory, lanterns}

IN THE REAL GAME

セーブとロードは同じ形を往復する

セーブでは現在値をsnapshotへコピーし、ロードでは同じ項目をgameへ戻します。追加したシステムは保存項目にも追加します。

type snapshot struct {
    tiles [worldH][worldW]int
    px, py int
    wood, stone, crystal int
    lanterns [3]bool
    pickaxe bool
}
saved = snapshot{tiles, px, py,
    wood, stone, crystal, lanterns, pickaxe}
// load: 同じ項目をgameへ戻す

WHY COMPOSE?

一つの変更が次の遊びになる

採取だけ、AIだけではなく、互いの結果を使うことで目的のある世界になります。

WHY CHUNKS?

同じ目標を場所ごとに持てる

プレイヤーのxをchunkWで割り、どの地域に明かりを置いたか記録します。
