DEEP DIVE / RECIPE TRANSACTION

材料を確かめてから、まとめて消費し、完成品を足す。

同じ素材は 種類と個数の組として在庫へ束ねます。レシピは必要数の表です。全部そろうか先に確認し、成功する時だけ材料を減らして完成品を増やします。

1

採取をスタックへ足す

木を2個拾ったら、別々の物体を増やす代わりにwoodの個数へ2を足します。表示も保存も小さくできます。

inventory[Wood] += 2
2

レシピ全部を照合

木だけでなく、石、繊維、ロープの必要数をすべて比べます。1種類でも不足なら何も消費しません。

if haveWood && haveStone && haveRope
3

消費と生成を一組にする

条件を通った後で必要数を引き、ツルハシなどの完成品を追加します。途中だけ実行しません。

wood -= 2; stone -= 2; pickaxe = true

IN THE REAL GAME

不足ならreturnして在庫を守る

必要数を全部確認してから引くので、失敗したクラフトで一部の材料だけ失いません。

r := recipes[selected]
if wood < r.wood || stone < r.stone ||
   fiber < r.fiber || rope < r.rope {
    return // 何も変えない
}
wood -= r.wood; stone -= r.stone
fiber -= r.fiber; rope -= r.rope
addOutput(r.name)
