BOMB MAZE · STEP 03

壊れる壁とアイテム

爆風が木の壁に当たると、そのマスのデータを「壁」から「床」へ書き換えます。ときどき現れるアイテムを拾い、10枚の壁を壊そう。

むずかしさ ★★★☆☆PLAYABLE

PLAY FIRST

壁の中身を確かめよう

矢印キー / WASDで移動、Space / Xで爆弾。スマホやタブレットでは画面下のボタンをタップします。爆風に触れるか75秒たつと失敗です。

DEEP DIVE

マスは絵ではなく「状態」

壁が壊れるとは、壁の画像を消すことではありません。二次元配列の値を softWall から floor へ変えることです。次のフレームから移動判定も描画も、新しい床の値を読みます。

01

爆風が壁に届く

十字にマスを調べ、固い壁なら止まり、木の壁ならそのマスまで爆風を置きます。

02

壁を床へ変える

board[y][x] = floor と書けば、通れなかった場所が通れる場所になります。

03

中身を決める

座標から決めた値で、火力・爆弾数・速さのアイテム、または空を選びます。

TRY IT / CELL STATE

1マスの変化を順番に進めよう

木の壁 → 爆風 → 床＋アイテム。配列の値を書き換える順番です。先に床へ変えるから、出たアイテムのマスへ歩けます。

絵を消すのではなく、board[y][x] の値を変えています。

THE STATE WRITE

softWall + blastつぎにboard[y][x] = floor; maybe item

判定も描画も同じ配列を読むので、見た目と通行がずれません。

GO + EBITENGINE

データを変えてから出現させる

同じ座標は何度遊んでも同じ結果になるので、コードを追う練習がしやすくなります。ゲーム本体では座標ハッシュを使い、グローバルな乱数に頼らずドロップを決めています。

if board[p.y][p.x] == softWall {
    board[p.y][p.x] = floor
    destroyed++
    if drop := deterministicDrop(p); drop != noItem {
        items[p] = drop
    }
    break // 壁の向こうへ爆風を通さない
}

WHY IT WORKS

判定と見た目がずれない

Update の移動判定と Draw が同じ配列を読むため、消えたように見えるのに通れない、という食い違いを防げます。

CHALLENGE

落ちる割合を変えよう

deterministicDrop の余りを変え、アイテムが少ない難しい版を作ってみよう。どの種類が何マス出るか先に予想してから試そう。
