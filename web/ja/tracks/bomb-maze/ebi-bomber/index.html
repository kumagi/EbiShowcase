Ebi Bomber

海老・天次郎が壁を壊し、能力を集め、敵を倒して出口を開きます。個別に学んだタイマー、爆風、タイル変化、強化、危険回避を同じ更新順序で統合します。

★★★★★PLAYABLE

PLAY FIRST

敵を倒してEXITへ

矢印／WASDで移動、Spaceで爆弾。画面下はクリック／タッチ対応です。壁を8個壊し、敵を爆風で倒すと出口が開きます。

DEEP DIVE

ゲームは「部品」より「更新する順番」でつながる

入力、爆弾、爆風、壁とアイテム、敵、勝敗の順に処理すると、同じフレームの出来事が予測しやすくなります。当たり判定はスプライトではなくタイル座標で行います。

01

世界を先に変える

爆風で壁を床へ変え、アイテムを出してから取得を調べます。

02

能力をデータで反映

火力、同時爆弾数、移動量を数値として、同じ処理へ渡します。

03

最後に進行判定

敵撃破と壁8個の条件が揃ったら出口を開き、到着でクリアします。

TRY IT / FRAME ORDER

爆弾ゲームの1フレームを並べる

総合はパイプです。入力→移動→爆弾タイマー→爆風→壁/連鎖→敵AI→描画。順番を固定すると「爆風中に歩けるか」が説明できます。

描画は最後。AIは爆風更新のあとに置くことが多いです。

FIXED SYSTEMS ORDER

update bombs before AIつぎにdraw last

順序を変えると、同じ入力でも生死が変わることがあります。

GO + EBITENGINE

統合順序をUpdateに書く

各関数は小さく保ち、Updateがゲーム全体の設計図になります。

readPlayerInput()
updateBombTimers()
updateFlames()      // 壁・敵・自分へ作用
collectItems()      // 能力値を更新
moveEnemy()         // 危険マスを避ける
if enemyDown && broken >= 8 { exitOpen = true }
checkWinOrLoss()

統合で大切なこと

共有するのは同じタイル、爆風セル、能力値です。同じ情報を別々に計算しないようにします。

小さな関数に分ける理由

教材ごとの仕組みを保ったまま、順番だけを上位のUpdateで読めるからです。

チャレンジ

ステージデータを配列にし、出口到着後に次の壁配置を読み込んでみよう。
