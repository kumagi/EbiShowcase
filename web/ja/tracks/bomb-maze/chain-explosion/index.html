爆発イベントを次の爆弾へ渡す

爆弾の連鎖爆発

爆風に別の爆弾が入ったら、その爆弾のタイマーを待たずに起爆します。起爆する爆弾を待ち行列へ追加し、処理済みの印で同じ爆弾を二度爆発させません。

★★★★☆PLAYABLE

PLAY FIRST

3個以上を連鎖させ、リレーを5個消そう

矢印／WASDで移動、SpaceまたはXで爆弾を置きます。クリックとタッチは下のボタンを使います。爆弾を射程3以内の同じ通路へ並べ、最初の爆発前に遠くへ逃げましょう。

DEEP DIVE

「次に起爆する爆弾」の列を作る

爆弾Aの爆風がBへ届き、Bの爆風がCへ届くと、調べるものが次々増えます。そこで先に見つけたものから取り出すキューを使います。

01

起点をキューへ入れる

タイマーが0になった爆弾の番号を、これから処理する列へ追加します。

02

爆風内の爆弾を追加

先頭を起爆し、その爆風セルに待機中の爆弾があれば列の末尾へつなぎます。

03

処理済みなら飛ばす

複数の爆風から同じ爆弾が見つかっても、processedの印があれば二度目は何もしません。

TRY IT / CHAIN

爆風が隣の爆弾に燃え移る

「点火」で最初の爆弾が BLAST になり、隣の爆弾へ連鎖します。「消去」で炎を片付けます。キューや再帰で同じことができます。

同じフレームで連鎖を最後まで解決するか、次フレームに延ばすかを選べます。

GO + EBITENGINE

キューが空になるまで伝播する

ゲーム本体では爆弾の番号をキューに入れます。processedは1回の連鎖ごとに作るので、その連鎖で起爆済みかをすぐ調べられます。

queue := append([]int(nil), starts...)
processed := map[int]bool{}
for len(queue) > 0 {
    index := queue[0]
    queue = queue[1:]
    if processed[index] {
        continue
    }
    processed[index] = true
    bombs[index].blast = blastCells(bombs[index].at)
    for other := range bombs {
        if contains(bombs[index].blast, bombs[other].at) {
            queue = append(queue, other)
        }
    }
}

なぜその場で再帰しない？

キューなら「これから何個処理するか」が見え、途中で見つけたイベントも同じ順番で扱えます。長い連鎖でも処理の流れを追いやすくなります。

二重処理で何が困る？

AとCの両方がBへ届くと、Bの爆風や得点が二度発生するかもしれません。処理済みの印がゲームの結果を安定させます。
