BOMB MAZE · STEP 05

爆風から逃げる敵

青い偵察員は、すべての爆弾が「何フレーム後に、どのマスを焼くか」を予測します。間に合う安全マスまでの道を幅優先探索で選びます。

むずかしさ ★★★★☆PLAYABLE

PLAY FIRST

AIの逃げ道を観察しよう

矢印 / WASDで赤いプレイヤーを動かし、Space / Xで爆弾を置きます。画面下のボタンはマウスとタッチに対応。青いAIを6回の爆発から生き残らせよう。赤い数字は危険になるまでのフレーム、青い点は予定経路です。

DEEP DIVE

安全は「場所」と「時刻」の組み合わせ

今は安全なマスでも、到着した瞬間に爆発するなら逃げ先にはできません。そこで各爆弾の残り時間から危険表を作り、AIがそのマスへ着く時刻と比べます。

01

未来の爆風を重ねる

各爆弾から十字のマスを列挙します。同じマスに複数の爆弾が届くときは、いちばん早い時刻を記録します。

02

時刻つきで道を探す

幅優先探索の1歩に移動時間をかけます。到着時刻が爆発時間と重なる候補はキューへ入れません。

03

少し先まで安全を確かめる

着いた瞬間だけでなく、その場に50フレームいられるマスを出口として選びます。

TRY IT / TIME MAP

到着時刻と導火線を比べる

AIは「今どのマスが空いているか」だけでなく、「到着までに導火線が残るか」を見ます。ETA+余裕 < 導火線なら通れる経路です。

時間が進むと安全だった道が危険になります。だから毎フレーム張り直します。

ETA vs FUSE

eta = steps * framesPerStepつぎにsafe if eta + margin < fuse

空間の最短路に、時間の制約を足したのが回避AIです。

GO + EBITENGINE

到着時刻で候補をふるい分ける

danger はマスをキー、爆発までのフレームを値にした表です。探索は上下左右に加えて「待つ」候補も持ちます。

arrival := nextStep * moveEvery
blastAt, threatened := danger[next]
safe := !threatened ||
    arrival+8 < blastAt ||
    arrival > blastAt+blastLife
if safe {
    queue = append(queue, nextNode)
}

WHY IT WORKS

最短距離より間に合う道

ただ近いマスへ逃げるAIは、曲がり角で爆風を待ってしまいます。未来の時刻を含めると、遠回りでも爆発前に抜けられる道を選べます。

CHALLENGE

慎重さを変えよう

arrival+8 の8を大きくすると、AIはもっと早く危険地帯を避けます。大きすぎると道が見つからなくなる理由も考えてみよう。
