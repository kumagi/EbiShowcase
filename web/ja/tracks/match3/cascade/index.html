DEEP DIVE / RESOLUTION LOOP

「もう一度調べる」が連鎖を生む。

消去、落下、補充のあとでscanを呼びます。結果が空でなければ同じ処理へ戻り、空なら操作を返します。盤面が止まる条件がループの出口です。

1

一つの連鎖を解決

現在のmarkedを消し、列を詰め、空きを補充します。途中でプレイヤーの交換入力は読みません。

clear(); fall(); refill()
2

新しい盤面を再検査

落ちた結果は前と違う盤面です。そこでscanを再実行し、新しい3個並びをmarkedへ入れます。

marked = scan(board)
3

連鎖数を得点へ

1連鎖目は×1、2連鎖目は×2と増やします。偶然続いた大連鎖ほど大きなごほうびになります。

score += cells * 10 * combo

TRY IT / RESOLUTION QUEUE

残っている連鎖を処理しよう

連鎖候補を追加し、1個ずつ解決します。残りが0になった瞬間だけ、次の操作を受け付けられます。

未解決0

IN THE REAL GAME

見つからなくなるまで戻る

Updateのphaseとして分ければ、アニメーションをはさみながら同じ循環を進められます。

clearMatches()
fallAndRefill()
matches = scan(board)
if len(matches) > 0 {
    combo++
    phase = ClearPhase
} else {
    phase = PlayerPhase
}

WHY RECHECK?

落下後は別の問題

動く前になかった並びが生まれるため、古い検査結果は使えません。

WHY SETTLE?

入力を混ぜない

連鎖中に交換させないことで、盤面データと見た目がずれません。
