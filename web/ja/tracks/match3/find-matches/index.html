DEEP DIVE / LINE SCAN

一つずつ比べれば、何個続いたか数えられる。

行の左端から進み、種類が変わるまでの区間を覚えます。区間の長さが3以上なら、その座標をmarkedという集合へ入れます。次に同じことを列でも行います。

1

区間の始まりを覚える

startを同じ色の始まりに置きます。今の色が変わった場所xまでなら、長さはx-startです。

runLength := x - start
2

3以上を全部記録

長さが3以上ならstartからxの手前までをmarkedへ追加します。4個や5個の並びも同じルールです。

for i := start; i < x; i++ { mark(i,y) }
3

交差は1回だけ数える

横と縦の両方に入る中央マスも、map[point]boolなら同じキーが上書きされるので1マスとして残ります。

marked[point{x,y}] = true

IN THE REAL GAME

行を調べてから列を調べる

方向を混ぜず、まず全行、次に全列と二つの単純なループへ分けます。

for y := 0; y < rows; y++ {
    scanRow(y, marked)
}
for x := 0; x < cols; x++ {
    scanColumn(x, marked)
}

WHY TWO PASSES?

処理を読みやすくする

横と縦を別々に考えると、添字の間違いを見つけやすくなります。

WHY A SET?

交差を重ねない

T字や十字の中央を二度消したり、二度得点したりする事故を防げます。
