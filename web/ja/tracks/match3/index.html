キャンディクラッシュ型ゲームを作る

3マッチパズル

キャンディクラッシュのような、隣り合う2個を入れ替えて3つ並べるパズルを作ります。盤面の交換から、3個並びの探索、落下、補充、連鎖、特殊ピースへ進みます。

はじめて読む人へ

このコースの組み立て図

交換→検索→消去→落下→再検索の順に、海老・天次郎（えび・てんじろう）と一部品ずつ作ります。 このページの1本はコース全体で共通するgame状態の境界です。ルールはUpdateへ足し、Drawの順番や見た目は自由に差し替えられます。下の01から順に、その中へ各仕組みを実装していきます。

このページの1本
説明と次のコードを対応させてから、ゲームで変化を確かめよう。

func (g *Game) Update() error { return nil }
01 交換→02 検索→03 消去→04 落下→05 再検索

同じ STATE、4つの DRAW

描き方を替えても、ゲームのルールは替わらない。

Updateが作った同じ game を、ASCII、ワイヤーフレーム、ドット絵、美麗なスプライトの4通りで描けます。4分割画面へ同時に描いても、Update・入力・物理は一つのままです。DrawはUpdateの直後に必ず呼ばれる相棒ではなく、状態を読む差し替え可能な投影です。

LEARNING PATH

つないで完成へ。

各ステップを単独で遊べる大きさにし、アルゴリズムを目で確認してから次の仕組みへ進みます。

01

となり同士を交換

盤面をキャンディの行と列（2次元グリッド）で管理し、隣り合う2個だけをドラッグで交換します。

行と列のマス目（2次元グリッド）と隣接判定
→ 02

3個並びを探す

横方向と縦方向をそれぞれ順に見ていき（走査）、同じ種類が3個以上続く場所を集めます。

横・縦を順に見ていく処理（走査）とグループ化
→ 03

消して落とす

そろったピースを消し、列ごとに下へ詰め、上から新しいピースを補充します。

空きマスと重力詰め
→ 04

連鎖リアクション

落下後にもう一度盤面を検査し、動きが止まるまで連鎖を繰り返します。連続で消えるほど得点が上がる倍率（連鎖倍率）もつけます。

処理フェーズと、連続で消えるほど得点が上がる倍率（連鎖倍率）
→ 05

特殊ピース工房

4個、5個、L字の並びから、行を一気に消したり広い範囲を消したりする特別なピース（特殊ピース）を作ります。

並びの形の判定と、広い範囲を消す特殊ピース
→ 06

交換をアニメーションにする

2個のピースを動かし、揃う交換は消去へ、揃わない交換は元の場所へ戻します。

交換→判定→消去→落下を分ける
→ 07

3つの海底ステージをつなぐ

手数・目標点・初期盤面・背景をデータにして、難しさの違う3面とベスト記録を作ります。

同じルールへ違う目標データを渡す
→ 08
