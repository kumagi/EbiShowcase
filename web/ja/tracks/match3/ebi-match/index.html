DEEP DIVE / LEVEL DATA

ゲームのルールと、ステージの数字を分けよう。

「3個なら消える」はどのステージでも使うルールです。「12手で650点」はこのステージだけのデータです。別々にすると、プログラムを書き直さずに次の面を作れます。

1

ステージを表にする

名前、手数、目標点、最初の盤面を一つのstageに入れます。これはゲームへの注文票です。

stage{moves: 12, targetScore: 650}
2

1手を最後まで解決

正しい交換だけ手数を1減らし、消去→落下→補充→再検査を、並びがなくなるまで続けます。

moves--; resolveCascades()
3

勝ちと負けを比べる

得点が目標以上ならクリア。まだ届かず手数が0なら失敗です。クリア時の残り手数はボーナスになり、3つの海を通したベスト記録を何度でも更新できます。

won = score >= level.targetScore

TRY IT / STAGE DATA

手数と目標スコアを同時に見よう

ステージデータは「残り手数」と「目標」のセットです。マッチするたび手数を減らし得点を足す。0手で未達なら失敗です。

盤面レイアウトも同じステージJSONに置けます。

残り手数10
スコア0/30

ルールとステージ数値を分離します。

IN THE REAL GAME

データを読み、同じルールを動かす

newGameへ別のstageを渡すだけで、手数も目標も盤面も変わります。Updateの交換ルールはそのままです。

type stage struct {
    moves       int
    targetScore int
    board       [rows][cols]int
}

func newGame(level stage) *game {
    return &game{level: level,
        board: level.board,
        moves: level.moves}
}

WHY DATA?

面を増やしやすい

かんたんな面は手数を多く、むずかしい面は目標点を高くするだけ。ルールのバグを増やしません。

WHY INVALID SWAPS RETURN?

1手の意味を守る

3個並びを作らない交換は元へ戻します。「考えて並びを作る」という遊びがはっきりします。
