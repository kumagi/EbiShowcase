IN THE REAL GAME

データを読み、同じルールを動かす

newGameへ別のstageを渡すだけで、手数も目標も盤面も変わります。Updateの交換ルールはそのままです。

type stage struct {
    moves       int
    targetScore int
    board       [rows][cols]int
}

func newGame(level stage) *game {
    return &game{level: level,
        board: level.board,
        moves: level.moves}
}

WHY DATA?

面を増やしやすい

かんたんな面は手数を多く、むずかしい面は目標点を高くするだけ。ルールのバグを増やしません。

WHY INVALID SWAPS RETURN?

1手の意味を守る

3個並びを作らない交換は元へ戻します。「考えて並びを作る」という遊びがはっきりします。
