LOGIC THEN LERP

compact nowつぎにDraw: lerp(fromY,toY,progress)

圧縮とアニメを同じループに混ぜると、連鎖判定のタイミングが狂います。分けると安定します。

lerp のイメージ（数直線）: fromY=2、toY=5 のとき、progress が 0→0.25→0.5→0.75→1 と進むたび、描画Yは 2→2.75→3.5→4.25→5 と動きます。Updateは「いま何割進んだか」だけ覚え、Drawがその場所に絵を置きます。

IN THE REAL GAME

①列を圧縮してから ②progress で追う

いちばんバグりやすいのは列の圧縮です。下から write 位置を決め、空き以外を下へ詰めたあと、上を empty にします。デモは消去のあと自動で落下アニメに入り、その間は入力を無視します。

// 1) 論理：列を下へ詰める（compactColumns）
write := rows-1
for y := rows-1; y >= 0; y-- {
    if board[y][x] != empty {
        board[write][x] = board[y][x]
        write--
    }
}
for write >= 0 { board[write][x] = empty; write-- }

// 2) 見た目：falling 中は入力無視
if g.falling {
    g.progress += 0.07
    if g.progress >= 1 { g.falling = false }
    return
}
// Draw: py = fromY + (toY-fromY)*progress

WHY SEPARATE?
