どうして円は
だんだん速く落ちるの？

毎フレーム 同じ重力 を速度へ足すからです。Flappy と同じく、速度→位置の順。床では位置を戻し、速度を反転して小さくします。

1

円データ

位置・速度・半径を1つの ball にまとめます。

type ball struct
2

固定ステップ

Update ごと vy += 0.55。時間の進み方が一定です。

vy += .55
3

床の反発

めり込んだら押し戻し、vy を反転×減衰。小さいと settled。

vy = -vy * .45

TRY IT / FIXED STEP

毎フレーム同じ重力を足そう

「進める」で vy に重力を足し、位置を更新します。床で跳ねて小さくなり、やがて静止します。

Update が 1/60 秒ごとに呼ばれる前提の固定時間ステップです。

Y40位置
vy0.00速度
g+0.55毎フレーム

TRY IT · GO

vy += g
y += vy
if y > floor { y = floor; vy = -vy * 0.45 }

—

THE TWO-LINE RULE

vy = vy + gravity つぎに y = y + vy

時間刻みが一定だから、同じコードでも見た目の速さが安定します。可変フレームだけに頼ると物理がブレます。

IN THE REAL GAME

重力→移動→床

未静止の円だけ更新し、床で跳ねて十分遅ければ settled にします。

b.vy += .55
b.y += b.vy
if b.y+b.r > 660 {
    b.y = 660 - b.r
    b.vy = -b.vy * .45
    if math.Abs(b.vy) < .8 {
        b.settled = true
    }
}
