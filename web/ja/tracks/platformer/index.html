A

マリオ型ゲームを作る

横スクロールアクション

スーパーマリオのような横スクロールアクションを作ります。ジャンプから始めて、動く足場、敵、カメラ、パワーアップの順に少しずつ組み立てます。

共通基礎の続きです。LEVEL 01 の Update / Draw の上に、このジャンルの部品を1ステップずつ足します。主人公はオリジナルの 海老・天次郎（えび・てんじろう） です。

はじめて読む人へ

このコースの組み立て図

動かす→床で止める→跳ぶ→カメラの順に、海老・天次郎（えび・てんじろう）と一部品ずつ作ります。 このページの1本はコース全体で共通するUpdate→Drawの骨格です。下の01から順に、その中へ各仕組みを実装していきます。

このページの1本
説明と次のコードを対応させてから、ゲームで変化を確かめよう。

func (g *Game) Update() error { return nil }
01 動かす→02 床で止める→03 跳ぶ→04 カメラ

LEARNING PATH

小さな完成を
積み重ねる。

各ステップは単独で遊べるミニゲームです。前の仕組みを再利用しながら、最後に1本の作品へ統合します。

01

1画面プラットフォーマー

1画面の足場を飛び移り、床・壁・天井との衝突を解決します。

→ 02

動く足場の谷

上下左右にスライドする足場に乗ります。足場が動いた分だけプレイヤーも一緒に動かして、足場の上に立っていられるようにします。

足場の動きを一緒に伝える仕組み（相対移動）
→ 03

踏める敵の庭

巡回する敵を上から踏み、横から触れたときはダメージにします。

敵の状態と踏み判定
→ 04

スクロールする丘

画面より広いステージをカメラで追いかけ、見えている部分だけを描きます（画面の外を描かない工夫＝カリング）。

カメラ・見える範囲だけ描く工夫（カリング）・チェックポイント
→ 05

走る海老のアニメーション

止まる・走る・跳ぶ状態から表示フレームを選び、速度に合わせて絵を切り替えます。

状態と時間でフレームを選ぶ
→ 06

ステージをデータで増やす

同じ移動ルールへ、足場・背景色・ゴールの違う3つのステージデータを読み込みます。

ルールとステージデータを分ける

手応えのあるタイムアタック

粒子、画面揺れ、タイムと自己ベストを加え、8枚のコインを集め直したくなる短いゲームにします。

反応と記録がもう一度を作る
