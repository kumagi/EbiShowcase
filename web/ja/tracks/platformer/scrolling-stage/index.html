DEEP DIVE / CAMERA SCROLL

広いステージを、
どうやって表示するの？

ゲーム内部ではすべての座標を ワールド座標 で管理します。描画するときだけカメラの値（camera）を引くと、画面上の座標（スクリーン座標）が求まります。カメラはプレイヤーを少し遅れて追いかけることで、なめらかなスクロール感を生み出します。

1

ワールド座標

ゲームのロジックはすべてワールド座標で計算します。プレイヤーの x は 0〜2470 の範囲を動き、画面サイズには縛られません。

g.p.x = clamp(g.p.x+g.vx, 0, 2470)
2

カメラの追従

目標カメラ位置（プレイヤーより少し前）に対して、毎フレーム差の 8% だけ近づきます。一定速度でなく割合で動くので、自然な滑らかさが出ます。

camera += (target-camera)*.08
3

カリング

描画前にスクリーン X を求め、マイナスまたは画面幅を超えていたら continue でスキップします。見えないものを描かないと処理が速くなります。

if x+w < 0 || x > width { continue }

TRY IT / LAB

ワールド座標から画面座標へ

プレイヤーを動かし「カメラ追従」で目標へ寄せます。画面座標 = ワールド − カメラです。

本物のゲームの公式を、手でゆっくり回す模型です。

camera0
player200
screen X200

THE ONE-LINE RULE

screenX = worldX - camera それだけ // ロジックはワールド座標のまま

引き算1つでワールドとスクリーンを橋渡しします。カメラが動いても衝突判定はワールド座標で行うため、ロジックをいじる必要がありません。

IN THE REAL GAME

カメラ計算と
カリング付き描画

Update ではワールド座標で物理演算。カメラ更新もここで行います。Draw ではスクリーン X を計算し、範囲外なら continue でスキップします。

// Update: カメラをプレイヤーに追従させる
target := g.p.x - width*.4
g.camera = clamp(g.camera+(target-g.camera)*.08, 0, 2020)

// Draw: ワールド座標をスクリーンに変換して描画
for _, b := range g.grounds {
    x := b.x - g.camera // スクリーンX
    if x+b.w < 0 || x > width {
        continue // 画面外はスキップ
    }
    vector.DrawFilledRect(s, float32(x), float32(b.y), ...)
}

WITHOUT WORLD COORDS

ステージがスクロールできない

すべてをスクリーン座標で管理すると、カメラが動いたときに全オブジェクトの座標を書き換えなければなりません。ワールド座標が分離の鍵です。

WITH SMOOTH CAMERA

割合追従がゲームらしさを出す

差の 8% ずつ追いかけることで、ステージのつなぎ目や方向転換でもカメラがぎこちなくジャンプしません。

YOUR FIRST TUNING

カメラの追従速度を変えよう

.08 を大きくするとカメラがピタリついてきて視認しやすくなります。小さくするとゆっくり追いかけ、速く動いたときにスクリーンの端ギリギリになります。
