DEEP DIVE / GROUNDED JUMP

どうして、
空中でジャンプできないの？

ゲームの中の赤い四角のプレイヤーは grounded（地面にいる） という旗を持っています。地面に着いた瞬間だけ旗を立て、空中では旗を倒します。ジャンプキーは旗が立っているときしか効かないので、二段ジャンプができません。物理の重力はtickごとに vy に少しずつ足す、たった1行で再現します。これは LEVEL 04 の加速度の続きです。

1

重力

tickごとに vy に +0.65 を足します。何もしなければ加速しながら落ちていく感覚を生み出します。上限 14 で速度が頭打ちになります。

vy = min(vy+.65, 14)
2

接地判定

縦移動のあと、各ブロックと重なっているか調べます。落下中（vy > 0）に重なったら足をブロック上に置き直し、grounded = true にします。

grounded = true
3

衝突の分離

横移動 → 横衝突解決 → 縦移動 → 縦衝突解決、の順番が重要です。同時に処理すると斜めの角で引っかかるバグが起きます。

p.x += vx // 横を先に

TRY IT / GROUNDED JUMP

接地のときだけ跳ばせてみよう

「ジャンプ」は grounded のときだけ効きます。空中でもう一度押しても飛びません。「1 tick進める」で重力がかかり、床に着くとまた接地します。

本物のゲームではtickごとに自動で進みます。ここはスローモーションです。

THE ONE-LINE RULE

if jump && grounded { vy = -12 } tickごとに grounded = false → 衝突チェックで再び true

tickごとに grounded を false にリセットし、その後の衝突チェックで再び true を立てます。「地面にいる」はtickごとに証明しなければなりません。この仕組みが「地面にいる時だけジャンプ」を実現します。

IN THE REAL GAME

縦の衝突解決が
接地を決める

落下中（vy >= 0）でブロックに重なったとき、頂面へスナップして vy = 0、grounded = true。上昇中（vy < 0）に天井へ当たったときは底面へスナップして vy = 0。

g.p.y += g.vy
g.grounded = false
for _, b := range g.blocks {
    if overlap(g.p, b) {
        if g.vy > 0 {
            g.p.y = b.y - g.p.h // 足をブロック上に
            g.grounded = true
        } else {
            g.p.y = b.y + b.h   // 頭をブロック下に
        }
        g.vy = 0
    }
}

WITHOUT GROUNDED FLAG

空中で何度でも跳べてしまう

旗がないと「ジャンプキーを押したら常に上昇」になります。ゲームバランスが崩れ、ジャンプ系のパズルが成立しません。

WITH SEPARATE AXES

斜め衝突もスムーズに解ける

横・縦を分けることで、プレイヤーが足場の角に引っかかる「コーナー問題」をシンプルに回避できます。
