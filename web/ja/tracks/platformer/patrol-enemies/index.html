踏める敵の庭

足場を巡回する丸い敵を踏んで倒し、一番上の旗を目指します。上から踏むと倒せる、それ以外は体力を失う、というマリオ式判定のしくみを学びます。

DEEP DIVE / PATROL & STOMP

どうして上から踏むと
倒せるの？

接触判定は「重なっているか」だけでなく 「どの方向から当たったか」 も見ます。プレイヤーが落下中（vy > 0）で、かつ接触前に敵の頭より上にいた場合に限り「踏んだ」と判定します。それ以外の接触はダメージです。敵は足場の端に来たら向きをひっくり返し、サポートがなければ折り返します。

1

巡回（パトロール）

敵は毎フレーム vx だけ動きます。足場の端（左端・右端）に達したら vx = -vx で向きを反転。足場の外に出ると支えがなくなるので、その前に折り返します。

e.vx = -e.vx // 折り返し
2

踏み判定

重なりを検知したとき、落下中（vy > 1）かつ前フレームの足が敵の頭付近より上なら「踏んだ」。alive = false にして敵を消します。

if vy > 1 && oldBottom <= e.y+8
3

ダメージ処理

踏み以外の接触はライフを1減らし、プレイヤーを初期位置へ戻します。ライフが0になるとゲームオーバー。

g.life--; g.player.x = 30

落下中だけ踏みつけ

vy がプラス（落下）のときだけ踏みつけ成功。上昇中はダメージです。

本物のゲームの公式を、手でゆっくり回す模型です。

THE ONE-LINE RULE

if vy > 1 && oldBottom <= e.y+8 { 踏んだ！ } それ以外 else { ダメージ }

「上から」という条件を oldBottom（前フレームの足の位置）と敵の頭（e.y+8）を比較して判断します。重なり判定より少し余裕を持たせることで、ギリギリの踏みも気持ちよく成立します。

IN THE REAL GAME

接触の種類を
ひとつの if-else で分ける

重なりが確認できてから、落下速度と前フレームの足位置で踏んだか否かを判断します。踏んだら跳ね返り（vy = -8）で連続踏みも可能です。

if overlap(g.player, e.rect) {
    if g.vy > 1 && oldBottom <= e.y+8 {
        // 踏んだ！
        e.alive = false
        g.player.y = e.y - g.player.h
        g.vy = -8 // 踏んだら少し跳ね返る
    } else {
        // 横または下から当たった
        g.life--
        g.player.x = 30
        g.player.y = 590
        g.vx, g.vy = 0, 0
        if g.life <= 0 {
            g.over = true
        }
    }
}

WITHOUT DIRECTION CHECK

踏んでもダメージになってしまう

単純な重なり判定だけでは、上から踏んでもダメージを受けます。方向チェックで接触の「種類」を分けることが必須です。

WITH EDGE PATROL

敵が自動で足場を往復する

足場の端を調べて折り返す処理を入れると、マップに合わせた巡回パターンが自動的にできます。配置を変えるだけで難易度が変わります。

YOUR FIRST TUNING

敵の速さとライフを変えよう

各敵の vx を大きくするとすばやくなり、g.life を増やすと余裕が生まれます。敵を増やしたり、足場に乗せる位置を変えると難しさが大きく変わります。
