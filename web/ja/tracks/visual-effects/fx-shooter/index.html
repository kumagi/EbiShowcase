DEEP DIVE / EVENT QUEUE + BUDGET

同時多発する演出を、
どう順序と上限つきで処理する？

ゲーム側は「どのエフェクト関数を呼ぶか」を知りません。Kind・位置・Strengthだけを発行します。dispatcherが見せ方を選ぶため、低品質モードではBudgetを下げるだけで弾数や得点を変えずに負荷を落とせます。

1

型付きイベント

文字列でなくEventKindを積む。

events.Push(Event{Kind: ...})
2

一度だけDrain

発行順を保って空にする。

for _, e := range Drain()
3

任意処理に上限

粒子個数だけBudget.Takeで削る。

n := budget.Take(wanted)

IN THE DEMO

ルールはEventを発行するだけ

発射処理からBurstを外し、演出の対応と負荷制御をdispatcherへ集約します。

g.events.Push(vfxmotion.Event{
    Kind: vfxmotion.EventEnemyDestroyed, X: e.x, Y: e.y, Strength: fx,
})

for _, event := range g.events.Drain() {
    n := g.budget.Take(wantedParticles(event))
    g.dispatch(event, n)
}

TEST THE CONTRACT

Queueの順序・一回性・Budgetを試す

二イベントをPushしてDrain後に空になること、Limit 5へ4+4を要求すると4+1になることを確認します。

FAILURE MODE

イベントそのものを負荷対策で捨てない

得点や音まで同じBudgetで省くと、端末性能でゲーム結果が変わります。削るのは任意の粒だけです。
