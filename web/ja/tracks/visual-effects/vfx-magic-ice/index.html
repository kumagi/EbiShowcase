phase で二幕を切り替える

forming のあと shatter へ。同じ Particle 型でも役割が変わります。

// Act 1: pull frost inward
f.x += (castX - f.x) * 0.06
// Act 2: shatter shards outward
spawn(IcePNG, vx: cos(a)*sp, grav: 0.12)
spawn(Ring) // bloom on break

WHY TWO ACTS?

溜めが印象を作る

いきなり砕くより、一度見せてから壊す方が「魔法っぽい」です。

WHY ICE PNG?

形が属性を語る

丸い火花では氷に見えません。尖ったテクスチャが属性の半分です。

phaseを形成と粉砕に分け、前半は枝を伸ばし、後半は同じ枝を破片へ分けます。二段階だから育って砕けたように読めます。 Go用語メモ：整数型（int）は小数を持たない数、浮動小数点型（float32/float64）は小数を持てる数、typeはデータの箱の種類を決める言葉、funcは処理をひとまとめにする関数の印です。[]TはTを順番に並べるリスト、appendはその末尾へ一つ足す命令です。Updateは数字と状態を進め、Drawはその結果を絵にします。

海老・天次郎の見方
Updateが天次郎の数字を変え、Drawが次の一コマを描きます。

if phase == Form { grow() } else { shatter() }

func (g *Game)の(g *Game)は、この仕事がGameの値を読むための名札（レシーバー）です。
