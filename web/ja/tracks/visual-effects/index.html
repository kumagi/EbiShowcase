VFX

花火・光・歩き——見た目の道具箱

ビジュアルエフェクト工房

ここは「ゲームのルール」ではなく「見た目のかっこよさ」を練習する工房です。スタンプ・回転・色・光・歩きなど、絵を動かす道具を1つずつ触ります。後半では、すでに遊んだ LEVEL のミニゲームにキラキラを足します。全部やらなくても大丈夫——気になる見出しからでOKです。

共通基礎(LEVEL 01〜12)の続きです。前半は描画道具、A01〜A12はルールの事実→独立した演出状態→Draw/FXという実装設計を12種類のゲームで身につけます。この設計が次の25ジャンル共通の前提になります。

はじめて読む人へ

このコースの組み立て図

絵を置く→変形→色→重ね方→アニメ→粒の順に、海老・天次郎（えび・てんじろう）と一部品ずつ作ります。 このページの1本はコース全体で共通するgame状態の境界です。ルールはUpdateへ足し、Drawの順番や見た目は自由に差し替えられます。下の01から順に、その中へ各仕組みを実装していきます。 Go用語メモ：整数型（int）は小数を持たない数、浮動小数点型（float32/float64）は小数を持てる数、typeはデータの箱の種類を決める言葉、funcは処理をひとまとめにする関数の印です。[]TはTを順番に並べるリスト、appendはその末尾へ一つ足す命令です。Updateは数字と状態を進め、Drawはその結果を絵にします。

このページの1本
説明と次のコードを対応させてから、ゲームで変化を確かめよう。

func (g *Game) Update() error { return nil }
01 絵を置く→02 変形→03 色→04 重ね方→05 アニメ→06 粒

LEARNING PATH

基本編と応用編。

基本編：画面上のスライダーで「置く・回す・染める・光らせる」を体感。応用編：ルール（得点や当たり）と演出（花火）を別の箱に入れる練習。まずはスタンプからで十分です。

VFX CHEAT SHEET

数字を変えると、見た目が変わる。

speed・size・alpha・lifeを変えると、同じルールでも手ざわりが変わります。

値意味小さく大きく
speed粒の速さ漂う飛ぶ
size大きさ細かいフレア
alpha透明度薄い濃い
life寿命（フレーム）一瞬余韻
