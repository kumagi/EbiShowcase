1体をデータで作る

種族の名前・最大HP・攻撃力は共有し、HPと経験値は1体ずつ別々に変化させます。

このステップで学ぶこと種族の定義データと個体の可変状態

PLAYABLE / WEBASSEMBLY

同じ種族の双子を、8手以内でXP 9まで育てよう。

カードをタップするか1・2で個体を選び、TRAIN/RESTをタップ。PCではT/SpaceとR/H。HPが低いBUBBLEは休ませないと倒れてしまいます。

● LIVE / GO + WASMSTEP 01

DEEP DIVE / DEFINITION VS STATE

図鑑に1回書くことと、1体ずつ覚えることを分ける。

LEVEL 01のUpdate / Drawで数字を変えて描いたように、今回もUpdateが個体のHPと経験値を変えます。ただし、REEFLETという種族の名前や最大HPは何体いても同じです。共有する定義と変化する状態を分けます。

1

種族は読み取り専用の表

種族表には名前、最大HP、攻撃力、色を一度だけ置きます。トレーニングしても、この表の最大HPは10のままです。

speciesTable[0] = Species{"REEFLET", 10, 3}
2

個体は種族IDを指す

AQUAもBUBBLEもspeciesID: 0を持ち、同じ定義を読みます。名前を2回コピーしないので、修正する場所も1か所です。

Creature{nickname: "AQUA", speciesID: 0}
3

HPとXPは個体へ保存

現在HPや経験値は冒険で変わるので、個体構造体に置きます。同じ種族でもAはHP10、BはHP6から始められます。

creature.hp -= 2; creature.xp += 3

TRY IT / DEF vs INST

種族定義から個体を増やそう

種族データは変えません（maxHPなど）。個体は定義を参照しつつ、今の HP だけ別持ち。「スポーン」で個体増、「ヒット」で個体だけ減ります。

図鑑は定義の一覧、パーティは個体の一覧です。

DEF ≠ STATE

Species.MaxHPつぎにInstance.HP

定義を直接減らすと全個体が壊れます。

IN THE REAL GAME

回復の上限だけ種族定義から読む

変えるのは選択個体のhpだけです。上限はその個体のspeciesIDから共有表を引きます。

c := &creatures[selected]
definition := speciesTable[c.speciesID]

// 個体の状態だけを書き換える
c.hp = min(definition.maxHP, c.hp+3)
c.xp += 3

WHY SHARE?

同じ情報をコピーしない

100体の名前や基礎能力を直す代わりに、種族表の1行を直せます。

WHY INSTANCES?

冒険の違いを残せる

同じ種族でも、受けたダメージ、経験値、ニックネームを別々に保存できます。
