DEEP DIVE / INTEGRATED GAME STATE

6個の仕組みを、1本の「冒険の流れ」にする。

完成ゲームは機能を横に並べただけではありません。地域選択が遭遇データを選び、戦闘結果が捕獲確率を変え、捕獲成功が図鑑・経験値・パーティ・箱をまとめて更新します。各画面をmodeMapとmodeBattleで分け、結果を次の状態へ渡す状態遷移が背骨です。

1

地域から個体を作る

地域ごとの遭遇表は種族IDの並びです。訪問回数で次のIDを選び、種族定義から野生個体のHPとタイプを作ります。

wildSpecies = encounterTables[region][visits[region]%2]
2

戦闘結果を一つの取引にする

技の威力・命中・回数と相性表でHPを下げます。捕獲成功時だけ、図鑑登録、個体追加、全員への経験値、進化判定、マップ復帰をまとめて行います。

dex[id]=true; addMonster(); shareEXP(); mode=modeMap
3

必要な状態だけ保存する

画面の色やボタンは保存しません。パーティの種族とEXP、箱、図鑑、地域訪問数をsaveDataへ写し、JSONバイト列にします。

saveSlot, _ = json.Marshal(payload)

IN THE REAL GAME

セーブ用構造体を実行中データから作る

実行中のmonsterにはHPもありますが、この教材のセーブは種族・EXP・図鑑・地域進行を選びます。保存したい約束を専用構造体にすると、JSONの形が安定します。

payload := saveData{Dex: g.dex, Visits: g.visits}
for _, m := range g.party {
    payload.PartySpecies = append(payload.PartySpecies, m.speciesID)
    payload.PartyExp = append(payload.PartyExp, m.exp)
}
for _, m := range g.box {
    payload.BoxSpecies = append(payload.BoxSpecies, m.speciesID)
}
encoded, err := json.Marshal(payload)
if err == nil {
    saveSlot = append(saveSlot[:0], encoded...)
}

WHY MODES?

入力の意味を混ぜない

1キーはマップでは地域、戦闘では技です。modeを先に見れば、同じ入力を安全に別処理へ渡せます。

WHY ONE CAPTURE TRANSACTION?

データの食い違いを防ぐ

図鑑だけ登録して個体を入れ忘れる、といった半端な成功を防げます。

SHIP IT

保存先を交換する

JSONを作る部分はそのままに、メモリのsaveSlotをブラウザ保存やサーバーへ渡すアダプターを考えよう。
