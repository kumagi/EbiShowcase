捕獲チャレンジ

残りHP、SLEEP、道具から捕獲率を計算し、成功した個体を空いている所持枠へ保存します。

PLAYABLE / WEBASSEMBLY

HPと道具を守りながら3体を捕獲しよう。

GENTLE HITでHPを下げ、SLEEPを付け、捕獲率を見てBASIC/REEF ORBを選びます。クリック/タップ、または1〜4。失敗は反撃と道具消費です。

● LIVE / GO + WASMSTEP 05

DEEP DIVE / VISIBLE PROBABILITY

運だけにせず、準備したぶん確率を上げる。

捕獲率はいつも画面に見えます。基本15%へ、減ったHPから最大55%、SLEEPなら20%、REEF ORBなら15%を足します。最後に0〜99のrollが確率より小さいかを比べます。

1

今の条件から確率を作る

HP100ならボーナス0。HP20なら44。SLEEPや道具も足し、強すぎないよう最大95%で止めます。

chance = min(95, 15+hpBonus+status+item)
2

乱数を一度だけ引く

rollがchance未満なら成功です。画面へ両方を出すと、「79 < 84だから成功」のように結果を説明できます。

captured := rng.Intn(100) < chance
3

成功と失敗でデータを分ける

成功はstorageへ名前を追加。失敗は道具だけ消え、SLEEPが解けて反撃を受けます。どちらも同じ状態にはしません。

storage = append(storage, wild.name)

TRY IT / CATCH RATE

捕獲率を上げてロールしよう

捕獲は確率です。エサで率を上げ、「ロール」で 0–99 を振って率未満なら成功。失敗リスクも同じ乱数の反対側です。

所持枠チェックはロールの前に行います。

IN THE REAL GAME

確率計算と保存を別の関数にする

captureChanceは数字を返すだけ。tryCaptureが乱数を引き、成功したときだけstorageを更新します。

func captureChance() int {
    hpBonus := (maxHP-wildHP) * 55 / maxHP
    statusBonus := 0
    if sleeping { statusBonus = 20 }
    return min(95, 15+hpBonus+statusBonus+itemBonus)
}

if rng.Intn(100) < captureChance() {
    storage = append(storage, wildName)
}
