集めて育てるバトルを作る

モンスター収集RPG

ポケモンのような、モンスターを捕まえて育てて戦わせるゲームを作ります。1体のモンスターデータから、技、タイプの相性、パーティ、捕獲、成長、図鑑へ発展します。

はじめて読む人へ

このコースの組み立て図

種族→相性→戦闘→捕獲→成長の順に、海老・天次郎（えび・てんじろう）と一部品ずつ作ります。 このページの1本はコース全体で共通するgame状態の境界です。ルールはUpdateへ足し、Drawの順番や見た目は自由に差し替えられます。下の01から順に、その中へ各仕組みを実装していきます。

このページの1本
説明と次のコードを対応させてから、ゲームで変化を確かめよう。

func (g *Game) Update() error { return nil }
01 種族→02 相性→03 戦闘→04 捕獲→05 成長

共通基礎の続きです。LEVEL 01 の Update / Draw の上に、このジャンルの部品を1ステップずつ足します。主人公が登場するステップではオリジナルの 海老・天次郎（えび・てんじろう） を使います。

LEARNING PATH

ひとつの仕組みから、
遊べる世界へ。

データの変化とアルゴリズムを小さなゲームで確かめ、前のステップを再利用しながら完成形へ進みます。

01

1体をデータで作る

名前や基礎能力といった「種族としての設計図」と、いま目の前にいる1匹のHPや経験値を分けて管理します。

種族の設計図と、この1匹の状態（種族データと個体状態）
→ 02

技を選ぶバトル

威力、命中率、回数を持つ技を選び、順番に効果を解決します。

技データ・命中・ダメージ
→ 03

タイプ相性研究所

攻撃側と防御側のタイプから、効果が上がるか下がるかの倍率を表（相性表）で引きます。

タイプの得意・不得意の一覧（相性表）と倍率
→ 04

パーティと交代

複数の仲間を持ち、倒れたときや作戦に応じて交代します。

チーム編成と戦闘中の入れ替え
→ 05

捕獲チャレンジ

残りHPが少ないほど、また道具によって捕まえやすくなる確率を計算し、新しい仲間として保存します。

捕まえやすさの確率と所持枠
→ 06

経験値と進化

経験値からレベルを求め、条件を満たした個体の種族データを切り替えます。

成長曲線・レベル・進化条件
→ 07

地域ごとの出会いを設計する

3つの地域に別々の出現表と研究スタンプを持たせ、行き先を選ぶ旅にします。

場所・出現表・進み具合をひと組にする
→ 08

捕獲の一瞬を演出する

投げる、飛ぶ、3回揺れる、成功して弾ける、という中間フレームで捕獲を待ちたくなる時間にします。

確率の答えを短い演出シーケンスで見せる
→ 09

Ebi Monsters

地域ごとの出現、捕獲記録、パーティと預かり箱を統合します。

遭遇表・図鑑・セーブ
