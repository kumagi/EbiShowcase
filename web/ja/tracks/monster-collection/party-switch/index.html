DEEP DIVE / ACTIVE INDEX

パーティ全員を消さず、「今戦う番号」だけを変える。

3体はずっとsliceの中に残り、それぞれのHPを持ち続けます。前衛は別のコピーではなくactiveという番号です。交代はこの番号を控えの番号へ変える状態遷移で、自分から交代する時と倒れて選び直す時ではターンの扱いが違います。

1

前衛を番号で指す

party[active]だけが攻撃し、反撃を受けます。控えのHPはそのまま残り、あとで前衛になった時に続きから使えます。

front := &party[active]
2

予定交代は1手を使う

元気な控えを選ぶとactiveを変え、turnを1増やして敵の反撃を受けます。強い相性を得る代わりに時間を払います。

active = reserve; turn++; enemyAttack()
3

HP0なら強制交代状態

前衛が倒れたら攻撃を止め、mustSwitchをtrueにします。生きている控えを選ぶまで次の通常ターンへ進めません。

if front.hp == 0 { mustSwitch = true }

控えが0なら交代ボタンを押せなくします。

IN THE REAL GAME

同じswitchToで2種類の交代を扱う

強制交代なら敵の追加攻撃なしで状態を解除し、予定交代ならターンを消費します。入口を共通にすると、HP0の個体を選べない確認も一か所です。

func switchTo(next int) {
    if party[next].hp == 0 { return }
    active = next
    if mustSwitch {
        mustSwitch = false // 強制交代
        return
    }
    turn++                // 予定交代
    enemyAttack()
}

WHY KEEP RESERVES?

受けた傷を忘れない

控えも同じparty配列に残すので、再び出した時に正しいHPを使えます。

WHY SPEND A TURN?

交代が作戦になる

いつでも無料だと毎回相性だけを選べます。反撃の危険があるから今替えるかを考えます。
