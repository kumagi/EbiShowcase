DEEP DIVE / MOVE DATA AND ORDER

「技を選ぶ」と「効果を起こす」の間に、順番を作る。

技は名前だけではなく、威力power、命中accuracy、残り回数uses、先に動きやすいpriorityを持つデータです。両者の選択をactionキューへ入れ、並べてから一つずつ取り出すと、表示とHP変化が混ざりません。

1

技をデータで比べる

REEF BURSTは威力42でも命中70%、JET PINCHは威力18で必ず命中。回数も違うため、いつ使うかが選択になります。

moveData{power:42, accuracy:70, maxUses:3}
2

2行動を順番へ並べる

priorityが大きい技を先へ。同じなら素早さ12と10を比べます。SHELL GUARDはpriority 3なので攻撃前に守れます。

sort.SliceStable(actions, byPriorityThenSpeed)
3

命中してから効果を出す

0〜99の乱数がaccuracy未満なら命中。攻撃はHPを減らし、ガードは次のダメージを半分、回復はHPを戻します。

hit := rng.Intn(100) < move.accuracy

IN THE REAL GAME

先頭の行動だけを解決する

選択時に2行動を保存し、resolveNextは先頭だけを外して命中と効果を計算します。敵が倒れれば残り行動は実行しません。

queue = []action{playerAction, rivalAction}
sortByPriorityThenSpeed(queue)

func resolveNext() {
    current := queue[0]
    queue = queue[1:]
    if rng.Intn(100) < current.move.accuracy {
        apply(current.move)
    }
}

WHY USES?

強い技を考えて使う

何度でも使えると一番強い技だけでよくなります。残り回数が、今使うか残すかの作戦を生みます。

WHY A QUEUE?

同時に見せない

行動を保存して順番に解決すると、メッセージ、アニメーション、HP変化を一つずつ見せられます。
