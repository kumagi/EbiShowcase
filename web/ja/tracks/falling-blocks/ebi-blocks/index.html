DEEP DIVE / SYSTEM PIPELINE

たくさんの仕組みを、「今すること」の順番でつなぐ。

完成ゲームは大きな一枚のルールではありません。入力、移動、衝突、固定、ライン検査、得点、次の形という小さな仕事の列です。一つのpieceとboardを順番に受け渡すと、機能が増えても迷子になりません。

1

落下中の形を操作

activeには種類、向き、位置があります。移動や回転の候補がboardと重ならないときだけ値を更新します。

if !collides(next) { active = next }
2

動けなければ盤面へ固定

下へ進めなくなった4マスをboardへ書き、完成行を消し、得点とレベルを更新します。この順番が大切です。

lock(); clearLines(); addScore()
3

次の形を準備

7バッグからqueueを補充して次を取り出します。HOLDは落下中1回だけ、ゴーストは衝突直前まで位置を試します。

spawn(takeNext())

IN THE REAL GAME

固定を境目に一連の処理を完了する

activeの4マスを盤面へ移した後は、行・得点・次の形までまとめて処理し、それから入力へ戻ります。

func (g *game) lock() {
    writeActiveToBoard()
    cleared := g.clearLines()
    g.addScore(cleared)
    if g.score >= targetScore {
        g.won = true
        return
    }
    g.spawn(g.takeNext())
}

WHY ONE BOARD?

7種類を混ぜて使い切ってから次の袋へ進むため、同じ形だけが長く来ない遊びになります。

BONUS / DRAW ONLY

ゲームは同じ。絵だけを、別の作り方にする。

これは上の完成ゲームと同じパッケージから作ったWASMです。UpdateとLayoutには手を加えず、そのまま呼びます。元ゲームを等倍のまま操作でき、右には同じ瞬間の輪郭線と、文字だけでゲームを描くASCII画面のモックを並べます。ゲームのルールと見た目は別々に設計できます。

01

UPDATE

入力とゲームの事実を進める。3画面で共通。

02

LAYOUT

元ゲームの大きさをそのまま返す。

03

DRAW

画像・輪郭線・ASCIIなど、描き方を選ぶ。

THE ONLY BOUNDARY

UpdateとLayoutは、元のゲームへそのまま渡す。

等倍snapshotが入力と一致するゲーム画面です。輪郭版はKageで隣の画素との差だけを線にします。ASCIIモックは、同じようなゲーム場面を文字だけでも伝えられることを示します。描画はルールを変更しません。

func (g *gallery) Update() error {
    return g.inner.Update()
}

func (g *gallery) Layout(w, h int) (int, int) {
    return g.inner.Layout(w, h)
}

func (g *gallery) Draw(screen *ebiten.Image) {
    snapshot := ebiten.NewImage(width, height)
    g.inner.Draw(snapshot)
    screen.DrawImage(snapshot, nil) // native-size playable input surface
    drawEdgeInset(screen, snapshot)
    drawASCIIGameMock(screen)
}
