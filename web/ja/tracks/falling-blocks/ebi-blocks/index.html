IN THE REAL GAME

固定を境目に一連の処理を完了する

activeの4マスを盤面へ移した後は、行・得点・次の形までまとめて処理し、それから入力へ戻ります。

func (g *game) lock() {
    writeActiveToBoard()
    cleared := g.clearLines()
    g.addScore(cleared)
    if g.score >= targetScore {
        g.won = true
        return
    }
    g.spawn(g.takeNext())
}

WHY ONE BOARD?

当たり判定の答えを一つにする

固定済みマスはboardだけに置き、activeとghostは必要なとき計算します。三つの記録がずれません。

WHY A BAG?

全種類が順番に巡る

7種類を混ぜて使い切ってから次の袋へ進むため、同じ形だけが長く来ない遊びになります。
