DEEP DIVE / FAIR QUEUE AND PREDICTION

見えない運を、見えるデータに変える。

完全なくじ引きでは、欲しい形が何十回も来ないことがあります。7種バッグなら7個を一度ずつ混ぜて配るので大きな偏りがありません。さらにHOLDとゴーストを加えると、プレイヤーは運だけでなく予定を立てることができます。

1

7個を入れて混ぜる

I・O・T・S・Z・J・Lを一つずつsliceへ入れ、順番だけをシャッフルします。空になったときだけ新しい袋を作ります。

bag = []int{I, O, T, S, Z, J, L}
2

現在とHOLDを交換

棚が空なら現在の形を預けて次を取り、入っていれば二つを交換します。同じ形で何度も交換できないよう、固定するまで一回だけです。

held, current = current, held
3

コピーを下へ試す

現在位置は変えず、仮のyだけを1ずつ増やします。次が衝突する直前のyが、点線で描くゴースト位置です。

for canPlace(x, ghostY+1) { ghostY++ }

IN THE REAL GAME

ゴーストは盤面を変更しない

本物の形と同じcanPlaceを使いますが、調べるのはローカル変数だけ。予告を描いてもゲーム状態は進みません。

func (g *Game) ghostY() int {
    y := g.y
    for g.canPlace(g.kind, g.rotation, g.x, y+1) {
        y++
    }
    return y
}

WHY A BAG?

ランダムでも約束がある

どの7個にも全種類が一度ずつ入り、長すぎる待ち時間を防げます。

WHY ONE HOLD?

選択に値段をつける

固定まで一回という制限があるから、「今使うか」がゲームの考えどころになります。
