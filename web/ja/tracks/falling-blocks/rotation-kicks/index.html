DEEP DIVE / ROTATE THEN TEST

回してから、「ここに置ける？」を調べる。

ブロックの中心を原点(0, 0)と考えると、マス(x, y)を右へ90度回した場所は(-y, x)です。ただし回転後に壁へめり込むことがあります。そこでずらす候補を小さい順に試します。

1

4マスを同じ式で回す

T字もL字も「中心から何マス離れたか」の4座標です。それぞれへ同じ変換を使います。

rotated = point{-p.y, p.x}
2

候補を順番に試す

まず移動なし。だめなら左1、右1、左2、右2、上1。最初に衝突しない候補を採用します。

kicks := []point{{0,0}, {-1,0}, {1,0}}
3

全部だめなら回さない

候補を全部調べても壁・床・積んだマスに重なるなら、位置も向きも元のままにします。

if noCandidate { return }

IN THE REAL GAME

回転とずらしを一組で判定

候補の位置で回転後の4マスを作り、collidesがfalseになった最初の組だけをゲームへ反映します。

nextRotation := (rotation + 1) % 4
for _, kick := range kickTests {
    candidate := pos.add(kick)
    if !collides(candidate, nextRotation) {
        pos = candidate
        rotation = nextRotation
        break
    }
}

WHY CENTER?

形ごとの式がいらない

中心からの座標で形を表すと、T字にもL字にも同じ回転計算を使えます。

WHY ORDER?

勝手な大移動を防ぐ

移動なしから試すので、必要なときだけ最小の壁蹴りが選ばれます。
