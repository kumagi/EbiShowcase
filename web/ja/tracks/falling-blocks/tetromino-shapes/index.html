4マスの形

4個のマスを中心からの相対座標で覚え、ばらばらにせず一つの形として落とします。

このステップで学ぶこと相対座標と形データ

PLAYABLE / WEBASSEMBLY

4種類の形を点線のドックへ届けよう。

左右ボタンまたは←→・A/Dで移動し、中央ボタンかSpaceですぐ落とします。3回外す前にI・O・T・Sの全部をぴったり置けばクリアです。

● LIVE / GO + WASMSTEP 02

DEEP DIVE / RELATIVE COORDINATES

形は「親の場所＋4個のずれ」で動かせる。

4個それぞれの画面座標を動かすと、途中で離れてしまうかもしれません。そこで形に一つの基準点 (x, y) を持たせ、各マスはそこからのずれだけを覚えます。親を1マス動かせば、4個が同時に動きます。

1

形をデータで書く

O形なら、基準から右、下、右下の4地点です。絵ではなく数字なので、同じ描画や当たり判定をどの形にも使えます。

{{0,0}, {1,0}, {0,1}, {1,1}}
2

親の座標を足す

実際の盤面マスは x + offset.x。左へ動くときは親のxだけを1減らせば十分です。

cellX := piece.x + offset.x
3

4マス全部を検査

一つでも壁の外へ出るなら、その移動は取り消します。「全員が入れるか」を確かめてから親を動かします。

for _, p := range shape.blocks { check(x+p.x) }

TRY IT / OFFSET LIST

一つの形へマスを集めよう

「ずれを追加」で形の部品を増やします。ゲームでも4個のずれを順番に読み、同じ親座標を足します。

IN THE REAL GAME

Drawも衝突判定も同じ配列を読む

形データを一か所に置くと、描いた形と当たり判定の形が食い違いません。新しい形も4個の座標を追加するだけです。

type Shape struct {
    blocks [4]Point
}

for _, offset := range shape.blocks {
    boardX := piece.x + offset.x
    boardY := piece.y + offset.y
    drawCell(boardX, boardY)
}

WHY RELATIVE?

一つの物体として動く

親だけを動かすので、4マスの間隔はいつでも同じです。

WHY DATA?

ルールを使い回せる

I形でもT形でも、同じループで描画と壁判定ができます。
