SYSTEM LAB

素早さ・行動ゲージ・待ち時間

ゲーム内の処理を一段ずつ進め、データが次の処理へ渡る順番を確かめます。

START

UPDATE と DRAW の境界

数字を決める場所と、絵にする場所を分けよう

1

Update が担当

キー・タッチを読み、位置・HP・ゲージ・タイマー・盤面を変えます。判定やキュー操作もこちらです。

入力 → ルール → game を更新
2

Draw が担当

できあがった game の値を読んで、絵・文字・エフェクトを置きます。game の値は変えません。

game を読む → ピクセルへ投影

このページのルールは Update 側（または Update が呼ぶ純粋な関数）へ置き、Draw はその結果を表示するだけにします。

GO + EBITENGINE

素早さ・行動ゲージ・待ち時間

実ゲームで使っている中心ルールを小さく抜き出します。

actor.gauge += actor.speed
if actor.gauge >= maxGauge {
    actor.ready = true
}
// command resolved
actor.gauge = 0

なぜこの分け方？

全員が順番を待つのではなく、速いキャラクターほど早く、何度も行動できます。ゲージは時間を数値として保存する時計です。

データを見る

上限に届いた役だけ行動できます。
