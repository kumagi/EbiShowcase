素早さが行動順を作るバトル

ゲージ式アクティブRPG

味方と敵の行動ゲージへ素早さを足し、満タンになった仲間からコマンドを選びます。

はじめて読む人へ

このコースの組み立て図

素早さ→ゲージ→READY→コマンド→演出の順に、海老・天次郎（えび・てんじろう）と一部品ずつ作ります。

このページの1本
説明と次のコードを対応させてから、ゲームで変化を確かめよう。

func (g *Game) Update() error { return nil }

共通基礎の続きです。LEVEL 01 の Update / Draw に、このジャンル固有の仕組みを統合します。

Ebi Active Battle

全員が順番を待つのではなく、速いキャラクターほど早く、何度も行動できます。ゲージは時間を数値として保存する時計です。

01

素早さのゲージ競走

3人のゲージを同時に進め、素早さの差が行動回数の差になる様子を観察します。

時間をゲージへ足して順番を作る
→ 02

READYを待ち行列に入れる

ほぼ同時に満タンになった仲間を順番に保存し、行動を取りこぼさず解決します。

満タンの役を安全に一列へ並べる
→ 03

攻撃・大技・回復の役割

安全な攻撃、反動のある大技、回復を比べ、READYの一回を何に使うか選びます。

同じ入力タイミングへ違う効果を結ぶ
→ 04

行動を3つの時間で見せる

予備動作、命中、戻りへ分け、命中フレームだけでダメージと粒子を発生させます。

数値が変わる瞬間と絵を合わせる
→ 05

3つの敵編成をデータにする

敵の数、速さ、役割、背景を編成表にし、回復をはさんだ3連戦と成績を作ります。

戦闘ルールを変えずに難しさを組み立てる
→ 06
