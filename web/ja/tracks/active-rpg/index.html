SPECIALIZATION / 好きなジャンルから開始

VFXは必要な時に戻る。
まずアクティブ戦闘RPGのルールを作る。

新しい課題は速度ゲージ、READY列、入力待ち、敵行動、戦闘継続です。
下の3つは演出で迷った時の参考ページです。未受講でも、このコースから始められます。

MOTION BASELINE

READYを型付き列、各行動をリアクション時間軸、必殺技と戦闘終了をCue台本で再生します。

はじめて読む人へ

このコースの組み立て図

素早さ→ゲージ→READY→コマンド→演出の順に、海老・天次郎（えび・てんじろう）と一部品ずつ作ります。 このページの1本はコース全体で共通するgame状態の境界です。ルールはUpdateへ足し、Drawの順番や見た目は自由に差し替えられます。下の01から順に、その中へ各仕組みを実装していきます。 ゲージだけを足すと、同じtickに満タンになった役を一つの変数で上書きしてしまいます。READYキューなら、満タンになった順を全員ぶん保存してから一人ずつ解決できます。

このページの1本
説明と次のコードを対応させてから、ゲームで変化を確かめよう。

func (g *Game) Update() error { return nil }
01 素早さ→02 ゲージ→03 READY→04 コマンド→05 演出

共通基礎の続きです。LEVEL 01の状態境界に、このジャンル固有の仕組みを統合します。ルールはUpdate、見た目は自由なDrawです。

3人のゲージを同時に進め、素早さの差が行動回数の差になる様子を観察します。

時間をゲージへ足して順番を作る
→ 02

READYを待ち行列に入れる

ほぼ同時に満タンになった仲間を順番に保存し、行動を取りこぼさず解決します。

満タンの役を安全に一列へ並べる
→ 03

攻撃・大技・回復の役割

安全な攻撃、反動のある大技、回復を比べ、READYの一回を何に使うか選びます。

同じ入力タイミングへ違う効果を結ぶ
→ 04

行動を3つの時間で見せる

数値が変わる瞬間と絵を合わせる
→ 05

3つの敵編成をデータにする

敵の数、速さ、役割、背景を編成表にし、回復をはさんだ3連戦と成績を作ります。

戦闘ルールを変えずに難しさを組み立てる
