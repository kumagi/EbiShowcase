リズムゲーム

時間のずれを測る1入力から、4レーン、長押し、連打、難度、3曲のツアーまで進みます。音源素材は借りず、拍を視覚化します。

SPECIALIZATION / VFX基礎は習得済み

演出を足す練習ではなく、
リズムゲームのルールを作る。

新しい課題は音楽時刻、判定窓、譜面、長押し、連打、入力補正です。

A02型付き演出レシピA12決定的エフェクト台本A05上限つき描画履歴

MOTION BASELINE

動いて見えることも完成条件

判定を型付きレシピへ写し、曲と結果画面をCue台本、最近の入力差を上限つき履歴として可視化します。

TEST SEAM

派手さを外してもルールは同じ

音源なしの仮想frameで判定境界を試し、粒子や画面揺れが譜面時計を停止・加速しないこと。

LEVEL 01のUpdate / Drawを、譜面時刻と入力時刻を比べるゲームへ広げます。

LEARNING PATH

1拍から、3曲のツアーへ。

01

拍を見てタップする

光の脈が判定線へ届く瞬間にタップし、時間のずれを数えます。

tick差からタイミングを判定
→ 02

ノートを先読みする

上から落ちるノートの移動時間を読み、判定線で入力します。

譜面時刻を画面位置へ変換
→ 03

4レーンの入力をさばく

D・F・J・Kと4つのタッチ領域をlane番号へまとめます。

入力方法を同じlaneへ変換
→ 04

長押しの始まりと終わり

ノートの頭で押し、しっぽまで離さず保つ状態を作ります。

押下中という状態を時間をまたいで保存
→ 05

連打区間を数える

ピンクの区間が続く間に何回押したかを数え、必要回数と比べます。

時間の区間と入力回数
→ 06

譜面データで難度を変える

同じ再生プログラムへEASYとHARDのノート列を渡して遊び比べます。

ルールではなくデータを差し替える
→ 07
