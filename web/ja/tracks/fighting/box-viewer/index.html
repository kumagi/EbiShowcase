DEEP DIVE / HITBOXES

どうして「当たった」が
わかるの？

ゲームの中では見えない 矩形（箱） を3種類用意します。青い押し合い判定 は体同士が通り抜けないよう押し返し、緑の喰らい判定 は攻撃を受ける面積を決め、赤い攻撃判定 は技が出ている瞬間だけ現れます。赤と緑が重なった時にだけヒットが成立します。

1

押し合い判定（Push Box）

青い矩形が重なるとキャラクターを押し戻します。壁と同じように体が通り抜けるのを防ぎます。押し合いは毎フレーム調べ続けます。

if overlap(push1, push2) { p1 = p2 - 36 }
2

喰らい判定（Hurt Box）

攻撃を受ける領域です。体より少し大きめに設定すると当たりやすい爽快感が出ます。相手の攻撃判定と比べる対象です。

hurt := rect{p2 - 24, 500, 48, 105}
3

攻撃判定（Hit Box）

技が有効なフレームにだけ生成します。判定が出ている間に喰らい判定と重なったらヒットです。存在時間が短いほど技は鋭くなります。

hit := rect{p1 + 15, 535, 90, 32}

赤い攻撃判定と緑の喰らい判定が重なったときだけヒットです。「近づける／離す」で距離を変えられます。

押し合い判定（青）は別枠です。ここでは攻撃と喰らいだけに注目します。

AABB OVERLAP TEST

a.x < b.x+b.w // aの左端 < bの右端 かつ a.x+a.w > b.x // aの右端 > bの左端 Y軸も同じように判定する a.y < b.y+b.h && a.y+a.h > b.y

4つの条件がすべて満たされたとき、2つの矩形は重なっています。この判定を「AABB（軸平行境界ボックス）テスト」と呼び、格闘ゲームだけでなく多くの2Dゲームで使われます。

IN THE REAL GAME

押し返す → 判定を出す → 重なりを調べる

まず青同士が重なったらキャラクターを押し戻します。次に技のフレームになったら赤い箱を作り、緑と重なるか調べます。3つの処理を順番に行うのがポイントです。

push1 := rect{g.p1 - 18, 510, 36, 100}
push2 := rect{g.p2 - 18, 510, 36, 100}
if overlap(push1, push2) {
    g.p1 = g.p2 - 36   // p1を外へ押し戻す
}

if g.attack == 12 {   // 技のアクティブフレーム
    hit  := rect{g.p1 + 15, 535, 90, 32}
    hurt := rect{g.p2 - 24, 500, 48, 105}
    if overlap(hit, hurt) {
        g.hits++
    }
}

func overlap(a, b rect) bool {
    return a.x < b.x+b.w && a.x+a.w > b.x &&
           a.y < b.y+b.h && a.y+a.h > b.y
}

WITHOUT BOXES

「通り抜け」も「当たり」も管理できない

位置の数値だけでは面積を表せません。矩形を使うことで「どこからどこまで」をXY・幅・高さの4つの数値で表現できます。

WITH SEPARATE BOXES

役割ごとに独立して調整できる

喰らい判定だけ広くする、攻撃判定を短くするなど、見た目を変えずに当たり感を細かく調整できます。これが格闘ゲームの「感触設計」の核心です。

YOUR FIRST TUNING

攻撃判定の幅を変えてみよう

rect{p1 + 15, 535, 90, 32} の幅 90 を 120 にすると、遠くからでも当たる技になります。逆に 50 にすると近接専用の技になります。
