DEEP DIVE / COMMAND BUFFER

どうして連続入力が
必殺技になるの？

答えは入力履歴です。方向を記号（D, DR, R）にして並べ、パンチ時に末尾がコマンド表と一致すれば必殺技。古い入力は age で消えます。

1

履歴

方向が変わるたびトークンを append。長さ上限で古いものを捨てます。

history = append(history, token)
2

有効期限

age を減らし、0 で履歴を空にします。

if age == 0 { history = nil }
3

末尾照合

パンチ時に ends(history, {D,DR,R}) なら必殺技。

ends(history, []string{"D","DR","R"})

THE MATCH RULE

history ends with D, DR, R かつ punch pressed → special

途中の余分な入力は末尾一致なら許します。期限切れでリセットするので、ゆっくりすぎる入力は失敗します。

IN THE REAL GAME

パンチで末尾を照合

履歴へ方向を足し、パンチ時に ends で必殺技判定します。

if punch {
    if ends(g.history, []string{"D", "DR", "R"}) {
        g.projectile = 35
        g.message = "EBI WAVE!"
    }
    g.history = nil
}
