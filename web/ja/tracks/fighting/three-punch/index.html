DEEP DIVE / FRAME DATA

どうしてボタンを押した
瞬間に当たらないの？

技には 発生（Startup）・持続（Active）・硬直（Recovery） の3段階があります。ボタンを押すと「発生」が始まり、腕が伸びきるまでの間（フレーム1〜8）は当たりません。フレーム9〜12の4フレームだけ赤い攻撃判定が生まれ、フレーム13〜30は元の姿勢に戻るまで次の行動ができません。

1

発生（Startup）

フレーム1〜8は腕を伸ばす準備中です。攻撃判定はまだ出ていません。この時間が長いと「遅い技」、短いと「速い技」になります。

frame <= 8 → STARTUP（黄色）
2

持続（Active）

フレーム9〜12の4フレームだけ赤い攻撃判定が存在します。この時間に相手の喰らい判定と重なったときだけヒットです。

frame 9〜12 → ACTIVE（赤）
3

硬直（Recovery）

フレーム13〜30は姿勢を元に戻す時間です。この間は次のボタン入力を受け付けません。硬直が長いと「隙が大きい技」になります。

frame >= 13 → RECOVERY（青）

FRAME RANGES

STARTUP フレーム 1 〜 8 // まだ攻撃できない ↓ ACTIVE フレーム 9 〜 12 // 攻撃判定が存在する ↓ RECOVERY フレーム 13 〜 30 // 次の行動ができない

フレームは毎秒60回カウントされます（Ebitengineのデフォルト）。1フレームは約16.7ミリ秒です。フレーム9のときボタンを押しても、この技のカウンターが先に動くため同じtickでは当たりません。

IN THE REAL GAME

押す → カウンターを進める → ACTIVE で当たりを調べる

ボタンを押したtickで g.frame = 1 にし、tickごとに1ずつ増やします。フレーム9〜12のときだけ攻撃判定を作り、相手の喰らい判定と重なるか調べます。

if g.frame == 0 && press() {
    g.frame = 1        // ボタンを押したら開始
    g.hitThis = false
} else if g.frame > 0 {
    g.frame++
    if g.frame >= 31 { g.frame = 0 }  // 技終了
}

// ACTIVE フレームだけ攻撃判定を作る
if g.frame >= 9 && g.frame <= 12 && !g.hitThis {
    attack := rect{235, 495, 105, 35}
    if overlap(attack, g.target()) {
        g.hits++
        g.hitThis = true    // 同じ技で2回当たらないよう
    }
}

WITHOUT FRAME DATA

ボタンを押したら即当たりになってしまう

フレーム分割がなければすべての技が瞬時に当たります。読み合いや反応の面白さが生まれません。

WITH FRAME DATA

技に「個性」が生まれる

発生が速い技は隙が大きく、発生が遅い技は当たったときのリターンが大きいなど、バランス設計の基礎になります。

YOUR FIRST TUNING

ACTIVE フレームの長さを変えてみよう

g.frame <= 12 を g.frame <= 16 にすると ACTIVE が8フレームに延び、当てやすい技になります。逆に 10 にすると瞬間的な技になります。
