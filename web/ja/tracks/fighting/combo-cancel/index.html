DEEP DIVE / COMBO CANCEL

どうして技の途中に
次の技がつながるの？

格闘ゲームのコンボは「入力バッファ」と「キャンセルウィンドウ」の組み合わせで実現します。ボタンを押した瞬間にバッファへ技を保存し、最大8フレームだけ「次の技のリクエスト」を覚えておきます。技が当たり（frame == 8）、キャンセル可能な時間（フレーム8〜13）に入ると、バッファにある技をすぐ開始します。タイミングが合えばコンボがつながります。

1

入力バッファ

ボタンを押したフレームに buffer = 技 と bufferLife = 8 をセットします。毎フレーム残り時間を減らし、0になったら忘れます。

buffer = light; bufferLife = 8
2

キャンセルウィンドウ

フレーム8〜13の6フレーム間だけ、今の技を中断して次の技を始められます。バッファに技があり、正しい順序のときだけキャンセルが発動します。

cancel := frame >= 8 && frame <= 13
3

コンボ進行カウンター

combo 変数で何段目かを管理します。1段目と2段目は弱攻撃、3段目は強攻撃のみ受け付けます。順序が違うと combo がリセットされます。

if combo < 2: expected = light else: heavy

TRY IT / LAB

早め入力をバッファしてキャンセル

早めの入力を数フレーム覚え、キャンセル窓で次の技へつなぎます。

本物のゲームの公式を、手でゆっくり回す模型です。

—

COMBO TIMING

バッファ有効期間: 8フレーム（≈ 133ms） 技の構成 LIGHT: 20フレーム (発生8 + 持続1 + 硬直11) ↓ フレーム8〜13でキャンセル可能 HEAVY: 28フレーム (発生8 + 持続1 + 硬直19)

キャンセルウィンドウ（フレーム8〜13）は技の当たった直後の6フレームです。ここでバッファに次の技がなければコンボは終了します。バッファは8フレームあるので、少し早めに押しても間に合います。

IN THE REAL GAME

押す → 覚える → 条件を待つ → つなぐ

ボタンを押したらバッファへ保存し、毎フレーム減らします。キャンセル条件が整ったとき、期待する技と一致すればすぐ開始します。コンボカウンターで進行状況を追います。

// 入力があればバッファに保存
if in != none {
    g.buffer = in
    g.bufferLife = 8   // 8フレーム覚える
}
if g.bufferLife > 0 {
    g.bufferLife--
} else {
    g.buffer = none
}

// キャンセル: 当たったフレームから6フレーム間
cancel := g.frame >= 8 && g.frame <= 13
if cancel && g.buffer != none {
    expected := light
    if g.combo >= 2 { expected = heavy }
    if g.buffer == expected {
        g.start(g.buffer)   // 次の技を即開始
    }
}

WITHOUT BUFFER

コンボが繋がらないもどかしさが残る

バッファなしでは技の終わりの1フレームを正確に押す必要があります。人間には難しく、コンボが成立しにくくなります。

WITH BUFFER

「少し早めに押す」でつながる快感

8フレームの猶予があることで、技の感触を確認しながら次を押せます。スキルの上達に伴って精度を上げる余地も生まれます。

YOUR FIRST TUNING

バッファ期間を変えてみよう

bufferLife = 8 を 16 にすると押しやすいコンボに、4 にするとシビアな上級者向け仕様になります。cancel := frame >= 8 && frame <= 13 のウィンドウも調整してみましょう。
