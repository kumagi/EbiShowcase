ドットを食べる迷路

文字マップから固定の壁と変化するドットを別配列へ作り、取得のたびに状態と残数を更新します。

このステップで学ぶこと固定レイヤー・収集状態・残数によるクリア

PLAYABLE / WEBASSEMBLY

赤い追跡役を避け、60秒以内に全ドットを集めよう。

LEFT/UP/DOWN/RIGHTを長押し。PCでは矢印またはWASD、マウスでもボタンを長押しできます。残数0でクリア、敵と同じマスまたは時間切れで失敗です。

● LIVE / GO + WASMSTEP 03

DEEP DIVE / STATIC AND MUTABLE LAYERS

同じマスにある「通れるか」と「まだあるか」を分ける。

迷路の壁はプレイ中に変わりませんが、ドットは取るたびに消えます。両方を一つの文字へ押し込む代わりに、wallsとpelletsという2枚の同じ大きさの表を重ねます。位置は同じ[y][x]でも、固定データと可変状態の役目が違います。

1

文字列を2レイヤーへ読む

#ならwallsをtrue、.ならpelletsをtrueにします。Drawは壁を先に、その上へ残っているドットだけを描きます。

walls[y][x] = tile == '#'; pellets[y][x] = tile == '.'
2

取得はドットだけを書き換える

プレイヤーが新しいタイルへ入った時、pelletsがtrueならfalseへ変えます。同じ場所をもう一度通っても得点は増えません。

if pellets[y][x] { pellets[y][x] = false }
3

残数を同時に1減らす

毎フレーム盤面全部を数え直さず、取得成功と同じ場所でremainingを減らします。0になった瞬間がステージクリアです。

remaining--; if remaining == 0 { clear = true }

TRY IT / PELLET LAYER

残ドットを1個ずつ消そう

ドットは迷路と別レイヤーの bool 配列です。通ったら false にし、残り0でクリア。壁データは変えません。

初期配置だけマップデータ、実行中の残りは状態です。

IN THE REAL GAME

成功した取得を一つの関数にまとめる

初期位置や敵位置のドットを消す時も同じ関数を使います。すでにfalseなら何もしないため、残数が二重に減りません。

func (g *Game) removePellet(p Point) {
    if g.pellets[p.y][p.x] {
        g.pellets[p.y][p.x] = false
        g.remaining--
    }
}

removePellet(player)
if remaining == 0 { clear = true }

WHY TWO LAYERS?

壁を壊さず収集できる

ドットをfalseにしてもwallsはそのままなので、通路判定が収集状態に影響されません。

WHY A COUNTER?

クリア判定が一定時間

マップが大きくなっても、remainingが0かを見るだけなら1回の比較です。
