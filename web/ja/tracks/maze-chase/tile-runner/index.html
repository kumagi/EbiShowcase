迷路を走る

迷路を壁と通路のタイルで表し、通れる次タイルの中心までまっすぐ進みます。

このステップで学ぶことタイル中心・通行判定・中心間移動

共通基礎の続きです。ゲームの1コマが初めてなら、先にLEVEL 01 の Update / Drawを見てください。ここではUpdateが現在タイルと移動の進み具合を変え、Drawがその途中位置へ走者を描きます。

PLAYABLE / WEBASSEMBLY

45秒以内に迷路の真珠を12個集めよう。

中心で進みたい方向を押し続けます。壁方向なら出発しません。矢印/WASD、画面下ボタンのマウス長押し・タッチ長押しに対応。

● LIVE / GO + WASMSTEP 01

DEEP DIVE / CENTER TO CENTER

自由な座標を、迷路のマスへ結びつける。

迷路データはmaze[y][x]の文字表です。#なら壁、.なら通路。走者は現在タイルを覚え、方向を足した候補タイルが通路のときだけ、その中心へ移動を始めます。

1

タイルから中心座標を作る

列xへタイル幅を掛け、盤面左と半マスを足します。行yも同じ式なので、どの通路にも中心があります。

centerX = mazeX + tileX*tileSize + tileSize/2
2

次タイルが通れるか調べる

現在タイルへ方向(-1,0)などを足します。範囲内でmaze[nextY][nextX]が#でなければ進めます。

passable := maze[next.y][next.x] != '#'
3

中心から中心へ補間する

progressを0から1へ増やし、fromとtoの間を描きます。1になったら現在タイルをtargetへそろえます。

screenX = fromX + (toX-fromX)*progress

TRY IT / LEGAL TILE

壁のマスには入れない

矢印は行と列を整数で動かします。赤いマスは壁。入れなければ座標は変わりません。本番は通った2つの中心の間だけを滑らかに描きます。

中心に着いてから次の入力を受け付けると、通路からずれません。

IN THE REAL GAME

中心でだけ新しい移動を始める

moving中はprogressだけを進めます。中心へ到着してmoving=falseのときだけ入力を候補タイルへ変換します。

if moving {
    progress += 0.105
    if progress >= 1 {
        tile = target
        moving = false
    }
    return
}

next := tile.add(direction)
if passable(next) {
    target = next
    moving = true
}

WHY TILE DATA?

壁判定が一文字になる

複雑な壁の四角を全部比べず、行と列から#かどうかを見るだけで通行を決められます。

WHY CENTERS?

通路からずれない

毎回中心へ到着してから次へ進むため、長く走っても少しずつ壁側へずれません。
