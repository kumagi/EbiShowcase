曲がり角の先行入力

PLAY FIRST

光るゲートを10個通ろう

矢印／WASD、クリック、タッチで次の方向を早めに入力します。CURRENTとQUEUEDの表示を比べてください。

DEEP DIVE

「いま進む向き」と「次に進みたい向き」を分ける

入力ですぐ向きを変えると、壁にめり込んだり中心からずれたりします。そこでcurrentとqueuedを別々に持ちます。

01

押した方向を保存

キーやタッチを受けたら、まずqueuedだけを書き換えます。

02

中心で予約を試す

次のタイルが通路ならqueuedをcurrentへ移します。

03

だめなら直進

予約方向が壁なら消さずに待ち、今の方向へ進める間は直進します。

TRY IT / CURRENT vs QUEUED

押した向きを予約して、中心で反映

方向ボタンは queued だけを変えます。「移動中」にしてから「中心到着」で、予約が current に入ります。壁判定はこの瞬間だけです。

すぐに current を書き換えると、マスの途中で壁にめり込みやすくなります。

THE BUFFER RULE

input → queuedつぎにat center: current = queued if passable

入力の瞬間と向きの確定を分けます。

GO + EBITENGINE

中心に着いた瞬間だけ向きを決める

Updateは毎フレーム入力を保存できますが、方向変更はタイル到着時だけにします。

if dir, ok := directionInput(); ok {
    queued = dir
}
if reachedTileCenter {
    if passable(tile.add(queued)) {
        current = queued
    } else if !passable(tile.add(current)) {
        return // 中心で止まる
    }
    target = tile.add(current)
}

なぜ操作しやすくなる？

ぴったり1フレームで押さなくてもよくなり、プレイヤーの「曲がりたい」をゲームが少し待てるからです。

予約はいつ消す？

このゲームでは成功するまで残します。短い時間で消す方式もあり、操作感を調整できます。

チャレンジ

予約を10フレームだけ有効にして、永久保存との遊びやすさを比べてみよう。
