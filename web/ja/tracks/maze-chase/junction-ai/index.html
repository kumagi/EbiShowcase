敵に「性格」を持たせる

交差点AI

敵は交差点で進める方向を調べ、引き返す方向を外し、ターゲットまでの距離が短い道を選びます。ねらう場所を変えるだけで追い方も変わります。

★★★★☆PLAYABLE

PLAY FIRST

2つの予測をかわしてチップを8個集めよう

矢印／WASD、クリック、タッチで移動します。赤いHUNTERは今いるマス、水色のAMBUSHERは進行方向の3マス先をねらいます。色の輪が敵のターゲット、黄色い線が選んだ方向です。

DEEP DIVE

追いかけ方を「候補・評価・性格」に分ける

敵をプレイヤーへ直接動かすのではなく、まず通路というルールを守れる候補だけを集めます。それぞれの次のマスからターゲットまでのマンハッタン距離を計算し、いちばん小さい方向を選びます。

01

合法方向を列挙

上下左右の次のマスを調べ、壁ではない方向だけを候補にします。

02

逆走を除く

候補が複数あるときは来た道を外します。ただし行き止まりでは引き返せるようにします。

03

性格の目標を評価

HUNTERは現在地、AMBUSHERは3マス先を目標にし、距離が最小の候補を選びます。

TRY IT / JUNCTION

交差点で1本だけ選ぶ

通路の交点では、入れる方向のうち目標に一番近いものを1つ選びます。Uターン禁止などのルールもここで足せます。

候補を列挙 → スコア付け → 最大を採用、が基本形です。

採用chase:right

SCORE EDGES

candidates = open neighborsつぎにpick min distance to target

ランダムより、距離スコアの方が読みやすい敵になります。

GO + EBITENGINE

同じ選択関数へ性格ごとのtargetを渡す

aheadが0なら今いるマス、3なら進行方向の3マス先です。選択の仕組みを共通にし、データだけで敵の個性を作れます。

target := player
if enemy.personality.ahead > 0 {
    target = player.add(playerDirection.mul(3))
}

legal := legalDirections(enemy.position)
if len(legal) > 1 {
    legal = removeReverse(legal, enemy.direction)
}
best := legal[0]
for _, dir := range legal[1:] {
    if distance(enemy.position.add(dir), target) <
       distance(enemy.position.add(best), target) {
        best = dir
    }
}

なぜ逆走を禁止する？

毎回いちばん近い方向だけを選ぶと、敵が2マスを往復しやすくなります。交差点で前へ進ませると追跡に意志があるように見えます。

攻略にどう使う？

HUNTERには曲がって距離を作り、AMBUSHERには直進を読ませてから早めに別の通路へ入ります。性格を読むことがプレイになります。

チャレンジ

遠くではプレイヤーを追い、近くでは自分の担当する角へ逃げるSCATTER型を追加してみよう。
