DEEP DIVE / XP AND LEVEL-UP

レベルアップで
ゲームが変わるのはなぜ？

経験値 xp が閾値 need を超えると draft = true になり、ゲームの更新が一時停止してカード選択画面になります。カードを選ぶと能力値が書き変わり、need がさらに増えます。数字を変えるだけでキャラクターの「成長」が生まれます。

1

ジェム引き寄せ

自機の90ピクセル以内のジェムはtickごとに自機に向かって引き寄せられます。18ピクセル以内で取得、XPに加算します。

if d < 90 { gem に近づける }
2

レベルアップ判定

xp >= need になった瞬間に draft = true を立てます。Updateは別関数 updateDraft に切り替わり、ゲームが一時停止します。

if g.xp >= g.need { g.draft = true }
3

ドラフト選択

3枚のカードからキーまたはタッチで選択。選んだ能力値が書き変わり、need を3増やして次のレベルアップをより遠くします。

g.speed += .65 など

THE LEVEL-UP RULE

if g.xp >= g.need { g.draft = true }選択後g.level++; g.xp = 0; g.need += 3

need を毎レベルで増やすことで、レベルが上がるほどたくさん経験値を集める必要が生じます。この「加速的な閾値」がゲームに長さを生み出します。

IN THE REAL GAME

選択 → 能力値書き換え → 再開

キーまたはタッチで選択したカードに応じて speed・radius・life のいずれかを直接加算します。フラグを下ろすだけで通常の更新に戻ります。

func (g *game) updateDraft() error {
    choice := -1
    if inpututil.IsKeyJustPressed(ebiten.Key1) { choice = 0 }
    if inpututil.IsKeyJustPressed(ebiten.Key2) { choice = 1 }
    if inpututil.IsKeyJustPressed(ebiten.Key3) { choice = 2 }
    if choice < 0 { return nil }
    switch choice {
    case 0: g.speed += .65
    case 1: g.radius += 14
    case 2: g.life++
    }
    g.level++; g.xp = 0; g.need += 3
    g.draft = false
    return nil
}

WITHOUT DRAFT

ゲームが最初から最後まで同じ

成長要素がないと、プレイヤーは同じ操作を繰り返すだけで飽きます。レベルアップが「変化」を生み、続ける理由になります。

WITH DRAFT

選択がゲームを別物にする

3枚のカードの組み合わせで毎回違う強さになります。何を選ぶかの判断が楽しさの本体です。

YOUR FIRST TUNING

カードの効果を変えてみよう

g.speed += .65 の量を変えると速さの成長が変わります。need += 3 を大きくするとレベルアップが遅くなり、難易度が上がります。
