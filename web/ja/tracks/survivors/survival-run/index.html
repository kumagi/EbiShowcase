DEEP DIVE / WAVES AND DIFFICULTY

どうして時間とともに
ゲームが難しくなるの？

スポーン間隔と移動速度を経過秒数から計算します。interval = max(14, 42 - sec/2) で間隔が徐々に短くなり、speed = 0.85 + sec * 0.018 で速度が線形に上昇します。式1本が難易度曲線全体を決めるのが時間ベースの難易度スケーリングです。ボスは固定秒数で登場し、HPバーとともに別の脅威を加えます。

1

スポーン間隔の縮小

42フレームから始まり、経過秒数の半分ずつ短くなります。ただし14フレームを下限にして無限スポーンを防ぎます。

interval := max(14, 42-sec/2)
2

速度の線形上昇

敵の移動速度は0.85から始まり1秒ごとに0.018増えます。35秒後には約1.48、ウェーブ3では明らかに速くなっています。

speed := .85 + float64(sec)*.018
3

ボスとHPバー

35秒で出現する紫の大きな敵は HP 60 です。オーラが14フレームごとに1ずつ削り、ゼロになったらクリアです。HPバーは矩形2つで描きます。

g.mobs = append(g.mobs, mob{..., 60, 38, true})

TRY IT / LAB

秒数が間隔と速度を変える

経過秒で出現間隔を短くし、敵速度を上げます。難しさは時間の関数です。

本物のゲームの公式を、手でゆっくり回す模型です。

max(14, 42-sec/2)
sec0
interval42
speed0.85

THE DIFFICULTY CURVE

interval := max(14, 42 - sec/2)そしてspeed := .85 + float64(sec) * .018

この2式でウェーブ1〜3の難易度差が生まれます。42 - sec/2 は放物線ではなく直線なので、難易度上昇のペースが予測しやすく、プレイヤーが「慣れ→緊張」のリズムを体感できます。

IN THE REAL GAME

時間→間隔→速度→ボス出現

毎フレーム経過秒数を計算し、スポーン間隔と移動速度を更新します。35秒に達したらボスを1体追加。ボスはスピードが遅い代わりにHP 60 を持ちます。

sec := g.frame / 60
interval := max(14, 42-sec/2)
if g.frame%interval == 0 && sec < 35 {
    g.mobs = append(g.mobs, mob{..., 1, 13, false})
}
if sec >= 35 && !g.bossSpawned {
    g.bossSpawned = true
    g.mobs = append(g.mobs, mob{240, 95, 60, 38, true})
}
speed := .85 + float64(sec)*.018
for i := range g.mobs {
    m := &g.mobs[i]
    ms := speed
    if m.boss { ms = .65 }
    m.x += (g.px - m.x) / d * ms
}

WITHOUT SCALING

一定の難易度はすぐ飽きる

スポーン間隔も速度も固定だと、プレイヤーが最初の30秒で「このゲームはこれだけ」と判断してしまいます。

WITH DIFFICULTY CURVE

緊張のリズムが続きを促す

時間とともに圧力が増すと、プレイヤーは「もう少しだけ」と耐えようとします。ボス出現がその緊張の頂点になります。

YOUR FIRST TUNING

ウェーブの長さとボスHPを変えよう

42 - sec/2 の 2 を大きくすると急加速します。ボスの HP 60 を増やせばボス戦が長くなり、ゲームに重みが増します。
