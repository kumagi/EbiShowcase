DEEP DIVE / AUTO AIMING

どうして砲台は
勝手に狙えるの？

全ての敵との距離を一つずつ比べて、最も近い敵だけを記録します。この「全部と比べる」ループが最近傍探索です。その敵への方向を正規化して速度にするだけで、自動照準の完成です。クールダウンで発射間隔を制限すれば、Survivors型攻撃の基本形になります。

1

最近傍探索

全ての敵との距離を順番に測ります。これまでの最小値より小さければ更新し続けます。最後に残ったのが最近傍です。

if d < dist { best = i }
2

クールダウン

毎フレーム発射すると多すぎます。frame%28==0 で28フレームに1回だけ照準・発射します。間隔を変えると連射速度が変わります。

if g.frame%28 == 0
3

弾の速度計算

「敵の位置 − 自機の位置」を距離で割って正規化し、速さ7を掛けます。これで敵に向かうちょうど良い速度ベクトルになります。

vx = (e.x - g.px) / dist * 7

THE TARGETING RULE

best, dist := 0, math.MaxFloat64全敵をループ → 最近傍に向けてvx = (e.x - g.px) / dist * 7

「最大値から始めて、より小さければ更新」のパターンは最小値を探す定番の書き方です。距離の比較には毎回 math.Hypot を使い、最近傍が決まったら同じ距離を割り算に再利用します。

IN THE REAL GAME

探す → 待つ → 狙って撃つ

28フレームごとに最近傍ループを走らせ、その敵への正規化ベクトルに速さを掛けた弾をスライスに追加します。弾は毎フレーム進み、画面外か敵に当たれば削除します。

if g.frame%28 == 0 && len(g.enemies) > 0 {
    best, dist := 0, math.MaxFloat64
    for i, e := range g.enemies {
        d := math.Hypot(e.x-g.px, e.y-g.py)
        if d < dist { best, dist = i, d }
    }
    e := g.enemies[best]
    g.shots = append(g.shots, body{
        g.px, g.py,
        (e.x - g.px) / dist * 7,
        (e.y - g.py) / dist * 7, 5,
    })
}

WITHOUT TARGETING

ランダム方向では当たらない

敵の方向を計算せずにランダムに撃つと、敵が増えても弾がほとんど当たらない退屈なゲームになります。

WITH AUTO AIM

プレイヤーは「避ける」に集中できる

攻撃を自動化すると、プレイヤーは移動だけを考えればよくなります。これがSurvivors型ゲームの中核設計です。

YOUR FIRST TUNING

クールダウンと弾速を変えよう

frame%28 の 28 を小さくすると連射になります。弾の速さ 7 を上げると遠くまで届き、下げると敵が接近してから当たります。
