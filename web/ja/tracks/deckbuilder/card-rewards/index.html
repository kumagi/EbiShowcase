● LIVE / GO + WASMSTEP 05

DEEP DIVE / DECK EDITING

報酬は「数字を増やす」だけでなく、次の遊び方を変える。

候補をカードデータのリストとして作り、選ばれたたった1枚だけをdeckへappendします。レア度は色や出やすさを表すデータで、強さそのものとは分けて考えます。

1

候補を作る

戦闘番号に合う報酬表から3枚を表示します。画面に見えるカードと選択時に追加するカードは同じデータです。

rewards := rewardTable[stage]
2

1枚だけ追加

選ばれた番号のカードをデッキの末尾へ入れます。ほかの2枚は追加しないので選択に意味が生まれます。

deck = append(deck, rewards[choice])
3

編集は元データを変える

強化はdamageを増やし、削除はスライスから要素を抜きます。次の戦闘は編集後のdeckを読みます。

deck[i].damage += 3

TRY IT / REWARD PICK

3枚から1枚をデッキへ足そう

報酬は候補配列です。選んだ1枚だけを自分のデッキへ append。レア度は候補の作り方、編集は配列の操作です。

スキップ（足さない）も同じ画面の選択肢にできます。

デッキ枚数0
結果—

PICK ONE

show candidates[]つぎにdeck = append(deck, choice)

候補と所持デッキは別配列です。

IN THE REAL GAME

選択番号を境界チェックする

入力が候補の範囲内にあるときだけ追加し、報酬画面を閉じて次のステージへ進めます。

if choice >= 0 && choice < len(rewards) {
    chosen := rewards[choice]
    deck = append(deck, chosen)
    stage++
    selecting = false
}

WHY CHOICE?

自分のデッキになる

毎回全部もらうのではなく1枚を選ぶから、遊ぶ人ごとに違うデッキが育ちます。

WHY REMOVE?

少ないほど狙いやすい

弱いカードを抜くと、残した強いカードを引く確率が上がります。
