カードデータは名前・必要エナジー・効果を一組にした設計図。効果解決はカードの指示どおりHPや防御を変える処理です。同じresolve関数へ違うカードを渡すと、同じ手順で別効果を実行できます。 PLAYABLEは『このページで実際に操作できる』という印です。

resolve(card)

DEEP DIVE / CARD DATA

カード1枚は、絵ではなく小さな命令書

名前、コスト、効果の種類、数値を1セットにします。画面は違っても、実行する仕組みは同じresolve関数です。

energy >= card.Cost

効果解決

Kindで攻撃・防御・回復を選びます。

switch card.Kind

TRY IT / RESOLVE

カードを使って数字を変えよう

攻撃・防御・回復は見た目が違っても、同じ「コストを払って Kind で分岐」です。エナジーが足りないと使えません。ターン終了で敵が殴り、エナジーが戻ります。

ゲーム本体も switch card.Kind で同じ解決をしています。

THE PAY → BRANCH RULE

energy -= costつぎにswitch kind { damage / block / heal }

カードを増やすときはデータ1行。解決の仕組みは変えません。

IN THE REAL GAME

データを読んで命令へ変える

カードごとに別関数を作らず、KindとValueを読みます。

if energy >= card.Cost { energy -= card.Cost; switch card.Kind { case "damage": enemyHP -= card.Value; case "block": block += card.Value; case "heal": hp += card.Value } }
