DEEP DIVE / INTENT & STATUS

「次に18ダメージ」と見えたら、作戦を変えられる。

敵が行動する前に予告データをDrawで表示します。毒や弱体は残り回数を整数で覚え、ターン終了時に効果を使ってから1減らします。

1

行動をデータにする

名前、攻撃力、防御力をひとまとめにします。同じデータを表示と実行の両方で使えば、予告と本当の行動がずれません。

type intent struct { damage, block int }
2

状態を数で覚える

毒4なら毎ターン傷つけたあと3へ。弱体2なら2ターンの間だけ敵の攻撃を小さくします。

enemyHP -= poison; poison--
3

更新順を固定する

毒、敵の攻撃、残り時間を減らす、次の予告、という順番を関数にまとめます。

resolveEnemyTurn()

TRY IT / EFFECT QUEUE

効果を処理する順番に並べよう

効果を追加してから「次を解決」。ゲームでも順番を決めておくと、毒で先に敵を倒したときなどの結果が毎回同じになります。

IN THE REAL GAME

表示した予告をそのまま実行

現在のintentを取り出し、弱体中だけ攻撃を4分の3へ。処理後に次のintentへ進みます。

in := intents[intentIndex]
damage := in.damage
if weak > 0 {
    damage = damage * 3 / 4
    weak--
}
hp -= max(0, damage-block)
intentIndex = (intentIndex+1) % len(intents)

WHY INTENT?

考えて選べる

危険が見えるから、防御、攻撃、状態効果のどれが今必要か判断できます。

WHY DURATION?

効果が終わる

残り回数が0になる規則があれば、いつまでも毒や弱体が続きません。
