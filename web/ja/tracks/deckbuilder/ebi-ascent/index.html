PLAYABLE / WEBASSEMBLY

自分のデッキを育て、深海の王を倒そう。

戦闘は1〜5・E、報酬と道は番号またはタップ。3体の敵を倒せばクリアです。

● LIVE / GO + WASMFINAL STEP

DEEP DIVE / GAME COMPOSITION

大きなゲームも、小さな画面を順番につないだもの。

戦闘、報酬、経路選択をphaseという現在の場面で切り替えます。デッキ、HP、階数は場面を越えて同じgame構造体に残るため、一つの旅になります。

1

場面を分ける

Battleではカード入力、Rewardでは3択、Routeでは2択だけを読みます。同じタップでも場面ごとに意味が決まります。

switch phase { case Battle: ... }
2

共有するデータを決める

HP、deck、floorはラン全体のデータ。energyやblockは戦闘ごとに戻す一時データです。

type game struct { deck []card; floor int }
3

出口を次の入口へ

敵を倒したらReward、報酬を選んだらRoute、道を選んだら次のBattleへ進めます。

phase = phaseReward

TRY IT / PHASE QUEUE

ゲームの場面を並べよう

場面を追加し、次へ進めてみましょう。完成ゲームはこの列を繰り返しながら、ラン全体のデータを育てます。

IN THE REAL GAME

Updateを現在の場面へ渡す

一つのUpdateを巨大にせず、場面ごとの小さな処理へ分けます。

switch phase {
case phaseBattle:
    updateBattle()
case phaseReward:
    chooseReward()
case phaseRoute:
    chooseRoute()
}

WHY ONE GAME?

選択が次へ残る

選んだカードや失ったHPが次の戦闘へ続くから、一回ごとの物語が生まれます。

WHY DATA TABLES?

敵やカードを増やせる

仕組みを変えずにencountersやrewardsへデータを足せます。
