G

Slay the Spire型ゲームを作る

デッキ構築カードゲーム

スレイ・ザ・スパイアのような、カードを選んでデッキを作りながら進むゲームを作ります。カードを1枚使うところから、手札が山札→手札→捨て札を巡る仕組み（手札循環）、敵の予告、毒や強化が数ターン続く効果（状態効果）、報酬、分岐マップへ広げます。

はじめて読む人へ

このコースの組み立て図

引く→コスト→効果→敵予告の順に、海老・天次郎（えび・てんじろう）と一部品ずつ作ります。

このページの1本
説明と次のコードを対応させてから、ゲームで変化を確かめよう。

func (g *Game) Update() error { return nil }

共通基礎の続きです。LEVEL 01 の Update / Draw の上に、このジャンルの部品を1ステップずつ足します。主人公が登場するステップではオリジナルの 海老・天次郎（えび・てんじろう） を使います。

LEARNING PATH

小さな仕組みから
完成形へ。

難しい部分を一度に作らず、単独で動くミニゲームへ分解します。各ステップで前のコードを再利用し、最後に統合します。

01

1枚のカードを使う

コスト、名前、効果をデータ化し、選んだカードの命令を実行します。

カードデータと効果解決
→ 02

エナジー制ターン

ターン開始、カード使用、ターン終了、敵行動という決まった順番（状態機械）で進めます。

ターンフェーズと行動資源
→ 03

山札・手札・捨て札

カードを山札、手札、捨て札、廃棄へ移し、山札が空なら混ぜ直します。

シャッフルとカード移動
→ 04

敵の予告と状態効果

敵の次の行動を表示し、毒や弱体、筋力などターンをまたいで続く効果（状態効果）を決まったタイミングで更新します。

予告行動と効果の持続期間（状態効果）
→ 05

カード報酬と強化

戦闘後に候補から1枚を選び、削除や強化でデッキを調整します。

選択報酬・レア度・デッキ編集
→ 06

分岐マップの旅

戦闘、休憩、店、イベントを持つ分岐マップを生成します。

ノード生成・遭遇表・進行保存
→ 07

カードが飛んで命中する

選んだカードを手札から敵へ動かし、接触フレームで効果と演出を発生させます。

カード解決を3つの時間へ分ける
→ 08

5階層のラン設計

危険度と報酬の違う2つの道を各階で選び、5階のボスまで進む流れをデータにします。

戦闘の外側に判断と成長を置く
→ 09

Ebi Ascent

カード、敵、イベントをデータ駆動で統合し、繰り返し遊べる1ランを完成させます。
