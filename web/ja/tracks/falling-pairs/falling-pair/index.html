DEEP DIVE / PIVOT AND CHILD

2個を別々に動かさず、軸から子の場所を求める。

落下中に保存する位置は軸のx, yです。子はいつもchild = pivot + (0,-1)。左右へ動くときも下へ落ちるときも、軸の候補位置から2マス両方が空いているかを調べます。

1

軸から2マスを作る

軸は(pivotX, pivotY)、子は(pivotX, pivotY-1)。組の距離はいつも1マスなので、位置がずれません。

childX, childY := pivotX, pivotY-1
2

次の場所を先に調べる

左右移動も自然落下も、候補の軸と子が盤面内で空かをblockedで確認します。通ったときだけ座標を変えます。

if !blocked(x+dx, y+dy) { x += dx; y += dy }
3

下へ行けなければ固定

次の1マスが塞がっていたら、現在の軸と子の色をboardの2マスへ書き、新しい組を入口に出します。

board[y][x] = pivot; board[y-1][x] = child

IN THE REAL GAME

自然落下も入力も同じmoveを使う

Updateのタイマーが40になったときも、DOWNを押したときもmove(0,1)へ渡します。失敗したらその場所で固定します。

func (g *game) move(dx, dy int) bool {
    if g.blocked(g.x+dx, g.y+dy) {
        return false
    }
    g.x += dx
    g.y += dy
    return true
}

if !g.move(0, 1) {
    g.lockPair()
}

WHY RELATIVE?

組がちぎれない

2個分の位置を別々に更新しないため、片方だけ動かし忘れるバグを防げます。

WHY CHECK BOTH?

子も壁を通れない

軸だけ空いていても子が埋まる場所には移動させません。1組全体が一つの候補です。
