DEEP DIVE / BREADTH-FIRST FLOOD FILL

1個から始め、同じ色の道だけを上下左右へ広げる。

盤面へペアを固定した直後、まだ調べていない色マスを入口にします。入口をキューへ入れ、取り出したマスの上下左右に同色があれば追加します。行った場所をvisitedへ記録すれば、隣同士が互いを何度も追加する無限往復を防げます。

1

入口をキューへ入れる

色マスを1個選び、待ち列へ入れた瞬間にvisitedをtrueにします。取り出す時まで待つと、別の隣から同じマスが二重に追加されます。

queue := []Point{start}; visited[start] = true
2

上下左右だけを見る

斜めは仲間にしません。右・左・下・上の4つを順番に調べ、盤面内で同じ色ならキューの最後へ足します。

directions := []Point{{1,0},{-1,0},{0,1},{0,-1}}
3

集め終わってから数える

キューが空なら、その色の島を全部調べ終えました。groupの長さが4以上なら、消去候補として全マスをmarkedへ入れます。

if len(group) >= 4 { mark(group) }

IN THE REAL GAME

キューが空になるまで同色を広げる

visitedは盤面全体で共有するため、一度見つけたグループを別の入口からもう一度数えません。ペア配置の良し悪しはgroupの長さとして返ります。

for len(queue) > 0 {
    current := queue[0]
    queue = queue[1:]
    group = append(group, current)
    for _, d := range directions {
        next := Point{current.x+d.x, current.y+d.y}
        if inside(next) && !visited[next] &&
           board[next.y][next.x] == kind {
            visited[next] = true
            queue = append(queue, next)
        }
    }
}

WHY MARK ON ADD?

同じマスを二重予約しない

キューへ入れた時点で予約済みにすると、待っている間も別の道から追加されません。

WHY VISUALIZE?

探索の順番を確かめられる

黄色い番号を出すと、斜めへ飛ばず、同色だけをたどることを盤面で確認できます。
