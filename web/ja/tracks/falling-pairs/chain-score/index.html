状態を輪につなぐ

Updateは待ち時間ごとに1段だけ進めます。落下後にsearchingへ戻す一行が連鎖の中心です。

switch phase {
case searching:
    marked = findGroups(board)
    if len(marked) == 0 { finishChain() }
    chain++
    score += len(marked) * 10 * chain
    phase = clearing
case clearing:
    clear(marked)
    phase = falling
case falling:
    compact(&board)
    phase = searching // もう一度探す
}

なぜ状態を分けるの？

消去と落下を同時に書くより、途中を画面で見せやすく、バグが起きた段階も見つけやすいからです。

得点はいつ足す？

グループを見つけた瞬間です。消した数 × 10 × 連鎖数なら、後の連鎖ほど価値が上がります。
