★★★★☆ / PLAYABLE

味方に当てる友情効果

味方へ触れた瞬間をイベントに変え、能力データからレーザー・回復・衝撃波を選んで発動します。

このステップで学ぶこと接触イベントと能力データ

PLAYABLE / WEBASSEMBLY

3種類の味方を使い分け、サンゴの敵を倒そう。

行きたい方向を長押し・タップ。PCでは矢印またはWASD。レーザーは近い敵、回復はHP、衝撃波は近くの敵全員へ働きます。50秒以内に全滅でクリアです。

● LIVE / GO + WASMSTEP 05

DEEP DIVE / CONTACT EVENT

「触れている」から「触れた瞬間」を取り出す。

円の重なりを毎フレーム調べるだけだと、くっついている間に60回も能力が出ます。今の接触と前の接触を比べ、今は触れていて、前は離れていたときだけイベントを作ります。そのイベントが味方の能力データを読みます。

1

今の接触を調べる

中心どうしの距離が二つの半径の合計以下なら接触です。画面の円と同じ簡単な円判定を使います。

touching := distance <= playerR + allyR
2

開始した瞬間だけイベント

touchingがtrueで、前回のwasTouchingがfalseなら新しい接触です。最後に今の値を次回用へ保存します。

if touching && !wasTouching { activate() }
3

能力データで処理を選ぶ

味方は名前、種類、強さを持ちます。共通のactivateが種類を見て、レーザー・回復・衝撃波へ分けます。

Ability{Name: "HEAL", Kind: Heal, Power: 2}

TRY IT / EVENT QUEUE

接触イベントを一つずつ処理しよう

接触が始まったらイベントを待ち列へ追加し、ゲームルールが順番に実行します。効果が同時に起きても順序を保てます。

待っている効果0
発動する順番—

IN THE REAL GAME

接触の入口は共通、結果はデータで分岐

味方ごとに当たり判定を作り直しません。同じ入口から能力番号を渡すので、新能力を足しても移動や接触コードは変わりません。

now := circlesTouch(player, ally)
if now && !ally.wasTouching && ally.cooldown == 0 {
    activate(abilities[ally.ability])
    ally.cooldown = 75
}
ally.wasTouching = now

WHY AN EDGE?

一回の接触を一回にする

接触開始だけを使えば、円が重なったままでも効果は連打されません。

WHY DATA?

味方を増やしやすい

位置や動きと能力内容を分けると、同じ接触ルールへ新しい効果を差し込めます。
