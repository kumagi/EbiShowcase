DEEP DIVE / MOVEMENT & AABB

どうしてカゴに
入ったとわかるの？

星もカゴも、見えない四角形（バウンディングボックス）だと考えます。2つの四角形が重なっていればキャッチ成功。このやり方を AABB（軸に平行な箱どうしの判定）と呼びます。

1

移動

星は毎フレーム Y に速さを足して落ちます。カゴはキーやポインタで X だけ動かします。

star.y += speed
2

たくさんの星

星は1個ではなくスライス（リスト）で持ちます。一定フレームごとに新しい星を追加します。

stars []star
3

重なり

左右と上下、どちらも食い込んでいればヒット。1辺でも離れていたらはずれです。

overlaps(a, b)

TRY IT / RECTANGLE OVERLAP

星を落として、重なりを見よう

「1フレーム進める」で星が落ちます。カゴの四角形と重なった瞬間、判定が YES になります。

丸く見えても、当たり判定は四角形で十分なことがよくあります。まずは箱で考え、必要ならあとから精密にします。

星の Y80毎フレーム下落
重なりまだAABB
ルール4辺左右上下すべて

THE OVERLAP RULE

左 < 相手の右 かつ 右 > 相手の左そして上 < 相手の下 かつ 下 > 相手の上

たてとよこの両方が重なって初めてヒットです。どちらか一方だけ重なっても、すれ違いです。

IN THE REAL GAME

リストを1個ずつ更新

落ちた星、キャッチした星、画面外へ消えた星を分けます。残すものだけ次のリストへ入れ直すと、管理がすっきりします。

for _, s := range g.stars {
    s.y += s.speed
    if overlaps(s, g.basket) {
        g.score++
        continue // キャッチしたので残さない
    }
    if s.y > height {
        g.lives--
        continue
    }
    next = append(next, s)
}
g.stars = next

WITHOUT LISTS

星が1個だけ

変数を星の数だけ用意すると、すぐに限界が来ます。リストなら何個でも同じルールで動けます。

WITH AABB

見た目より先に箱

絵がきれいでなくても、四角形の重なりさえ正しければ「キャッチした感じ」は出せます。

YOUR FIRST TUNING

速さと出現間隔を変えよう

星の speed や出現間隔 frame%45 を変えると難易度が変わります。カゴの幅を広げるのもやさしい調整です。
