Space Shooter

弾、敵、星、プレイヤーのように「種類の違うものがたくさん」現れるゲームでは、それぞれをスライスで管理し、不要になったら後ろから消していく「エンティティ管理」がポイントです。

このゲームで学ぶことエンティティ管理

PLAYABLE / WEBASSEMBLY

全ウェーブを生き残れ。

矢印キーまたはドラッグで自機を動かし、スペース／タッチで弾を発射します。ウェーブをすべて倒すと次のウェーブが来ます。ライフが0になるとゲームオーバーです。

● LIVE / GO + WASMLEVEL 08

DEEP DIVE / ENTITY MANAGEMENT

弾が増えても
コードが増えないのはなぜ？

答えはエンティティ管理です。「同じ種類のものを1つのスライスにまとめ、毎フレーム同じ処理をかける」という考え方です。弾が1発でも100発でも、ループは1つ。不要になったら後ろから安全に削除します。

1

種類ごとにスライス

プレイヤーの弾は []body、敵弾は []body、敵は []enemy、星は []star。種類が違っても、まとめ方は同じです。

bullets []body
2

毎フレーム一括更新

スライスを range でループし、全員を動かします。特定の1発だけ別扱いにする必要はありません。

for i := range g.bullets
3

後ろから削除

画面外に出た弾や倒した敵は削除します。後ろから削除すると、添え字がずれないので安全です。

bullets[:i], bullets[i+1:]

THE DELETE PATTERN

for i := len(g.bullets) - 1; i >= 0; i--条件が合えばg.bullets = append(g.bullets[:i], g.bullets[i+1:]...)

後ろから（インデックスを大きい方から）見ていくのがポイントです。要素を削除してもそれより前の添え字は変わらないので、削除漏れが起きません。

IN THE REAL GAME

弾の当たり判定と
削除を同時にやる

自機の弾と敵を総当たりで調べます。当たったら敵を削除し、弾も削除します。後ろからループしているので添え字がずれません。

for bi := len(g.bullets) - 1; bi >= 0; bi-- {
    hit := false
    for ei := len(g.enemies) - 1; ei >= 0; ei-- {
        if overlaps(g.bullets[bi], g.enemies[ei].body) {
            // 敵を削除
            g.enemies = append(g.enemies[:ei], g.enemies[ei+1:]...)
            g.score += 100
            hit = true
            break
        }
    }
    // 当たった弾か画面外の弾を削除
    if hit || g.bullets[bi].y < -20 {
        g.bullets = append(g.bullets[:bi], g.bullets[bi+1:]...)
    }
}

WITHOUT SLICES

変数が爆発する

弾を bullet1, bullet2, … と変数で持つと、最大数をあらかじめ決めなければなりません。数が増えるたびにコードを書き直す必要があります。

WITH ENTITY SLICES

数はデータ、ルールは1つ

スライスは必要なだけ伸び縮みします。ウェーブが増えて敵の数が変わっても、更新・描画のループは変わりません。

YOUR FIRST TUNING

数字を変えて実験しよう

spawnWave の count を増やすと敵が増えます。自機の弾の vy: -10 を変えると弾速が変わります。shootCooldown = 9 を小さくすると連射が速くなります。
