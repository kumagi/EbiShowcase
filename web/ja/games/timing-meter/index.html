どうして線は、
行ったり来たりするの？

ゲームはパラパラ漫画です。1コマごとに 位置 = 位置 + 速さ をくり返します。端に着いたら速さの符号をひっくり返すと、行ったり来たりになります。止めるときは「動かない状態」へ切り替えます。

1

フレーム

画面の1コマ。Ebitengineはだいたい毎秒60回 Update を呼びます。少しずつ足すので、なめらかに見えます。

marker += speed
2

反射

左端や右端へ着いたら speed = -speed。向きだけ変えて、同じ速さで戻ります。

speed = -speed
3

状態

動いているか、止まっているか。止まっているあいだは位置を進めず、中央との距離だけ採点します。

stopped bool

TRY IT / 1 FRAME AT A TIME

1コマずつ動かして、止めてみよう

「1フレーム進める」で線が動きます。端で向きが変わります。「止める」を押すと中央との距離で得点します。

THE TWO-MODE RULE

動くとき: marker += speed止まるときdistance = |marker - center|

同じボタンでも、状態によって意味が変わります。動いていれば「止める」、止まっていれば「次のラウンドへ」。これが状態機械の入口です。

IN THE REAL GAME

状態で処理を分ける

入力を受けたあとに「いま止まっているか？」を見ます。動いているときだけ位置を進め、端で向きを反転します。

func (g *game) Update() error {
    if pressed() {
        if g.stopped {
            g.stopped = false // 次のラウンド
        } else {
            g.stopped = true
            g.score += scoreFor(g.marker)
        }
    }
    if g.stopped {
        return nil
    }
    g.marker += g.speed
    if g.marker < 45 || g.marker > width-45 {
        g.speed = -g.speed
    }
    return nil
}

WITHOUT STATE

いつも同じ反応

ボタンを押すたびに同じ処理だけだと、「止める」と「再開する」を分けられません。

WITH STATE

今のモードで意味が変わる

stopped ひとつで、入力の意味と更新の有無を切り替えられます。
