DEEP DIVE / GRID AND HISTORY

ヘビの体は、
どうやって動くの？

答えは位置の履歴（りれき）です。ヘビの体は「通ってきたマス座標のリスト」に過ぎません。毎ステップ、新しい頭をリストの先頭に追加し、エサを食べていなければ末尾を1個消す。たったそれだけで、あのくねくねした体が生まれます。

スライスが体

体は []point（マス座標のスライス）です。先頭 [0] が頭、末尾が尻尾。長さがそのまま体の長さになります。

body []point

追加と削除

新しい頭を先頭に追加し、エサを食べていなければ末尾を1個切り捨てる。この2行でヘビの移動が完成します。

body[:len(body)-1]

TRY IT / GRID HISTORY

頭を足して、しっぽを消そう

「進む」は新しい頭を先頭に足して末尾を削ります。「食べる」は末尾を残すので体が伸びます。

体はマス座標のリストです。

(2,2) (1,2) (0,2)
長さ3body の要素数
履歴(2,2) (1,2) (0,2)頭 → しっぽ
ルール[]pointappend + trim

THE TWO-LINE MOVE

body = append([]point{head}, body...)そしてbody = body[:len(body)-1]

1行目で新しい頭を先頭に追加します。2行目で古い尻尾を取り除きます。エサを食べたときは2行目をスキップするだけで体が1マス伸びます。

IN THE REAL GAME

Updateの中で
体を動かす

毎フレームではなく、一定フレームに1回だけ体を動かすことで、スコアに応じてヘビがだんだん速くなります。

func (g *game) Update() error {
    g.frame++
    // スコアが上がると待ちフレームが減り、速くなる
    if g.frame%max(4, 10-g.score/3) != 0 {
        return nil
    }
    g.dir = g.next
    head := point{g.body[0].x + g.dir.x, g.body[0].y + g.dir.y}

    // 壁・自分との衝突判定
    if outOfBounds(head) || selfCollision(head, g.body) {
        g.over = true
        return nil
    }

    // 頭を追加、エサを食べていなければ尻尾を削除
    g.body = append([]point{head}, g.body...)
    if head == g.food {
        g.score++
        g.placeFood()
    } else {
        g.body = g.body[:len(g.body)-1]
    }
    return nil
}

WITHOUT A GRID

ピクセル管理の難しさ

ピクセル単位で体を管理すると、体がなめらかに動く分、「ちゃんとマスに揃っているか」の判定が複雑になります。グリッドなら整数の一致を確認するだけです。

WITH A SLICE HISTORY

体の長さはリストの長さ

体が何マスあるか知りたければ len(g.body) を見るだけ。体のどのマスにいるかも添え字でわかります。配列が体そのものです。

数字を変えて実験しよう
