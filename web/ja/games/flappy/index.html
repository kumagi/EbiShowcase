DEEP DIVE / ACCELERATION

どうして鳥は、
だんだん落ちるの？

答えは加速度（かそくど）です。むずかしそうな名前ですが、「速さを少しずつ変える量」と考えれば大丈夫。坂道を転がるボールが、だんだん速くなるのと同じです。

1

位置

鳥が「いま、どこにいるか」。画面の上が0で、下へ行くほど数字が大きくなります。

birdY = 320
2

速度

1回の更新で、どちらへどれだけ動くか。マイナスなら上へ、プラスなら下へ進みます。

velocity = -7.4
3

加速度

速度を毎回どれだけ変えるか。重力として +0.42 を足し続けると、上昇が止まり、やがて落下します。

gravity = +0.42

TRY IT / 1 FRAME AT A TIME

1コマずつ動かしてみよう

ゲームはパラパラ漫画のように、少しずつ計算して描き直します。ボタンを押すたび、重力が速度へ足され、その速度だけ鳥の位置が変わります。

「はばたく」は、上向きの速度 -7.4 をセットしてから1フレーム進めます。

THE TWO-LINE RULE

速度 = 速度 + 重力つぎに位置 = 位置 + 速度

順番が大切です。まず重力で速度を変え、次にその速度で位置を変えます。毎フレームくり返すだけで、自然なジャンプの曲線が生まれます。

IN THE REAL GAME

Ebitengineでは
Updateに書く

Updateはゲームの状態を進める場所です。入力を調べ、速度を変え、位置を動かします。Drawは計算後の位置に鳥を描きます。

const gravity = 0.42
const flapV   = -7.4

func (g *game) Update() error {
    // 押した瞬間、上向きの速度にする
    if justPressed() {
        g.velocity = flapV
    }

    // 1. 重力で速度を変える
    g.velocity += gravity

    // 2. 速度で位置を変える
    g.birdY += g.velocity
    return nil
}

WITHOUT ACCELERATION

ずっと同じ速さ

鳥が直線で上がり、直線で落ちるため、機械のような不自然な動きになります。

WITH ACCELERATION

なめらかな弧を描く

上向きの速度が弱まり、頂点で0に近づき、その後は下向きに速くなります。

YOUR FIRST TUNING

数字を変えて実験しよう

gravityを大きくすると重い鳥に、flapVをより小さなマイナスにすると力強い羽ばたきになります。ゲーム作りは、この「ちょうどいい」を探す作業です。
