ステージは、
どうやって作るの？

答えはタイルマップです。ステージを文字列の配列で書き、# が壁、@ がプレイヤー、$ が箱、. がゴールを表します。起動時にこの文字列を1文字ずつ読み、「どのタイル座標に何があるか」をデータへ変換します。絵は後からタイル座標を元に描けばよいので、ステージ設計とコードが完全に分離できます。

1

文字列でステージ

ステージはGoの文字列スライス。# や $ を読んで壁・箱のデータを作ります。見た目がそのままマップになるので設計しやすいです。

var level = []string{…}
2

mapで壁を管理

壁の集合は map[point]bool。「このタイルは壁か？」を g.walls[p] の1行で確認できます。スライスを検索するより高速です。

walls map[point]bool
3

Undoは履歴スタック

動く前にプレイヤーと箱の位置をスナップショットとして保存します。Undoで最後の状態を取り出して復元します。

history []snapshot

THE PARSE RECIPE

外側のループが行（y座標）、内側のループが列（x座標）を与えます。文字 c で壁・箱・ゴールを振り分けるだけで、ステージデータが完成します。

IN THE REAL GAME

moveでタイル座標を
1マス動かす

移動先が壁なら無視、箱があれば箱の向こうをチェックしてから押す。動く前に履歴へ保存するので、Undoが常に機能します。

func (g *game) move(d point) {
    next := point{g.player.x + d.x, g.player.y + d.y}
    if g.walls[next] {
        return // 壁は通れない
    }
    boxIndex := g.boxAt(next)
    if boxIndex >= 0 {
        beyond := point{next.x + d.x, next.y + d.y}
        if g.walls[beyond] || g.boxAt(beyond) >= 0 {
            return // 箱を押せない
        }
        g.save() // 動く前に履歴へ保存
        g.boxes[boxIndex] = beyond
    } else {
        g.save()
    }
    g.player = next
    g.moves++
}
