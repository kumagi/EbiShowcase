DEEP DIVE / INPUT & COORDINATES

どうして、
押した場所がわかるの？

画面はマス目のないグラフ用紙だと考えます。左上が (0, 0)。右へ行くほど X が、下へ行くほど Y が大きくなります。タッチした点と丸の中心の距離が、半径より短ければ「当たった！」です。

1

座標

画面のどこかを、2つの数で表します。丸の中心も、タッチした場所も、同じ X と Y で書けます。

circleX, circleY
2

距離

2点のあいだの長さです。直角三角形のたてとよこから求めます。平方根が面倒なら、二乗のまま比べても同じです。

hypot(dx, dy)
3

当たり判定

距離が半径以下ならヒット。得点して、次の丸を別の場所へ移します。

dist <= radius

TRY IT / HIT TEST

画面を押して、距離を見てみよう

右のボードをタップすると、押した点と丸の中心の差が出ます。距離が 70 以下ならヒットです。

本物のゲームでも、マウスとタッチを同じ「押した座標」にそろえてから判定します。

dx—左右の差
dy—上下の差
距離—タッチしてね

THE ONE-LINE RULE

dx = touchX - circleXそしてdy = touchY - circleY

あとは dx*dx + dy*dy が radius*radius より小さければヒット。平方根を毎回計算しなくても、二乗どうしなら同じ結果になります。

IN THE REAL GAME

Ebitengineでは
Updateに書く

押された瞬間だけ座標を読み、距離を調べます。Drawは丸とスコアを描くだけ。計算と描画を分けると、あとから直しやすくなります。

func (g *game) Update() error {
    px, py, pressed := pressedPosition()
    if !pressed {
        return nil
    }

    // 押した点と中心の距離
    dist := math.Hypot(float64(px)-g.x, float64(py)-g.y)
    if dist <= g.radius {
        g.score++
        g.moveTarget()
    }
    return nil
}

WITHOUT COORDINATES

「なんとなく」では作れない

「だいたいこのへん」ではプログラムは動きません。位置は必ず数字で表します。

WITH DISTANCE CHECK

丸もタッチも同じものさし

見た目が丸でも、判定は中心と半径の計算です。これでどの画面サイズでも同じルールになります。

YOUR FIRST TUNING

数字を変えて実験しよう

radius を小さくするとむずかしく、framesLeft を伸ばすとゆっくり遊べます。ゲーム作りは「ちょうどいい」を探す作業です。
