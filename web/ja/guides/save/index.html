SAVEEN

SYSTEM GUIDE / SAVE

続きから遊べるゲームを、
こわれにくく作る。

セーブは「今の状態をJSONにして置く」だけではありません。版番号を付け、自動保存の場所を決め、読めないデータから安全に戻り、場面をまたいで状態を渡す設計です。

FOUR PROMISES

保存の4つの約束

01

version を最初に書く

ルールを変えた未来の自分が、古いデータを見分けられます。知らない版は無理に読むより新しいゲームを始めます。

02

大事な変化の直後に保存

宝箱を開けた、部屋を出た、設定を変えた。その直後にAutosaveします。毎フレーム保存はしません。

03

読めなければ安全な初期状態へ

JSONが壊れていたら、遊べなくする代わりに初期状態へ戻し、「セーブを読み直せなかった」と知らせます。

04

画面は状態を持ち運ばない

sceneは見せ方、save modelは事実です。次の部屋へはモデルを渡し、新しいsceneを作ります。

savegame.Autosave(store, "adventure.slot", state, time.Now())version + updated_at + data

復旧の順番

Load → Decode → versionを確認 → payloadを読む → 新しいsceneを作る。 どこかで失敗したら、初期状態に戻して遊べるままにします。`internal/savegame` はWASMではlocalStorage、ローカルテストではメモリストアを同じ形で使えます。
