STEP 00

命令を打ち込む場所を開く

ブラウザではなく、パソコンに命令を出す画面を使います。名前は OS で違います。

Windows

  1. キーボードの Windows キーを押す
  2. 「powershell」と入力する
  3. 「Windows PowerShell」または「ターミナル」を開く

macOS

  1. Command + Space キーを同時に押す
  2. 「ターミナル」と入力する
  3. 「ターミナル」アプリを開く

文字を入力する画面が開き、カーソルが点滅していれば OK。Windows は黒いことが多く、Mac は白いこともありますが、役割は同じです。これからここにコマンドをコピーして貼り付けます。

用語メモ：Windows では「PowerShell」、Mac では「ターミナル」と呼ぶことが多いです。どちらも同じ役割——キーボードでパソコンに命令を出す黒い（または白い）窓です。「ターミナル＝Mac専用」ではありません。Windows 11 の「ターミナル」アプリでも PowerShell が使えます。このガイドでは Windows 欄を PowerShell、Mac 欄をターミナルと書きます。

STEP 01

Go をインストールする

Ebitengine は Go で書かれたゲームエンジンです。先に Go 本体が必要です。このリポジトリは Go 1.25 以降を使います。

Windows

  1. ブラウザで https://go.dev/dl/ を開く
  2. 「Microsoft Windows」の .msi を選ぶ（だいたい一番上の安定版）
  3. ダウンロードしたファイルを開き、画面の指示どおり「次へ」で進む
  4. 終わったら PowerShell（さっき開いた命令の窓）をいったん閉じて、もう一度開き直す（パスを読み直すため）

macOS

  1. ブラウザで https://go.dev/dl/ を開く
  2. 自分の Mac に合うパッケージを選ぶ。Apple Silicon（M1/M2/M3…）なら ARM64、Intel なら x86-64 の .pkg
  3. ダウンロードした .pkg を開き、画面の指示どおりインストールする
  4. 終わったらターミナルをいったん閉じて、もう一度開き直す

学校や会社の PC では管理者パスワードが必要なことがあります。入れられないときは保護者や先生に相談してください。

STEP 02

Mac だけ：C コンパイラを入れる

Ebitengine は中で C の部品も使います。Windows の学校 PC はこのステップを飛ばして大丈夫です。Mac だけ、無料の「コマンドラインツール」——プログラムを作るときに裏側で使う道具一式——を入れます。長い名前を覚える必要はありません。教室の PC が Windows なら、すぐ「3. 動作確認」へ進んでください。

Windows の人はこのステップを飛ばして「3. 動作確認」へ進んでください。

xcode-select --install
  1. ターミナルに上のコマンドを貼り付けて Enter
  2. ダイアログが出たら「インストール」を選ぶ（数分〜十数分かかることがあります）
  3. すでに入っている場合は「command line tools are already installed」のようなメッセージで大丈夫です

STEP 03

本当に入ったか確かめる

ターミナルに次を打ち、Enter を押します。

go version

例: go version go1.25.0 windows/amd64 のように、go1.25 以上の数字が出れば成功です。

「コマンドが見つかりません」「go は認識されていません」と出たら：インストール後にターミナルを開き直したか確認し、それでもだめなら PC を再起動してからもう一度。

さらに安心したい人へ（任意）

公式の回転サンプルを一度走らせると、画面が出るところまでまとめて確認できます。初回はダウンロードに少し時間がかかります。

go run github.com/hajimehoshi/ebiten/v2/examples/rotate@latest

ゴファーの絵がくるくる回る窓が開けば、環境はバッチリです。閉じたら次へ進みましょう。

STEP 04

ゲーム用のフォルダを作る

デスクトップでもドキュメントでも構いません。ここでは例として home の直下に ebi-empty を作ります。

  1. mkdir ebi-emptyフォルダを作る
  2. cd ebi-emptyその中へ入る
  3. go mod init example.com/ebi-emptyこのフォルダの目次 go.mod を作る

go.mod は、このゲームの名前（モジュール名）と、外から借りる部品（Ebitengine など）を Go が管理するための『プロジェクトの目次ノート』です。モジュール名は、コードの部品をどの名前で呼ぶかを決める住所のようなもの。これがあることで、Go は必要な外部パッケージを同じ組み合わせで用意できます。小さな go.mod ができていれば OK。名前は後から変えられます。

STEP 05

main.go を書く（何もしない窓）

テキストエディタ（メモ帳、メモ帳++、VS Code、Cursor など）で、いまのフォルダに main.go という名前のファイルを作り、次をそのままコピーします。

main.go 
package main

import (
	"image/color"
	"log"

	"github.com/hajimehoshi/ebiten/v2"
)

// Game は Ebitengine が求める3つのメソッドを持つ箱です。
type Game struct{}

// Update は入力と数字を進める場所。状態を書き換えるならここです。
func (g *Game) Update() error {
	return nil
}

// Draw は今の状態を画面へ写すだけ。ゲーム状態は書き換えません。
// 渡された screen へピクセルを書くことだけが、意図した出力です。
func (g *Game) Draw(screen *ebiten.Image) {
	screen.Fill(color.RGBA{20, 28, 48, 255})
}

// Layout はゲーム内部の解像度（幅×高さ）を返します。
func (g *Game) Layout(outsideWidth, outsideHeight int) (int, int) {
	return 640, 480
}

func main() {
	ebiten.SetWindowSize(640, 480)
	ebiten.SetWindowTitle("Empty Window")
	if err := ebiten.RunGame(&Game{}); err != nil {
		log.Fatal(err)
	}
}
Update毎フレームの入力・数字・状態更新担当。今は空。
Draw毎フレームの投影担当。色を一塗りし、状態は変えない。
Layoutゲーム内部の幅と高さ。
RunGameこのくり返しを起動するスイッチ。

STEP 06

依存関係を入れて、実行する

ターミナルのカレントディレクトリが ebi-empty のまま、次を順番に打ちます。

  1. go mod tidyEbitengine などを自動で取ってくる
  2. go run .プログラムを動かす

タイトルが Empty Window の、暗い青い長方形の窓が開けば成功です。閉じるボタンで終了できます。中身はまだ何も動きません——それでも「ゲームの土台」はもう動いています。

次は、この土台の上で Update と Draw を1行ずつ書き、最初の RULE まで進む Build Track へどうぞ。clone は必要ありません。

PowerShell とターミナル、どっち？

Windows なら「PowerShell」で大丈夫。Mac なら「ターミナル」。どちらも命令を打ち込む窓です。Windows の「ターミナル」アプリを開いても、中身が PowerShell なら問題ありません。

go: command not found / 認識されない

① Go を入れたあと、命令の窓を完全に閉じて開き直したか確認。② Windows ならスタートメニューに「Go」があるか確認。③ それでもだめなら PC を再起動してもう一度 go version。

xcrun: error: invalid active developer path…（Mac）

ステップ 02 の xcode-select --install をもう一度実行する。途中で止まったら再起動してから再実行。

go mod tidy や go run でネットエラー

インターネット接続と、会社・学校のプロキシ設定を確認。自宅の回線で試すと通ることがあります。

窓が一瞬で消える / パニック

命令の窓に赤いエラー全文が出ています。その文を検索するか、このページ末尾のフィードバックへ貼ってください。

WSL（Windows の Linux）を使っている

公式どおり、実行時に GOOS=windows が必要になることがあります。はじめてなら通常の PowerShell での手順を推奨します。
