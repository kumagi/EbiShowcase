SYSTEM GUIDE / PROJECT STRUCTURE

ゲームが大きくなったら、
役割ごとに部屋を分けよう。

`cmd` は起動する場所、`internal` はゲームの仕組み、`assets` は画像・音・マップの置き場です。

examples/project-structure/
├── cmd/reef-run/main.go
├── internal/game/game.go
├── internal/rules/score.go
└── assets/入口 / 表示 / 純粋ルール / 素材

本当に分けたサンプル

Spaceで宝石を取る最小ゲームを、実際にこの構成へ分割しました。`rules.Collect` はEbitengineをimportしないので `go test ./examples/project-structure/internal/rules` で画面を開かずに確かめられます。

まず `cmd` は薄く保ちます。次に `internal/game` が入力・更新・描画をつなぎます。最後に計算は `internal/rules` に移します。素材だけは `assets` に置き、ライセンスも隣に残します。
