STEP 01 / 04

画面からルールを取り出す

クリック位置と丸の当たり判定を、Ebitengineを知らない小さな関数へ移します。入力と描画はゲーム側、答えを出す計算は純粋ロジック側です。

入力・状態→純粋関数→次の状態・答え

なぜこれで安心できる？

同じ数字を渡せば、いつでも同じ答えが返る関数なら、画面を開かずに何度でも確かめられます。

先に、確かめたい例を並べる

中心(100, 80)true
ふち(130, 80)true
1px 外(131, 80)false

BEFORE / Updateに全部ある

func (g *game) Update() error {
  px, py := ebiten.CursorPosition()
  dx := float64(px) - g.circleX
  dy := float64(py) - g.circleY
  if math.Hypot(dx, dy) <= g.radius {
    g.score++
  }
  return nil
}

AFTER / ルールを分ける

// Ebitengineをimportしないファイル
func PointInCircle(px, py, cx, cy, radius float64) bool {
  dx := px - cx
  dy := py - cy
  return math.Hypot(dx, dy) <= radius
}

// Updateは入力と状態更新だけ
if lessonlogic.PointInCircle(px, py, g.circleX, g.circleY, g.radius) {
  g.score++
}

TEST / 期待する答えを書く

_test.go

同じ数字を渡せば、いつでも同じ答えが返る関数なら、画面を開かずに何度でも確かめられます。

func TestPointInCircle(t *testing.T) {
  got := PointInCircle(130, 80, 100, 80, 30)
  if !got {
    t.Fatal("円のふちも当たりにしたい")
  }
}
