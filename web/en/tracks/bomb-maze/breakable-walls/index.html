DEEP DIVE

A cell is state, not just a picture

Breaking a wall does not merely erase its picture. It changes a value in a two-dimensional array from softWall to floor. From the next frame onward, movement and drawing both read the new floor value.

1

GO + EBITENGINE

Change the data, then reveal the item

The same coordinate produces the same result on every retry, which makes the code easier to trace. The real game uses a coordinate hash instead of global randomness.

if board[p.y][p.x] == softWall {
    board[p.y][p.x] = floor
    destroyed++
    if drop := deterministicDrop(p); drop != noItem {
        items[p] = drop
    }
    break // do not pass through this wall
}

WHY IT WORKS

Collision and drawing stay together

Movement in Update and rendering in Draw read the same array, preventing a wall that looks gone but still blocks the player.
