BOMB MAZE · STEP 03

Breakable Walls and Items

When a blast hits a wooden wall, change that cell's data from wall to floor. Collect the items that sometimes appear and break ten walls.

DIFFICULTY ★★★☆☆PLAYABLE

PLAY FIRST

Discover what is inside each wall

Move with arrows / WASD and place a bomb with Space / X. On phones and tablets, tap the bottom buttons. Touching a blast or reaching 75 seconds is a loss.

DEEP DIVE

A cell is state, not just a picture

Breaking a wall does not merely erase its picture. It changes a value in a two-dimensional array from softWall to floor. From the next frame onward, movement and drawing both read the new floor value.

01

The blast reaches a wall

Check cells along each arm. A hard wall stops immediately; a wooden wall receives the final flame.

02

Wall becomes floor

board[y][x] = floor turns a blocked cell into a cell the player can enter.

03

Choose the contents

A value made from the coordinates selects fire power, bomb capacity, speed, or nothing.

TRY IT / CELL STATE

Step one cell’s state change

Wood wall → blast → floor + item. That’s the order of writing array values. Become floor first so the player can walk onto the drop.

You’re not erasing art—you’re changing board[y][x].

GO + EBITENGINE

Change the data, then reveal the item

The same coordinate produces the same result on every retry, which makes the code easier to trace. The real game uses a coordinate hash instead of global randomness.

if board[p.y][p.x] == softWall {
    board[p.y][p.x] = floor
    destroyed++
    if drop := deterministicDrop(p); drop != noItem {
        items[p] = drop
    }
    break // do not pass through this wall
}

WHY IT WORKS

Collision and drawing stay together

Movement in Update and rendering in Draw read the same array, preventing a wall that looks gone but still blocks the player.

CHALLENGE

Change the drop rate

Change the remainder cases in deterministicDrop to make a harder version with fewer items. Predict how many of each type will appear before testing it.
