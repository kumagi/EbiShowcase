Bomb Chain Reactions

When a blast reaches another bomb, trigger it without waiting for its timer. Add triggered bombs to an event queue and use a processed mark so no bomb explodes twice.

DEEP DIVE

Build a line of bombs to trigger next

If A's blast reaches B and B's reaches C, new work keeps appearing. A queue stores that work and handles bombs in the order they were found.

GO + EBITENGINE

Propagate until the queue is empty

The game queues bomb indexes. A fresh processed map records which bombs have already triggered during this one reaction.

queue := append([]int(nil), starts...)
processed := map[int]bool{}
for len(queue) > 0 {
    index := queue[0]
    queue = queue[1:]
    if processed[index] {
        continue
    }
    processed[index] = true
    bombs[index].blast = blastCells(bombs[index].at)
    for other := range bombs {
        if contains(bombs[index].blast, bombs[other].at) {
            queue = append(queue, other)
        }
    }
}

Why not recurse immediately?

A queue exposes how much work remains and gives every event the same ordering rule. Even a long chain stays easy to inspect.

Why prevent duplicates?

If both A and C reach B, B might create its blast and score twice. The processed mark keeps the result stable.
