Cross-shaped Blast

Walk outward from a bomb in four directions and decide which cells join the blast. Each ray stops when it reaches maximum range or meets a hard wall.

DEEP DIVE

Extend four cell-checking rays

Do not test one large cross-shaped picture. From the center, inspect cells one, two, then three steps away in each direction. That lets every ray stop on the near side of a wall.

Add the center cell

GO + EBITENGINE

Use nested direction and distance loops

break stops only the inner distance loop. A wall to the right therefore does not prevent the next up, down, or left ray from being checked.

cells := []point{center}
for _, direction := range rays {
    for step := 1; step <= blastRange; step++ {
        next := point{
            center.x + direction.x*step,
            center.y + direction.y*step,
        }
        if hardWall(next) {
            break
        }
        cells = append(cells, next)
    }
}

Why store a cell slice?

Drawing, player damage, and beacon hits can all use the very same blast slice.

Why not pass through walls?

The loop stops before adding a hard-wall cell. Reordering those two actions would be enough to design a wall that is hit but still blocks cells beyond it.

Challenge

Change range to 2 or 4 and compare the preview cell count and how difficult it is to escape.
