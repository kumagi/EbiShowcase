DEEP DIVE / MATCHUP TABLE

Look up two types instead of adding more if-statements.

Make a 3×3 multiplier table with attack types as rows and defense types as columns. Multiply move power by the result, and one damage formula handles strong, normal, and weak matchups.

row = attack / column = defense
TIDE defFLAME defLEAF def
TIDE atk120.5
FLAME atk0.512
LEAF atk20.51

Example: Flame vs Leaf → row FLAME, column LEAF → 2

1

Number the types

Arrange TIDE=0, FLAME=1, and LEAF=2. These values can directly index a table row or column.

type element int
2

Attack row × defense column

For Tide attacking Flame, read table[Tide][Flame] and get 2.0. Reversing the index order changes the answer.

mult := matchup[attack][defense]
3

Reuse the table for counters

Use the enemy counter as attack row and your chosen stance as defense column. A strong attack can still invite a painful counter.

counterMult := matchup[counter][chosen]

IN THE REAL GAME

Keep the matchup table in one data block

The damage formula does not know the table's details; it only uses the returned multiplier. Balance changes touch table numbers only.

var matchup = [3][3]float64{
    {1,   2, 0.5}, // Tide attack
    {0.5, 1,   2}, // Flame attack
    {2, 0.5,   1}, // Leaf attack
}
mult := matchup[chosen][enemyDefense]
damage := int(basePower * mult)
