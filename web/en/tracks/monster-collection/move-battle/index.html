DEEP DIVE / MOVE DATA AND ORDER

Build an order between choosing a move and applying it.

A move is more than a name: it stores power, accuracy, remaining uses, and priority. Put both choices in an action queue, sort it, then remove one action at a time so messages and HP changes stay clear.

1

Compare move records

REEF BURST has power 42 but 70% accuracy. JET PINCH always hits for 18. Different uses make timing part of the choice.

moveData{power:42, accuracy:70, maxUses:3}
2

Sort two actions

Higher priority acts first. Ties compare speed 12 against 10. SHELL GUARD priority 3 therefore protects before an attack.

sort.SliceStable(actions, byPriorityThenSpeed)
3

Check accuracy before effect

A random number from 0 to 99 below accuracy means hit. Attacks lower HP, guard halves the next hit, and healing restores HP.

hit := rng.Intn(100) < move.accuracy

IN THE REAL GAME

Resolve only the queue head

Store both actions on selection. resolveNext removes only the first and computes its accuracy and effect. If the rival falls, skip the remaining action.

queue = []action{playerAction, rivalAction}
sortByPriorityThenSpeed(queue)

func resolveNext() {
    current := queue[0]
    queue = queue[1:]
    if rng.Intn(100) < current.move.accuracy {
        apply(current.move)
    }
}

WHY USES?

Make powerful moves a choice

Unlimited moves make one strongest attack solve everything. Uses create the question of spending now or saving.

WHY A QUEUE?

Avoid simultaneous feedback

Stored ordered actions let messages, animation, and HP changes appear one at a time.
