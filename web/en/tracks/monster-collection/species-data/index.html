DEEP DIVE / DEFINITION VS STATE

Separate what the guidebook says once from what each creature remembers.

Just as LEVEL 01 Update / Draw changed numbers and rendered them, Update changes instance HP and XP here. But the REEFLET species name and maximum HP match for every member. Separate shared definitions from changing state.

1

Species is a read-only table

The species table stores name, maximum HP, attack, and color once. Training never changes its maximum HP of 10.

speciesTable[0] = Species{"REEFLET", 10, 3}
2

Instances point with an ID

AQUA and BUBBLE both store speciesID: 0 and read one definition. The name is not copied, so there is one place to fix it.

Creature{nickname: "AQUA", speciesID: 0}
3

Store HP and XP per individual

Current HP and experience change during an adventure, so they belong to each creature. Same-species A can start at HP10 while B starts at HP6.

creature.hp -= 2; creature.xp += 3

IN THE REAL GAME

Read only the healing cap from species

Only the selected creature's hp changes. Its cap comes from the shared table through speciesID.

c := &creatures[selected]
definition := speciesTable[c.speciesID]

// Change only this instance's state.
c.hp = min(definition.maxHP, c.hp+3)
c.xp += 3
