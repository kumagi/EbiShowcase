DEEP DIVE / INTEGRATED GAME STATE

Turn six systems into one adventure flow.

A complete game is not a row of disconnected features. Region choice selects encounter data, battle results change capture chance, and capture success updates bestiary, EXP, party, and box together. modeMap and modeBattle separate screens while state transitions pass results forward.

1

Create an instance from a region

Each region's encounter table is a list of species IDs. Visit count selects the next ID, then species data supplies wild HP and type.

wildSpecies = encounterTables[region][visits[region]%2]
2

Commit capture as one transaction

Move power, accuracy, uses, and matchup reduce HP. Only capture success registers the dex, adds an instance, shares EXP, checks evolution, and returns to map together.

dex[id]=true; addMonster(); shareEXP(); mode=modeMap
3

Save only required state

Screen colors and buttons are not saved. Copy party species and EXP, box, bestiary, and region visits into saveData, then produce JSON bytes.

saveSlot, _ = json.Marshal(payload)

IN THE REAL GAME

Build a save structure from runtime data

Runtime monsters also have HP, but this lesson's save chooses species, EXP, bestiary, and region progress. A dedicated structure keeps the JSON contract stable.

payload := saveData{Dex: g.dex, Visits: g.visits}
for _, m := range g.party {
    payload.PartySpecies = append(payload.PartySpecies, m.speciesID)
    payload.PartyExp = append(payload.PartyExp, m.exp)
}
for _, m := range g.box {
    payload.BoxSpecies = append(payload.BoxSpecies, m.speciesID)
}
encoded, err := json.Marshal(payload)
if err == nil {
    saveSlot = append(saveSlot[:0], encoded...)
}

WHY MODES?

Do not mix input meanings

Key 1 means a region on the map and a move in battle. Checking mode first routes the same input safely.

WHY ONE CAPTURE TRANSACTION?

Prevent split data

The game cannot register a dex entry but forget to store the captured instance.

SHIP IT

Swap the save destination

Keep JSON creation, but design an adapter that sends the in-memory save slot to browser storage or a server.
