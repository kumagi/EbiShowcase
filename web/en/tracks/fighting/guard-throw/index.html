DEEP DIVE / GUARD & THROW

How do you build a
rock-paper-scissors relationship?

The triangle is: Guard stops a Strike; Throw breaks a Guard; Strike stuffs a Throw attempt. Encoding all three win conditions in a single wins(a, b) function means every possible pairing is handled in three readable lines. Same choice? Draw — no special case needed.

1

Integer constants with iota

Represent strike, guard, and throw as 0, 1, 2 using iota. Integers let you index a name table, pass choices as function arguments, and compare them in one line.

strike = iota // 0, guard // 1, throw // 2
2

The wins() function

Three OR conditions cover every winning pair. Putting all win logic in one place means you only need to edit one function when adding options or re-balancing.

wins(guard, strike) // guard beats strike
3

Enemy telegraph

The foe telegraphs with stance during windup—not the move name. Reading that window is the fighting-game skill this lesson trains.

message = "Enemy prepares " + name(g.enemy)

THE TRIANGLE RULE

GUARD beats STRIKE // blocks the attack BUT THROW beats GUARD // breaks the defense BUT STRIKE beats THROW // stuffs the grab attempt

Same choice = clash; no advantage. Because the triangle is cyclic, every option is always at risk — which is exactly what creates the reading game. No option is universally safe.

IN THE REAL GAME

Enemy picks → player sees → wins() decides

The enemy choice is stored in g.enemy and shown on screen. After the player responds, three branches handle draw, win, and loss. The score and life counters stay clean because all logic routes through wins().

const (
    strike = iota  // 0
    guard          // 1
    throw          // 2
)

func wins(a, b int) bool {
    return (a == strike && b == throw)  ||
           (a == throw  && b == guard)  ||
           (a == guard  && b == strike)
}

// resolve the round
if c == g.enemy {
    g.message = "Clash!"
} else if wins(c, g.enemy) {
    g.score++
    g.message = name(c) + " beats " + name(g.enemy)
} else {
    g.life--
    g.message = name(g.enemy) + " beats " + name(c)
}

WITHOUT A WINS TABLE

Nine branches, easy to forget one

Writing every pair with nested if-else produces nine cases. wins() collapses that to three readable lines — and a unit test covers the whole table in one loop.

WITH TELEGRAPH

Both reaction skill and prediction matter

Showing the enemy choice creates a reaction game. Hide it entirely and you get pure prediction. Showing it briefly creates a middle ground that suits most players.
