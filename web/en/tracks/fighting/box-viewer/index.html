DEEP DIVE / HITBOXES

How does the game know
when a punch lands?

Every character carries three invisible rectangles at all times. The blue push box keeps bodies from passing through each other. The green hurt box is the area that receives attacks. The red hit box exists only during active attack frames. A hit registers only when the red box overlaps the opponent's green hurt box.

1

Push Box

Blue boxes stop characters from overlapping. When the two push boxes intersect, the player's position is corrected back out — exactly like a solid wall.

if overlap(push1, push2) { p1 = p2 - 36 }
2

Hurt Box

The area that can receive an attack. Setting it slightly larger than the visible sprite creates a generous, satisfying hit feel with fewer "missed" attacks.

hurt := rect{p2 - 24, 500, 48, 105}
3

Hit Box

The attack area, created only during the active window. If it overlaps the opponent's hurt box, the hit is registered. A shorter active window makes the attack sharper and harder to time.

hit := rect{p1 + 15, 535, 90, 32}

AABB OVERLAP TEST

a.x < b.x+b.w // a's left is before b's right AND a.x+a.w > b.x // a's right is after b's left repeat on the Y axis a.y < b.y+b.h && a.y+a.h > b.y

All four conditions must be true simultaneously. This test is called the Axis-Aligned Bounding Box (AABB) check. It works for any pair of upright rectangles and is used throughout 2D game development — not just fighting games.

IN THE REAL GAME

Separate → Spawn box → Test overlap

Each frame: first resolve push-box overlap so bodies stay apart. Then, when the attack reaches the right frame, spawn a hit box and run the AABB test against the opponent's hurt box.

push1 := rect{g.p1 - 18, 510, 36, 100}
push2 := rect{g.p2 - 18, 510, 36, 100}
if overlap(push1, push2) {
    g.p1 = g.p2 - 36   // push p1 back out
}

if g.attack == 12 {   // active attack frame
    hit  := rect{g.p1 + 15, 535, 90, 32}
    hurt := rect{g.p2 - 24, 500, 48, 105}
    if overlap(hit, hurt) {
        g.hits++
    }
}

func overlap(a, b rect) bool {
    return a.x < b.x+b.w && a.x+a.w > b.x &&
           a.y < b.y+b.h && a.y+a.h > b.y
}

WITHOUT BOXES

No way to separate collision from graphics

A single position point cannot express area. Rectangles let you test two regions simultaneously, and adjust them independently of the visible sprite.

WITH SEPARATE BOXES

Each concern is tuned in one place

Widening the hurt box makes the fighter easier to hit; shrinking the hit box tightens the attack range — all without touching the sprite's visual dimensions.

YOUR FIRST TUNING

Resize the hit box width

Edit rect{p1 + 15, 535, 90, 32} — try 120 for a long-range punch or 50 for a close-range jab and feel the difference.
