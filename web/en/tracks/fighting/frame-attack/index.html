DEEP DIVE / FRAME DATA

Why doesn't a punch land
the instant you press the button?

An attack is divided into three phases: Startup (frames 1–8), where the arm is still extending and no hit box exists; Active (frames 9–12), the four frames when a red attack box is live and a hit can register; and Recovery (frames 13–30), when the fighter returns to neutral and cannot act. Pressing the button early just starts the startup clock again.

1

Startup

Frames 1–8 prepare the arm swing. No hit box exists yet. Short startup means a fast move; long startup gives the opponent more time to react.

frame <= 8 → STARTUP (yellow)
2

Active

Frames 9–12 spawn the red hit box. A hit registers only if it overlaps the opponent's hurt box during this window. Only four frames — timing matters.

frame 9–12 → ACTIVE (red)
3

Recovery

Frames 13–30 return the fighter to neutral. No new actions can start. Long recovery means a punishable move on whiff or block.

frame >= 13 → RECOVERY (blue)

FRAME RANGES

STARTUP frames 1 – 8 // no hit box yet ↓ ACTIVE frames 9 – 12 // hit box lives here ↓ RECOVERY frames 13 – 30 // cannot act

Ebitengine targets 60 frames per second by default, so one frame ≈ 16.7 ms. The total move length here is 30 frames — exactly 500 ms. Adjusting the Active window boundary shifts how strict the timing must be without changing move length.

IN THE REAL GAME

Press → increment counter → test overlap in Active

On button press, set g.frame = 1. Each update, increment it and wrap at 31. During frames 9–12, spawn the attack rect and run the overlap test. The hitThis flag prevents double-counting within one punch.

if g.frame == 0 && press() {
    g.frame = 1        // begin move on press
    g.hitThis = false
} else if g.frame > 0 {
    g.frame++
    if g.frame >= 31 { g.frame = 0 }  // move ends
}

// create attack box only during Active frames
if g.frame >= 9 && g.frame <= 12 && !g.hitThis {
    attack := rect{235, 495, 105, 35}
    if overlap(attack, g.target()) {
        g.hits++
        g.hitThis = true    // once per punch
    }
}

WITHOUT FRAME DATA

Every button press is an instant hit

Without startup delay, attacks have no timing skill to master. There is no window for the opponent to dodge, block, or counter.

WITH FRAME DATA

Moves gain distinct character

Fast startup with long recovery trades safety for speed. Slow startup with large damage creates a high-risk, high-reward option. Frame data is the grammar of fighting-game design.

YOUR FIRST TUNING

Change the Active window length

Edit g.frame <= 12 to g.frame <= 16 for an 8-frame Active window that is easier to land, or try 10 for a precise one-frame window.
