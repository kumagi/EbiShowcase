DEEP DIVE / LEVEL DATA

Separate game rules from level numbers.

“Three pieces clear” is a rule shared by every level. “Score 650 in 12 moves” is data for this level. Keep them separate and you can build the next level without rewriting the game.

1

Make a level record

Put the name, move limit, target score, and starting board in one stage value. Think of it as an order sheet for the game.

stage{moves: 12, targetScore: 650}
2

Finish one move

Only a useful swap costs a move. Then clear, fall, refill, and scan again until the board settles.

moves--; resolveCascades()
3

Compare win and loss

Reaching the target clears the level. Reaching zero moves below the target loses it. Retry with the same stage data.

won = score >= level.targetScore

IN THE REAL GAME

Load data into shared rules

Pass a different stage to newGame and the moves, target, and board all change. Update keeps the same swap rules.

type stage struct {
    moves       int
    targetScore int
    board       [rows][cols]int
}

func newGame(level stage) *game {
    return &game{level: level,
        board: level.board,
        moves: level.moves}
}

WHY DATA?

Make more levels safely

Give an easy level more moves or a hard level a higher target without copying and breaking its rules.

WHY RETURN BAD SWAPS?

Protect the meaning of a move

A swap that makes no line returns to its old place. The puzzle stays about planning matches.
