DEEP DIVE / SHOP

Why does the button
go dark when you can't afford it?

The button color is determined by a single condition: sweets >= cost. When the player cannot afford a purchase, paint a dim color; when they can, paint a bright one. That one boolean drives all the buy/reject feedback.

1

Purchase condition check

When the shop area is pressed, test whether current sweets meet or exceed cost. If the condition fails, skip the purchase and set a negative flash counter to show the error message.

if g.sweets >= g.cost { … }
2

Split zones by touch Y

One press event can mean two different actions depending on where it landed. A y coordinate below 470 is the bake zone; above it is the shop. No separate buttons needed — the coordinate splits behavior.

if y < 470 { bake } else { buy }
3

Linearly rising cost

Each purchase adds 5 to cost. Starting at 10, the second oven costs 15, the third 20. The same baking pace that cleared the first oven will fall short for later ones, giving the game a natural difficulty curve.

g.cost += 5

THE SHOP RULE

sweets >= cost → buy: sweets -= cost; ovens++elsesweets < cost → reject: show error flash

The flash counter is positive for a success glow and negative for a failure message. Each frame its absolute value decreases by one until it reaches zero and fades out.

IN THE REAL GAME

Check → buy → raise cost

Branch on y coordinate to separate bake from shop. In the shop branch, test affordability. On success, deduct sweets, add an oven, increment cost, and set the success flash. On failure, set the error flash.

if ok {
    if y < 470 {
        g.sweets++
        g.flash = 8
    } else if g.sweets >= g.cost {
        g.sweets -= g.cost
        g.ovens++
        g.cost += 5
        g.flash = 12
    } else {
        g.flash = -15   // error flash
    }
}

WITHOUT COST CHECK

Free unlimited ovens

Without the condition, ovens++ runs on every press regardless of funds. The game collapses into a button-mashing exercise with no economy.

WITH RISING COST

The last oven is the hardest

A fixed cost would make every oven equally easy. Linear growth means each purchase demands more effort than the one before, stretching the play session naturally.

YOUR FIRST TUNING

Change the cost growth rate

Increase g.cost += 5 to += 8 for a harder shop, or += 2 for an easy one. Also try changing the oven goal from 5 to see how it feels.
