DEEP DIVE / INPUT

Why does the count rise
only at the instant of a press?

Ebitengine's inpututil.IsMouseButtonJustPressed returns true only on the first frame a button becomes pressed. Hold the button for ten frames and it still fires exactly once. That is why one tap means one sweet.

1

Detect the press instant

IsPressed is true on every frame the button is held. IsJustPressed is true only on the very first frame. Clickers need JustPressed; using IsPressed lets players hold the button to score infinitely fast.

inpututil.IsMouseButtonJustPressed(...)
2

Counter variable

A single integer count tracks the total. Each confirmed press adds one. Displaying it with fmt.Sprintf("%03d", g.count) pads to three digits so the number stays in place as it grows.

g.count++
3

Pulse for visual feedback

On each press, set pulse = 10 and decrement it by one every frame. Use that value to scale the circle radius slightly outward. The result is a quick expand-and-shrink animation.

scale := 1 + float64(g.pulse)*.012

THE INPUT RULE

IsPressed → true every frame while heldvsIsJustPressed → true on the first frame only

Touch input uses inpututil.AppendJustPressedTouchIDs(nil). The same single-frame rule applies on mobile, so one finger tap equals one event.

IN THE REAL GAME

Press → count → flash

The helper pressed() checks key, mouse, and touch in one call. When it returns true, increment count and set the pulse. Reaching 30 sets the clear flag.

func pressed() bool {
    return inpututil.IsKeyJustPressed(ebiten.KeySpace) ||
        inpututil.IsMouseButtonJustPressed(ebiten.MouseButtonLeft) ||
        len(inpututil.AppendJustPressedTouchIDs(nil)) > 0
}

if pressed() {
    g.count++
    g.pulse = 10
    if g.count >= 30 {
        g.clear = true
    }
}

WITHOUT JUSTPRESSED

Holding counts forever

With plain IsPressed, holding the button for one second at 60 fps adds 60 sweets. The game becomes trivial to cheat without trying.

WITH JUSTPRESSED

One tap, one sweet — always

JustPressed cannot fire twice in the same frame. Any input device — keyboard, mouse, or touch — produces exactly one event per deliberate action.

YOUR FIRST TUNING

Change the goal and bounce

Raise the clear target beyond 30, or increase the pulse start value and the coefficient .012 to make the sweet bounce more dramatically.
