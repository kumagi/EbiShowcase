DEEP DIVE / EXPONENTIAL

Why does every new lab
cost so much more?

Cost is computed as 10 × 1.18n, where n is the number of labs already owned. Each purchase multiplies the next price by 1.18, so costs start affordable and accelerate sharply. That is exponential growth.

1

Exponential cost

math.Pow(a, n) means multiply a by itself n times (2³ = 8). float64(g.labs) only converts an int so Pow can accept it. Cost is gentle early, steep later.

10 * math.Pow(1.18, float64(g.labs))
2

Quadratic production rate

Production is labs × labs × 8 energy per second. Doubling your lab count quadruples output. That super-linear scaling is what makes mid-game purchases feel powerful and keeps players motivated to buy more.

float64(g.labs*g.labs) * 8
3

K / M / B number format

1e6 is scientific notation for 1,000,000; 1e9 is a billion. In fmt.Sprintf("%.2fM", v), %.2f means “two digits after the decimal,” and M marks millions.

fmt.Sprintf("%.2fM", v/1e6)

IN THE REAL GAME

Compute cost → produce → format display

Two small methods keep the logic clean. cost() recalculates on every call so it always reflects the current lab count. short() is a pure formatting helper used in Draw.

func (g *game) cost() float64 {
    return 10 * math.Pow(1.18, float64(g.labs))
}
func (g *game) rate() float64 {
    return float64(g.labs*g.labs) * 8
}

func short(v float64) string {
    switch {
    case v >= 1e9: return fmt.Sprintf("%.2fB", v/1e9)
    case v >= 1e6: return fmt.Sprintf("%.2fM", v/1e6)
    case v >= 1e3: return fmt.Sprintf("%.2fK", v/1e3)
    default:      return fmt.Sprintf("%.1f", v)
    }
}

WITHOUT EXPONENTIAL

Linear cost is bought out quickly

Flat or linearly rising costs let players clear the shop in a constant number of taps. There is no late-game tension or the satisfying "almost there" feeling.

WITH BIG-NUMBER FORMAT

Millions fit on one line

Without formatting, seven-digit floats overflow the canvas. With short(), even a billion displays as a tidy "1.00B" that fits anywhere.

YOUR FIRST TUNING

Adjust the exponential base

Change 1.18 to 1.10 for gentler growth or 1.30 for brutal jumps. Try changing the rate multiplier from 8 to 12 and see how quickly you reach a million.
