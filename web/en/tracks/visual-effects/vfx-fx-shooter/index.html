ADVANCED-FX DESIGN RULE

Separate the answer
from the time used to show it.

updatePlay() first commits the correct collision, score, or board. It hands that fact across once as a value or event. Presentation then advances interpolation, poses, trails, and particles. Draw only reads state; it never edits score or coordinates. This boundary supports both a rich screen and unit tests that need no GPU.

DEEP DIVE / EVENT QUEUE + BUDGET

How do concurrent effects keep
order and a hard ceiling?

Gameplay never chooses an effect function. It emits Kind, position, and Strength. The dispatcher chooses the presentation, so a low-quality mode can lower Budget without changing bullets or score.

Rules only publish Events

Remove Burst from firing code; centralize mapping and load control in the dispatcher.

g.events.Push(vfxmotion.Event{
    Kind: vfxmotion.EventEnemyDestroyed, X: e.x, Y: e.y, Strength: fx,

for _, event := range g.events.Drain() {
    n := g.budget.Take(wantedParticles(event))
    g.dispatch(event, n)
}

TEST THE CONTRACT

Test order, once-only drain, and budget

Push two events, verify Drain empties the queue, and verify a limit-five budget grants requests 4+4 as 4+1.

FAILURE MODE

Never budget away the event itself

Dropping score or audio with particles makes game results depend on device speed. Only optional visuals are capped.
