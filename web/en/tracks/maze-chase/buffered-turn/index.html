GO + EBITENGINE

Choose direction only at tile centers

Update may remember input every tick, but it changes direction only when a tile is reached.

if dir, ok := directionInput(); ok {
    queued = dir
}
if reachedTileCenter {
    if passable(add(tile, queued)) {
        current = queued
    } else if !passable(add(tile, current)) {
        return // stop at the center
    }
    target = add(tile, current)
}

WHY BUFFER?

Why does this feel better?

The player need not hit one exact frame; the game waits briefly for their intention to become legal. add(tile, queued) adds a direction (e.g. right = +1 on x) to the tile to get the next cell.

WHEN TO CLEAR?

When does the queue expire?

This game keeps it until success. Other games use a short timer to tune the feel.
