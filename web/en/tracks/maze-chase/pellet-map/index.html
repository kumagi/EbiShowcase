DEEP DIVE / STATIC AND MUTABLE LAYERS

Separate “can I walk here?” from “is it still here?”

Maze walls never change during this run, but a pellet disappears when collected. Instead of forcing both facts into one character, overlay equal-sized walls and pellets arrays. They share position [y][x] while fixed data and mutable state have different jobs.

1

Parse text into two layers

A # sets walls true and a . sets pellets true. Draw paints walls first, then only pellets that remain.

walls[y][x] = tile == '#'; pellets[y][x] = tile == '.'
2

Collection changes pellets only

When the player enters a tile, change a true pellet to false. Passing through again cannot score twice.

if pellets[y][x] { pellets[y][x] = false }
3

Decrease remaining at the same time

Do not recount the whole map every frame. Decrement remaining in the successful collection operation. Zero means stage clear immediately.

remaining--; if remaining == 0 { clear = true }

TRY IT / PELLET LAYER

Pellets are a bool layer beside the maze. Stepping on one sets false; zero left clears the stage. Walls stay untouched.

IN THE REAL GAME

Put successful collection in one function

Starting player and enemy tiles use the same function. A false cell does nothing, preventing remaining from decreasing twice.

func (g *Game) removePellet(p Point) {
    if g.pellets[p.y][p.x] {
        g.pellets[p.y][p.x] = false
        g.remaining--
    }
}

removePellet(player)

WHY TWO LAYERS?

Collect without breaking walls

Changing a pellet to false leaves walls untouched, so movement never depends on collection state.

WHY A COUNTER?

Constant-time completion

Even on a huge map, testing remaining against zero needs one comparison.
