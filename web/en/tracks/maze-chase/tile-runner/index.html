DEEP DIVE / CENTER TO CENTER

Connect free-moving screen coordinates to maze cells.

Maze data is a maze[y][x] table of characters: # means wall and . means corridor. The runner remembers its tile, adds a direction to make a candidate tile, and begins moving only when that tile is open.

1

Turn a tile into its center

Multiply column x by tile width, then add the maze left and half a tile. The same equation for y gives every corridor a center.

centerX = mazeX + tileX*tileSize + tileSize/2
2

Check whether the next tile is legal

Add a direction such as (-1,0). Movement is legal when the candidate is inside and maze[nextY][nextX] is not #.

passable := maze[next.y][next.x] != '#'
3

Interpolate center to center

Increase progress from zero to one and draw between from and to. At one, set the current tile to target.

screenX = fromX + (toX-fromX)*progress

IN THE REAL GAME

Start new movement only at a center

While moving, advance only progress. Once moving becomes false at a center, convert input into a candidate tile.

if moving {
    progress += 0.105
    if progress >= 1 {
        tile = target
        moving = false
    }
    return
}

next := tile.add(direction)
if passable(next) {
    target = next
    moving = true
}

WHY TILE DATA?

Wall tests become one character

Instead of comparing many wall rectangles, use row and column to ask whether one character is #.

WHY CENTERS?

Never drift out of corridors

Every step finishes at an exact center, preventing small errors from accumulating toward a wall.
