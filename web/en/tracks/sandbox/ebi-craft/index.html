DEEP DIVE / COMPOSED WORLD STATE

Connect separate systems through the same world data.

Terrain, chunks, inventory, lanterns, creatures, and time of day are separate rules. They all read and write one game state. Harvest empties a tile and adds inventory; those items become light; light repels creatures. That chain of cause and effect becomes the game.

1

Split the world into states

tiles owns terrain, inventory owns counts, lanterns owns chunk goals, and creature owns AI. Their roles differ, but Update advances them in a clear order.

world → inventory → light → creatures
2

Connect rules by handoff

Harvest writes zero to tiles and adds items. Crafting spends items and writes a lantern. Nearby lantern tiles remove night creatures.

tile → item → recipe → light → AI
3

Save every needed state

Saving tiles alone cannot restore the same world if position and inventory reset. Copy every value required to continue into one snapshot.

snapshot{tiles, player, inventory, lanterns}

IN THE REAL GAME

Save and load travel through the same shape

Save copies current values into snapshot; load copies the same fields back into game. Add every new persistent system to this shape.

type snapshot struct {
    tiles [worldH][worldW]int
    px, py int
    wood, stone, crystal int
    lanterns [3]bool
    pickaxe bool
}
saved = snapshot{tiles, px, py,
    wood, stone, crystal, lanterns, pickaxe}
// load copies the same fields back
