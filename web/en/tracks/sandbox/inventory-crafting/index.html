DEEP DIVE / RECIPE TRANSACTION

Check ingredients, consume them together, then add the output.

Identical materials are stacked as a kind and quantity pair. A recipe is a table of required counts. Check every count first; only a successful match subtracts ingredients and creates the result.

IN THE REAL GAME

Return on shortage to protect inventory

Every required count is checked before subtraction, so a failed craft cannot lose only some ingredients.

r := recipes[selected]
if wood < r.wood || stone < r.stone ||
   fiber < r.fiber || rope < r.rope {
    return // change nothing
}
wood -= r.wood; stone -= r.stone
fiber -= r.fiber; rope -= r.rope
addOutput(r.name)

WHY CHECK FIRST?

No half-consumption

The complete craft either succeeds or leaves everything unchanged.
