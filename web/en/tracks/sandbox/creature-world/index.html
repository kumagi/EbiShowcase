DEEP DIVE / CREATURE STATE MACHINE

Separate “when and where” from “what it does.”

Spawning decides how a creature enters the world; a state machine decides its life afterward. Day plus meadow creates grazers, while night plus rock creates prowlers. Each creature then changes its own state from distance, food, and torch events.

1

Spawn from time and terrain

Check the day phase on a timer. Add grazers only in the left meadow by day and prowlers only in the right rocks at night.

if isDay() { spawnGrazer(meadow) }
2

Select behavior from state

WANDER picks random directions. A nearby grazer becomes CURIOUS, a prowler becomes CHASE, and torch light changes it to FLEE.

switch creature.state { ... }
3

Let actions cause transitions

Feeding a CURIOUS grazer changes it to FRIEND and follow. A CHASE contact costs HP, then sends that prowler into FLEE.

curious + feed → friend

IN THE REAL GAME

Change state before choosing state motion

First evaluate transitions from distance, then use the current state's direction and speed. Old and new behaviors never mix in one frame.

distance := distance(creature.pos, player)
switch creature.state {
case wander:
    if distance < 115 {
        creature.state = curious
    }
case curious:
    creature.dir = direction(creature.pos, player)
    if distance > 170 {
        creature.state = wander
    }
}

WHY TWO DISTANCES?

Avoid boundary flicker

Entering at 115 and leaving at 170 prevents tiny movements near one line from flipping state every frame.

WHY STATE PER CREATURE?

Individuals act differently

One grazer can be FRIEND while another stays WANDER because every creature remembers its own state.
