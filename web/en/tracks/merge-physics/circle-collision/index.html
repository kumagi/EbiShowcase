When have two circles
really touched?

They overlap when center distance is shorter than the sum of both radii. The center-to-center arrow (the normal) is the push direction. The formula differs from rectangle hits (LEVEL 03), but Update’s job—“if overlapping, push out”—is the same.

THE ONE-LINE RULE

distance < a.r + b.r then push out half-and-half along the normal

Moving only one circle makes the other feel like a wall. Splitting the push keeps the bounce fair.

IN THE REAL GAME

Share the push-out

Inside Update, measure distance; if overlapping, separate along the normal and add a bounce from relative velocity. vn := rvx*nx + rvy*ny is the dot product along the collision normal.

dx, dy := b.x-a.x, b.y-a.y
distance := math.Hypot(dx, dy)
if distance < a.r+b.r {
    nx, ny := dx/distance, dy/distance
    overlap := a.r + b.r - distance
    a.x -= nx * overlap / 2
    b.x += nx * overlap / 2
    a.y -= ny * overlap / 2
    b.y += ny * overlap / 2
    // project relative velocity onto the normal, then bounce
    rvx, rvy := a.vx-b.vx, a.vy-b.vy
    vn := rvx*nx + rvy*ny
    if vn < 0 {
        a.vx -= vn * nx * 0.9
        a.vy -= vn * ny * 0.9
        b.vx += vn * nx * 0.9
        b.vy += vn * ny * 0.9
    }
}
