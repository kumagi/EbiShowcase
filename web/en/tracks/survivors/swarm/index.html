DEEP DIVE / OBJECT POOL

How can 100 enemies update
every tick without slowdown?

Appending and removing from a slice allocates memory dynamically, which costs time. A fixed-size array declared upfront never allocates during the game loop. When an enemy is defeated, overwrite its slot immediately — the pool always stays at exactly 100 and allocation cost is zero.

1

Fixed array

Declaring [100]mob reserves exactly 100 slots in memory from the start. Unlike a slice, no new memory is allocated when updating or replacing elements.

mobs [100]mob
2

Instant respawn

On defeat, do not delete the enemy. Overwrite its slot with fresh coordinates via spawn(i). The count stays at 100 at all times.

g.spawn(i)
3

Aura attack

Kill once and seat 0 becomes N1. Array length stays 8.

A second benefit is cache efficiency. One hundred structs packed contiguously in memory let the CPU load many at once. Iterating the same array every tick is much faster than chasing pointers scattered across the heap.

IN THE REAL GAME

Overwrite to respawn — always 100

spawn(i) picks a random angle and radius around the player and writes new coordinates into slot i. Called immediately after a kill inside the update loop, it keeps the pool full without any slice operations.

func (g *game) spawn(i int) {
    a := g.rng.Float64() * math.Pi * 2
    r := 280 + g.rng.Float64()*120
    g.mobs[i] = mob{
        g.px + math.Cos(a)*r,
        g.py + math.Sin(a)*r, true,
    }
}

// inside the update loop
if d < 62 && g.frame%18 == 0 {
    g.kills++
    g.spawn(i)
    continue
}

WITHOUT A POOL

append/remove stresses the GC

Frequently adding and deleting slice elements creates garbage for the collector. Constantly spawning and destroying 100 enemies would cause noticeable frame hitches.

WITH A POOL

Fixed slots, zero allocation

The array always has exactly 100 slots. Overwriting a slot costs the same as a plain struct assignment — no GC pressure, no memory fragmentation.

YOUR FIRST TUNING

Scale the pool or aura interval

Change [100]mob to [200]mob for a denser swarm. Lower the aura interval from 18 for faster kills, or raise it to make survival tense.
