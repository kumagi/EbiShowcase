GO + EBITENGINE

Weighted movement and weapon range

Here is the central rule used by the playable game.

for queue.Len() > 0 {
    cell := popCheapest()
    for _, next := range neighbors(cell) {
        nextCost := cell.cost + terrainCost(next)
        if nextCost <= unit.move { push(next, nextCost) }
    }
}

Why this structure?

Five movement points do not always mean five cells. Add terrain costs and discover reachable cells from the cheapest path first.
