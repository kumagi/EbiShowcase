DEEP DIVE / STATE MACHINE

How a state machine
keeps turns clean

A state machine stores which phase the battle is in as a single integer, then runs different logic for each phase. This battle cycles through command → playerTurn → enemyTurn → result. Knowing the current state makes it trivial to block input during animations and skip enemy action until the player has acted.

1

Define states with iota

Use Go's iota to create sequential constants: command=0, playerTurn=1, enemyTurn=2, result=3. One integer variable tracks the entire battle phase.

state = command
2

Branch on state every frame

A switch g.state at the top of Update() separates input handling (command state) from animation waiting (playerTurn / enemyTurn states).

switch g.state { case command: ... }
3

Use a timer for animation gaps

After acting, set timer = 35 and decrement each frame. When the timer hits zero, advance to the next state. Changing the number changes how long the pause lasts.

g.timer--; if g.timer <= 0 { ... }

THE STATE FLOW

command → (player chooses) → playerTurn timer expires playerTurn → (HP check) → enemyTurn or result timer expires enemyTurn → (HP check) → command or result

Choosing Guard sets guard = true. In the enemy phase that flag halves the incoming damage — reusing the flag pattern from Step 02.

IN THE REAL GAME

switch drives every phase transition

Each call to Update() evaluates the state switch. Selecting Attack calculates damage and transitions to playerTurn with a timer. When the timer drains, the code checks whether the enemy is dead or moves to enemyTurn for the counter-attack.

switch g.state {
case command:
    if c == 0 {  // attack
        d := 8 + g.rng.Intn(7)
        g.enemyHP -= d
        g.state = playerTurn
        g.timer = 35
    }
case playerTurn:
    g.timer--
    if g.timer <= 0 {
        if g.enemyHP <= 0 { g.clear = true }
        else { g.state = enemyTurn; g.timer = 40 }
    }
case enemyTurn:
    g.timer--
    if g.timer <= 0 {
        d := 7 + g.rng.Intn(8)
        if g.guard { d = (d + 1) / 2 }
        g.hp -= d
        g.state = command
    }
}

WITHOUT STATE MACHINE

No state means timing bugs

Without a phase variable you would need nested if chains to block input during animations. Mistakes let the enemy act twice or accept player input mid-animation.

WITH STATE MACHINE

One variable, clear turn boundaries

The current state immediately tells you what is allowed. Player input is only meaningful in command state; everything else is locked out automatically.

YOUR FIRST TUNING

Change the animation timer length

Increase g.timer = 35 for slow dramatic pauses, decrease it for fast-paced battles. You could also use different timer lengths for attack, heal, and guard actions.
