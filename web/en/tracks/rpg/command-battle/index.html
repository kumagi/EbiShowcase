DEEP DIVE / STATE MACHINE

How a state machine
keeps turns clean

A state machine stores which phase the battle is in as a single integer, then runs different logic for each phase. This battle cycles through command → playerTurn → enemyTurn → result. Knowing the current state makes it trivial to block input during animations and skip enemy action until the player has acted.

1

Define states with iota

iota auto-numbers constants in a block: start at 0, then +1 each line. Write command = iota, then playerTurn, enemyTurn, result — Go fills 0,1,2,3. You name the phases; the numbers come free.

state = command
2

Branch on state every frame

A switch g.state at the top of Update() separates input handling (command state) from animation waiting (playerTurn / enemyTurn states).

switch g.state { case command: ... }
3

Use a timer for animation gaps

After acting, set timer = 35 and decrement each frame. When the timer hits zero, advance to the next state. Changing the number changes how long the pause lasts.

g.timer--; if g.timer <= 0 { ... }

IN THE REAL GAME

Damage lands when the motion starts

Update() runs every frame. Apply the player's effect the moment a command is chosen; the playerTurn timer is follow-through only. Enemy damage also lands on the frame you enter enemyTurn (attack pose starts). Waiting until the timer hits zero to change HP desyncs numbers from the animation.

const (
    command = iota // 0
    playerTurn     // 1 (auto +1)
    enemyTurn      // 2
    result         // 3
)

switch g.state {
case command:
    if c == 0 {  // Attack chosen = player motion starts
        d := 8 + g.rng.Intn(7)
        g.enemyHP -= d          // damage here
        g.state = playerTurn
        g.timer = 35            // then animate
    }
case playerTurn:
    g.timer--
    if g.timer <= 0 {
        if g.enemyHP <= 0 { g.clear = true } else {
            d := 7 + g.rng.Intn(8)
            g.hp -= d           // enemyTurn starts = hit frame
            g.state = enemyTurn
            g.timer = 40
        }
    }
case enemyTurn:
    g.timer--                   // HP already changed; wait out the pose
    if g.timer <= 0 { g.state = command }
}

WITHOUT STATE MACHINE

What goes wrong without states?

Nested ifs alone let the enemy move during command select, or accept input mid-animation. Timing bugs pile up fast.

WITH STATE MACHINE

One variable, clear turn boundaries

The current state immediately tells you what is allowed. Player input is only meaningful in command state; everything else is locked out automatically.

YOUR FIRST TUNING

Change the animation timer length

Increase g.timer = 35 for slow dramatic pauses, decrease it for fast-paced battles. You could also use different timer lengths for attack, heal, and guard actions.
