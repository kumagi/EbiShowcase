DEEP DIVE / TILE COLLISION

Check the next tile
before moving

The map is a slice of strings. A '#' character marks a wall; everything else is open floor. Before the player moves, the code checks whether the destination tile is a wall. Position updates only when it is not. The facing direction updates even at a wall, which later lets the player talk across it.

1

Map as a string table

A slice of strings doubles as the tile map. '#' is a wall, space is floor. Array indices directly equal tile coordinates — no conversion needed.

rows[y][x] == '#'
2

Check before moving

Input produces a direction vector d. Add it to the current position to get the next tile n. Only update coordinates when n is not a wall.

n := point{p.x+d.x, p.y+d.y}
3

Always update facing

Even when a wall blocks movement, the facing direction still changes. Later steps use facing to find the tile directly in front and trigger NPC dialogue.

g.facing = d

THE MOVEMENT RULE

g.facing = d compute next tile n := point{g.p.x + d.x, g.p.y + d.y} move only if open if !g.walls[n] { g.p = n }

Updating facing first means the talk system in later steps can always add facing to the player position to find the NPC in front, regardless of whether the player moved.

IN THE REAL GAME

Input → facing → movement check

The real code translates input into a direction vector, records the facing, then decides whether to move. Stepping on a star removes it and increments the counter. Reaching the shrine with all three stars triggers the clear state.

if d.x != 0 || d.y != 0 {
    g.facing = d
    n := point{g.p.x + d.x, g.p.y + d.y}
    if !g.walls[n] {
        g.p = n
        if g.stars[n] {
            delete(g.stars, n)
            g.collected++
        }
    }
}
if g.p == (point{8, 1}) && g.collected == 3 {
    g.clear = true
}

WITHOUT TILE CHECK

No check means walking through walls

Incrementing coordinates blindly lets the player walk through walls or off the edge of the map, breaking the game world.

WITH FACING

One field unlocks talk-across-walls

In a later step, adding facing to the player position gives the NPC tile instantly. No raycasting or special logic required.
