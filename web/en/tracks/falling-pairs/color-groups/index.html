DEEP DIVE / BREADTH-FIRST FLOOD FILL

Start at one cell and spread only along same-color paths.

Right after a pair locks, choose an unvisited colored cell as an entrance. Put it in a queue, remove one, and add same-colored neighbors. Recording each discovered cell in visited prevents adjacent cells from adding each other forever.

1

Put the entrance in a queue

Add one colored cell and mark visited immediately. Waiting until removal could let multiple neighbors enqueue the same cell.

queue := []Point{start}; visited[start] = true
2

Inspect only four directions

Diagonals do not connect. Check right, left, down, and up, appending in-bounds neighbors of the same color.

directions := []Point{{1,0},{-1,0},{0,1},{0,-1}}
3

Count after collection

An empty queue means the whole color island is collected. If group length is at least four, add every cell to marked.

if len(group) >= 4 { mark(group) }

IN THE REAL GAME

Spread until the queue becomes empty

Visited is shared across the board scan, so another entrance never counts an old group twice. Pair placement returns as group length.

for len(queue) > 0 {
    current := queue[0]
    queue = queue[1:]
    group = append(group, current)
    for _, d := range directions {
        next := Point{current.x+d.x, current.y+d.y}
        if inside(next) && !visited[next] &&
           board[next.y][next.x] == kind {
            visited[next] = true
            queue = append(queue, next)
        }
    }
}

WHY MARK ON ADD?

Never reserve twice

Marking when enqueued blocks another path from adding the same waiting cell.
