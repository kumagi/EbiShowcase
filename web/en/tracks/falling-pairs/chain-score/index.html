GO + EBITENGINE

Connect states into a loop

Update advances one phase at a time. Returning to searching after gravity is the heart of chaining.

switch phase {
case searching:
    marked = findGroups(board)
    if len(marked) == 0 { finishChain() }
    chain++
    score += len(marked) * 10 * chain
    phase = clearing
case clearing:
    clear(marked)
    phase = falling
case falling:
    compact(&board)
    phase = searching // scan again
}

Why separate phases?

Clear stages make intermediate results visible and tell you exactly where a bug entered the loop.

When should score be added?

Add it when a group is found. cells × 10 × chain makes later clears more valuable.
