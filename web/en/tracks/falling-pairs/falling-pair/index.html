DEEP DIVE / PIVOT AND CHILD

Do not move two cells separately; derive the child from the pivot.

The falling state stores pivot x and y. The child is always child = pivot + (0,-1). For sideways input or falling, test both cells at the candidate pivot position.

1

Build two cells from one pivot

The pivot is (pivotX, pivotY); its child is (pivotX, pivotY-1). Their distance remains exactly one cell.

childX, childY := pivotX, pivotY-1
2

Test the next position first

For sideways movement and natural fall, blocked checks whether both candidate cells are inside and empty. Update only on success.

if !blocked(x+dx, y+dy) { x += dx; y += dy }
3

Lock when downward motion fails

If the next row is blocked, write pivot and child colors into two board cells, then spawn a fresh pair at the entrance.

board[y][x] = pivot; board[y-1][x] = child

IN THE REAL GAME

Use the same move for gravity and input

When Update's timer reaches 40 and when DOWN is pressed, both call move(0,1). Failure locks at the current position.

func (g *game) move(dx, dy int) bool {
    if g.blocked(g.x+dx, g.y+dy) {
        return false
    }
    g.x += dx
    g.y += dy
    return true
}

if !g.move(0, 1) {
    g.lockPair()
}

WHY RELATIVE?

The pair cannot tear apart

Updating one stored position prevents the bug where only one of two cells moves.

WHY CHECK BOTH?

The child cannot pass through walls

An empty pivot cell is not enough if the child overlaps. The whole pair is one candidate.
