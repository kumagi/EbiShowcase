IN THE REAL GAME

Skip holes with read and write

read visits every cell. write marks the next packed row. Empty cells do not move write, so only remaining colors collect at the bottom.

for x := 0; x < cols; x++ {
    write := rows - 1
    for read := rows-1; read >= 0; read-- {
        if board[read][x] == empty { continue }
        board[write][x] = board[read][x]
        write--
    }
    for write >= 0 {
        board[write][x] = empty
        write--
    }
}

WHY SIMULTANEOUS?

Do not alter search results midway

Clearing one group at a time changes connections for groups checked later. Mark all, then change all.

WHY PER COLUMN?

Preserve horizontal position

Gravity moves only downward, so column processing closes holes without changing x.
