DEEP DIVE / CONTACT EVENT

Extract “contact began” from “still touching.”

Checking overlap every frame could fire an ability 60 times while circles remain together. Compare this frame with the previous one and create an event only when touching now but separated before. That event reads the ally's ability data.

1

Test contact now

Contact occurs when center distance is no greater than the two radii added together. It uses the same simple circles visible on screen.

touching := distance <= playerR + allyR
2

Emit only on the edge

If touching is true and old wasTouching is false, contact just began. Save the new value for next frame.

if touching && !wasTouching { activate() }
3

Dispatch from ability data

An ally stores name, kind, and power. Shared activate reads the kind and selects laser, heal, or pulse.

Ability{Name: "HEAL", Kind: Heal, Power: 2}

Process contact events one at a time

IN THE REAL GAME

One contact entrance, data-defined results

Each ally does not need a new collision test. Pass an ability number through the same entrance, so new effects do not change movement or contact code.

now := circlesTouch(player, ally)
if now && !ally.wasTouching && ally.cooldown == 0 {
    activate(abilities[ally.ability])
    ally.cooldown = 75
}
ally.wasTouching = now

WHY AN EDGE?

One effect per contact

Using contact start prevents repeated firing while two circles stay overlapped.

WHY DATA?

Add allies safely

Separating movement from ability behavior lets a new effect join the same contact rule.
