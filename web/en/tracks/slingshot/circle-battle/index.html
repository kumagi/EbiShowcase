DEEP DIVE / CIRCLE CONTACT

Measure center distance, then push out by the overlap.

Two circles overlap when the distance between centers is shorter than their radii added together. Move the pearl back to the edge, and deal damage only when impact is fast enough.

1

Find circle overlap

Compute center distance from the X and Y differences. With radii 20 and 30, any distance below 50 is contact.

distance < playerR + enemyR
2

Push back to the edge

Normalize the direction from enemy to pearl, then place the pearl one sum-of-radii away.

player = enemy + normal * minDistance
3

One hit per contact

Keep a 24-tick cooldown after damage. HP cannot fall on every tick while circles remain close.

if cooldown == 0 { hp--; cooldown = 24 }

IN THE REAL GAME

Test, correct position, then attack

Remove overlap first. Attack only when speed along the contact direction is at least 1.5 and cooldown is zero.

if distance < playerR+enemyR {
    normal := delta / distance
    player = enemy + normal*(playerR+enemyR)
    impact := abs(dot(velocity, normal))
    if cooldown == 0 && impact >= 1.5 {
        hp--; cooldown = 24
    }
}

WHY PUSH OUT?

Circles do not stick

Reflecting velocity alone leaves overlap, causing another collision next tick.

WHY SPEED?

Separate a touch from an attack

Contact-direction speed turns building momentum into the game's challenge.
