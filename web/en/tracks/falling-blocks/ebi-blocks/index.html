DEEP DIVE / SYSTEM PIPELINE

Connect many systems in “what happens now” order.

A complete game is not one giant rule. It is a line of small jobs: input, movement, collision, locking, line checks, scoring, and the next shape. Pass one piece and board through this pipeline so added features do not get lost.

1

Control the falling piece

active stores a kind, rotation, and position. Update those values only when the candidate does not overlap the board.

if !collides(next) { active = next }
2

Lock when movement ends

Write the four cells into board, clear complete rows, then update score and level. This order matters.

lock(); clearLines(); addScore()
3

Prepare the next piece

Refill queue from a seven-piece bag. HOLD works once per falling piece, while the ghost tests positions down to the final safe row.

spawn(takeNext())

IN THE REAL GAME

Finish the pipeline at the lock boundary

After active becomes board cells, finish lines, score, and next-piece work before returning control.

func (g *game) lock() {
    writeActiveToBoard()
    cleared := g.clearLines()
    g.addScore(cleared)
    if g.score >= targetScore {
        g.won = true
        return
    }
    g.spawn(g.takeNext())
}

WHY ONE BOARD?

Keep one collision answer

Store only locked cells in board. Calculate active and ghost when needed so three copies cannot drift apart.

WHY A BAG?

Every kind gets a turn

Shuffle all seven and use them before opening the next bag, avoiding very long waits for one shape.
