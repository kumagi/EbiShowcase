DEEP DIVE / LOCK AND COMPACT

Copy the moving piece into the board, then remove every full row.

The instant a piece lands, its two cells are locked into board. Then inspect every row, skip full rows, and copy every row worth keeping downward.

1

Detect landing

If the next cell is the floor or occupied, the piece cannot descend. Write it into the board array, not only onto the screen.

board[y][x] = true
2

Find full rows

Keep a row if even one cell is empty. Only a row with all eight cells set to true counts as cleared.

full = full && board[y][x]
3

Compact kept rows

Use read for the inspected row and write for its destination. Skip full rows and copy only the others downward.

board[write] = board[read]

TRY IT / ROW COMPACTION

Put only kept rows into the work queue

Add rows, then process one at a time. Skipping cleared rows lets the remaining rows form one unbroken stack.

IN THE REAL GAME

Read upward and write upward

Copy only non-full rows to write, then fill every unused row at the top with an empty row.

write := rows - 1
for read := rows - 1; read >= 0; read-- {
    if rowIsFull(read) { continue }
    board[write] = board[read]
    write--
}
for write >= 0 { board[write] = emptyRow; write-- }

WHY LOCK FIRST?

Count the completed row

If you scan first, the falling piece's final two cells are still missing.

WHY BOTTOM-UP?

Avoid overwriting rows

Writing from the bottom never destroys an upper row that has not been read yet.
