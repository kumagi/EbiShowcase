DEEP DIVE / RELATIVE COORDINATES

A shape can move as “one origin plus four offsets.”

Moving four screen positions separately could pull the cells apart. Instead give the piece one origin (x, y). Each cell remembers only its offset from that origin. Move the origin one cell and all four move together.

1

Write a shape as data

An O stores the origin, right, down, and down-right positions. Because these are numbers rather than a picture, every shape can share drawing and collision code.

{{0,0}, {1,0}, {0,1}, {1,1}}
2

Add the origin

A board cell is x + offset.x. To move left, decrease only the piece's origin x by one.

cellX := piece.x + offset.x
3

Check all four cells

If even one cell would leave the board, reject the move. Change the origin only after everyone fits.

for _, p := range shape.blocks { check(x+p.x) }

IN THE REAL GAME

Draw and collision read the same array

One source of shape data keeps the visible blocks and collision blocks in agreement. A new shape needs only four new coordinates.

type Shape struct {
    blocks [4]Point
}

for _, offset := range shape.blocks {
    boardX := piece.x + offset.x
    boardY := piece.y + offset.y
    drawCell(boardX, boardY)
}

WHY RELATIVE?

Move as one object

Only the origin moves, so spacing between the four cells never changes.

WHY DATA?

Reuse the rules

An I and a T can use the same loop for drawing and wall tests.
