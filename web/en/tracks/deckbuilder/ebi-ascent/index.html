DEEP DIVE / GAME COMPOSITION

A large game is still a chain of small screens.

Switch battle, reward, and route selection with a phase that names the current scene. Deck, HP, and floor remain in one game struct across scenes, turning them into one journey.

1

Separate scenes

Battle reads card input, Reward reads three choices, and Route reads two. The phase gives the same tap one clear meaning.

2

Choose shared data

HP, deck, and floor belong to the whole run. Energy and block reset for each battle.

type game struct { deck []card; floor int }

Join exits to entrances
