DEEP DIVE / INTENT & STATUS

When you see “18 damage next,” you can change your plan.

Before the enemy acts, Draw displays its intent data. Poison and Weak store remaining time as integers; use their effect at turn end, then subtract one.

1

Make actions data

Keep a name, attack, and block together. Use the same data for drawing and acting so the promise always matches the real move.

type intent struct { damage, block int }
2

Count a status

Poison 4 hurts, then becomes 3. Weak 2 reduces enemy attacks for two turns and then ends.

enemyHP -= poison; poison--
3

Fix the update order

Resolve poison, enemy attack, duration, then next intent in one function so every turn follows the same rules.

resolveEnemyTurn()

IN THE REAL GAME

Execute the intent that was displayed

Read the current intent, reduce attack to three quarters during Weak, then advance to the next intent.

in := intents[intentIndex]
damage := in.damage
if weak > 0 {
    damage = damage * 3 / 4
    weak--
}
hp -= max(0, damage-block)
intentIndex = (intentIndex+1) % len(intents)

WHY INTENT?

Make informed choices

Visible danger lets players decide whether attack, defense, or a status matters now.

WHY DURATION?

Let effects end

A count that reaches zero prevents poison and weakness from lasting forever.
