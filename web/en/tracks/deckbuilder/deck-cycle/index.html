DEEP DIVE / DECK CYCLE

How does a played card return to your hand later?

Do not delete it. Change only its home: draw pile → hand → discard pile. When draw is empty, move discard back, shuffle, and keep drawing.

1

Draw one

Move the last card of draw into hand. Remove it from draw so one card can never be in two places at once.

hand = append(hand, draw[last])
2

Discard after play

After calculating attack or block, remove the card from hand and append it to discard. Discard means waiting, not erased.

discard = append(discard, played)
3

Shuffle when empty

Only when draw reaches zero, return every discarded card and randomize their order. Do not reshuffle early.

if len(draw) == 0 { reshuffle() }

IN THE REAL GAME

Check draw before taking a card

Draw one at a time until hand has five. If draw is empty, move discard, clear it, and then shuffle.

for len(hand) < 5 {
    if len(draw) == 0 {
        draw = append(draw, discard...)
        discard = nil
        rng.Shuffle(len(draw), swap)
    }
    last := len(draw) - 1
    hand = append(hand, draw[last])
    draw = draw[:last]
}

WHY THREE PILES?

Know every card's job

Ready to draw, available to choose, and already used become easy rules when they are separate slices.

WHY SHUFFLE?

Each cycle changes

The same twelve cards make new combinations, creating interesting guesses about the next draw.
