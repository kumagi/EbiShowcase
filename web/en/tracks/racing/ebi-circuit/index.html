GO + EBITENGINE

Car angle, acceleration, and checkpoints

Here is the central rule used by the playable game.

speed += acceleration
speed *= friction
x += math.Sin(angle) * speed
y -= math.Cos(angle) * speed
if hit(checkpoints[next]) { next++ }

Why this structure?

Use dependable top-down 2D instead of forcing pseudo-3D. Convert angle into forward movement and prevent shortcuts with an ordered gate list.
