Here is the central rule used by the playable game.

type Choice struct { Text string; Next int }
type Scene struct {
    Speaker, Text, Expression, Enter string
    Choices []Choice
}
current = scenes[choice.Next]

Why this structure?

Keep speaker, line, expression, and next scene in data instead of hard-coded branches. The game becomes a player for the current scene number.
