THE UPDATE / DRAW BOUNDARY

Decide the numbers first, then paint them

1

Update owns

Read keys and touch, then change positions, HP, gauges, timers, boards, and queues. Rules and decisions live here too.

input → rule → change game
2

Draw owns

Read the finished game state and place pictures, text, and effects. It never changes game state.

read game → project pixels

For this lesson, put a new rule in Update (or a pure helper called by Update). Draw only shows the result of that rule.

ONE STATE, FOUR DRAWS

Change the picture, not the game.

The same game made by Update can be rendered as ASCII, wireframe, pixel art, or polished sprites. Draw it four ways in four panels and Update, input, and physics remain one unchanged system. Draw is a replaceable projection, not a partner that must run immediately after Update.
