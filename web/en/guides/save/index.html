SYSTEM GUIDE / SAVE

Make a game that resumes,
without becoming fragile.

A save is more than putting state into JSON. Give it a version, choose meaningful autosave moments, recover safely from unreadable data, and pass facts—not screens—between scenes.

FOUR PROMISES

Four promises for saves

01

Write a version first

Your future self can identify old data after rules change. Unknown versions start safely instead of guessing.

02

Autosave after meaningful change

Save after a chest, a room transition, or a setting change—not every frame.

03

Recover to a safe initial state

If JSON is broken, keep the game playable and say that the save could not be restored.

04

Screens do not carry facts

A scene presents; the save model is the fact. Pass the model, then build a new scene.

savegame.Autosave(store, "adventure.slot", state, time.Now())version + updated_at + data

Recovery order

Load → Decode → check version → read payload → create a new scene. On any failure, return to a safe initial state. `internal/savegame` uses localStorage in WASM and an in-memory store in local tests through one interface.
