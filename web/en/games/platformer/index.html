DEEP DIVE / COLLISION AND CAMERA

How does the player
stop exactly at surfaces?

The trick is resolving X and Y separately. Move horizontally, check every platform, fix only the X coordinate if overlapping. Then move vertically and fix only Y. This two-pass approach lets the player slide along walls, bonk the ceiling without losing horizontal momentum, and land cleanly on floors — all without special cases.

1

X collision

After moving horizontally, if the player overlaps a platform, push them back to the edge in the direction they came from. Y stays untouched.

player.x = p.x - player.w
2

Y collision

After moving vertically, check again. Landing from above sets onGround = true and zeros vertical velocity. Hitting a ceiling from below also zeros velocity.

onGround = true
3

Smooth camera

Every frame compute where the camera should be and move 9% of the remaining distance toward it. The larger the gap, the faster the camera catches up.

camera += (target - camera) * .09

THE CAMERA LERP

target = player.x - width * 0.38then every framecamera += (target - camera) * 0.09

The target keeps the player at 38% of the screen width. Each frame the camera moves 9% closer. Big gaps close quickly; small gaps close gently, giving a natural feel.

IN THE REAL GAME

X first, then Y —
order is everything

Mixing both axes in one pass causes corner-clipping bugs where the player can pass through the edge of a platform. Separate passes make the behavior predictable.

// move horizontally and resolve
g.player.x += g.vx
for _, p := range g.platforms {
    if overlap(g.player, p) {
        if g.vx > 0 {
            g.player.x = p.x - g.player.w
        } else if g.vx < 0 {
            g.player.x = p.x + p.w
        }
        g.vx = 0
    }
}
// move vertically and resolve
g.player.y += g.vy
g.onGround = false
for _, p := range g.platforms {
    if overlap(g.player, p) {
        if g.vy > 0 { // falling → land
            g.player.y = p.y - g.player.h
            g.onGround = true
        } else if g.vy < 0 { // rising → ceiling
            g.player.y = p.y + p.h
        }
        g.vy = 0
    }
}
// lerp camera toward player
target := g.player.x - width*.38
g.camera += (target - g.camera) * .09

WITHOUT AXIS SEPARATION

Corner trapping

Resolving both axes together lets the player get stuck on platform edges. Separating them eliminates the whole class of corner bugs.

WITH AXIS SEPARATION

Smooth surfaces, correct jumps

The player slides freely along walls. Hitting a ceiling mid-jump stops vertical movement but keeps horizontal momentum intact.

YOUR FIRST TUNING

Experiment with the numbers

Increase the camera lerp factor 0.09 for a snappier camera; lower it for a lazier follow. Change the jump velocity -12.5 and gravity 0.62 to drastically alter how the game feels to play.
