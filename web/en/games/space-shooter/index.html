DEEP DIVE / ENTITY MANAGEMENT

Why doesn't the code grow
when bullets multiply?

Entity management is the answer. Put objects of the same kind in a slice and run the same loop over all of them every frame. One bullet or a hundred bullets, the code is identical. When an object is gone, remove it safely by iterating backward.

1

One slice per kind

Player bullets are []body, enemy bullets are []body, enemies are []enemy, stars are []star. Each kind has its own slice; all share the same pattern.

bullets []body
2

Update all at once

Range over the slice and move every element. No element needs special casing; the loop applies the same rule uniformly.

for i := range g.bullets
3

Delete backward

Remove off-screen or dead entities by iterating from the last index toward zero. Deleting an element does not shift earlier indices, so nothing is skipped.

bullets[:i], bullets[i+1:]

THE DELETE PATTERN

for i := len(g.bullets) - 1; i >= 0; i--when readyg.bullets = append(g.bullets[:i], g.bullets[i+1:]...)

Counting down from the last index is the key. Any deletion only affects positions above i, which have already been visited, so the loop stays correct.

IN THE REAL GAME

Collision and deletion
in a single backward loop

After moving all bullets and enemies, a nested loop tests every (bullet, enemy) pair. A hit removes both. The outer loop runs backward so deletions stay safe.

for bi := len(g.bullets) - 1; bi >= 0; bi-- {
    hit := false
    for ei := len(g.enemies) - 1; ei >= 0; ei-- {
        if overlaps(g.bullets[bi], g.enemies[ei].body) {
            // remove the hit enemy
            g.enemies = append(g.enemies[:ei], g.enemies[ei+1:]...)
            g.score += 100
            hit = true
            break
        }
    }
    // remove the bullet if it hit or left the screen
    if hit || g.bullets[bi].y < -20 {
        g.bullets = append(g.bullets[:bi], g.bullets[bi+1:]...)
    }
}

WITHOUT SLICES

Variables explode

Storing bullets as bullet1, bullet2, … forces you to pick a maximum count upfront. Each new wave means rewriting code.

WITH ENTITY SLICES

Count is data; code is one

Slices grow and shrink as needed. Wave five with ten enemies uses the same update and draw loops as wave one with four.

YOUR FIRST TUNING

Experiment with the numbers

Raise count in spawnWave for denser waves. Change the bullet's vy: -10 to alter shot speed. Lower shootCooldown = 9 to allow faster firing.
