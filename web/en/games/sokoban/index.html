DEEP DIVE / TILE MAP

How is the stage
defined in code?

With a tile map. The stage is a slice of strings. At startup the code reads every character, decides what each tile contains, and fills the game structs. Rendering later reads those structs back and draws each tile. Stage design and game logic stay completely separate.

1

String-based stage

The level is a []string where each character encodes one tile. Looking at the source is like looking at the map itself.

var level = []string{…}
2

map for walls

Walls are stored in a map[point]bool. Checking whether a tile is a wall is a single map lookup — faster and cleaner than scanning a slice.

walls map[point]bool
3

Undo is a stack

Before any move, save the player and box positions as a snapshot. Undo pops the last snapshot and restores it.

history []snapshot

The outer loop provides the row (y coordinate) and the inner loop provides the column (x coordinate). A switch on character c fills walls, boxes, goals, and the player start.

IN THE REAL GAME

move steps one tile
at a time

If the next tile is a wall, do nothing. If a box is there, check the tile beyond before pushing. Always save a snapshot first so undo always works.

func (g *game) move(d point) {
    next := point{g.player.x + d.x, g.player.y + d.y}
    if g.walls[next] {
        return // wall blocks movement
    }
    boxIndex := g.boxAt(next)
    if boxIndex >= 0 {
        beyond := point{next.x + d.x, next.y + d.y}
        if g.walls[beyond] || g.boxAt(beyond) >= 0 {
            return // box cannot be pushed
        }
        g.save() // snapshot before moving
        g.boxes[boxIndex] = beyond
    } else {
        g.save()
    }
    g.player = next
    g.moves++
}

WITHOUT A TILE MAP

Pixel coordinates are fragile

Hardcoding wall positions in pixels means touching dozens of numbers for a small layout change. Tile coordinates shrink each position to a single integer pair.

WITH A TILE MAP

The string is the map

You can read the level string and immediately visualize the stage. Adding a new level is writing one more string.

YOUR FIRST TUNING

Design your own stage

Edit the level string to create a custom puzzle. Use # for walls, $ for boxes, . for goals, and @ for the player. Make sure the number of boxes equals the number of goals.
