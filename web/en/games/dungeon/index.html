DEEP DIVE / ENEMY AI AND STATE TRANSITIONS

How does the enemy
know when to chase?

With a state machine. Each enemy holds an integer that says what it is doing right now: 0 means patrol, 1 means chase. Every frame the game measures the distance to the player and flips that integer when the threshold is crossed. One if / else then picks the right movement rule for that state. A single integer, two rules — and the enemy feels alive.

1

state field

The enemy struct stores state int. Zero is patrol; one is chase. That one integer is the entire "mind" of the enemy.

state, timer int
2

Distance triggers

Every frame, compute the distance to the player. Below 165 pixels, switch to chase. Above 230, return to patrol. The gap between thresholds prevents jittery switching.

if dist < 165 { e.state = 1 }
3

One rule per state

Patrol rotates velocity by 90 degrees on a timer. Chase normalizes the vector toward the player and applies a constant speed.

if e.state == 0 { … } else { … }

THE CHASE DIRECTION

dx, dy := (player.x - enemy.x), (player.y - enemy.y)normalize then scalevx, vy = dx/dist * speed, dy/dist * speed

Dividing by the distance produces a unit vector that points toward the player regardless of angle. Multiplying by speed then sets an exact constant velocity in that direction.

IN THE REAL GAME

Measure distance, update state,
apply the matching rule

Every living enemy goes through this sequence once per frame: check distance, maybe change state, then move according to the current state.

for i := range g.enemies {
    e := &g.enemies[i]
    if !e.alive { continue }
    dist := math.Hypot((g.player.x+14)-e.x, (g.player.y+14)-e.y)
    // update state based on distance
    if dist < 165 {
        e.state = 1 // switch to chase
    } else if dist > 230 {
        e.state = 0 // return to patrol
    }
    if e.state == 0 { // patrol: rotate velocity on timer
        e.timer--
        if e.timer <= 0 {
            e.vx, e.vy = -e.vy, e.vx // 90-degree turn
            e.timer = 100
        }
    } else { // chase: move toward player
        e.vx = ((g.player.x+14) - e.x) / dist * 1.35
        e.vy = ((g.player.y+14) - e.y) / dist * 1.35
    }
}

WITHOUT A STATE MACHINE

Always chasing

An enemy that charges from the moment it spawns is easy to predict and hard to balance. The player always knows exactly what to expect.

WITH A STATE MACHINE

Patrol, notice, then chase

Enemies feel like they have intent. The moment they spot the player creates a recognizable beat of tension that makes encounters memorable.

YOUR FIRST TUNING

Experiment with the numbers

Raise the chase threshold 165 for earlier detection and more pressure. Increase chase speed 1.35 for harder enemies. Shorten the patrol timer 100 for more erratic wandering.
