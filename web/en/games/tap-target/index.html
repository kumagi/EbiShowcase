DEEP DIVE / INPUT & COORDINATES

How does the game
know where you tapped?

Think of the screen as graph paper without lines. The top-left is (0, 0). X grows to the right, Y grows downward. If the distance from your tap to the circle center is shorter than the radius, it counts as a hit.

1

Coordinates

Any place on screen is two numbers. The circle center and the tap use the same X and Y.

circleX, circleY
2

Distance

The length between two points, from the triangle’s width and height. Comparing squared values avoids a square root.

hypot(dx, dy)
3

Hit test

If distance is less than or equal to the radius, score a hit and move the target.

dist <= radius

THE ONE-LINE RULE

dx = touchX - circleXthendy = touchY - circleY

A hit happens when dx*dx + dy*dy is smaller than radius*radius. Comparing squares gives the same answer without computing a square root every frame.

IN THE REAL GAME

Put it inside
Update

Only read the position on the press frame, then check distance. Draw just paints the circle and score. Separating math from drawing makes later changes easier.

func (g *game) Update() error {
    px, py, pressed := pressedPosition()
    if !pressed {
        return nil
    }

    // Distance from tap to center
    dist := math.Hypot(float64(px)-g.x, float64(py)-g.y)
    if dist <= g.radius {
        g.score++
        g.moveTarget()
    }
    return nil
}

WITHOUT COORDINATES

“Somewhere around here” fails

Programs cannot use vague places. Every position must become numbers.

WITH DISTANCE CHECK

One ruler for circle and tap

Even when it looks round, the check is center plus radius. The rule stays the same on any screen size.

YOUR FIRST TUNING

Experiment with numbers

Smaller radius makes it harder. Larger framesLeft makes a calmer round. Game design is hunting for values that feel right.
