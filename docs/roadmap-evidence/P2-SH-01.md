# P2-SH-01 — Create the bilingual Shader Lab hub, shared Kage helpers, and a safe fallback path.

Status: PASS

## Required evidence

- [x] Implementation
- [x] Automated checks
- [x] Manual review

## Changes

- Added `internal/shaderlab`: an embedded Kage pulse shader with `Available` and `Draw`. Shader compilation failure produces an unavailable effect rather than a panic; callers then use their ordinary `DrawImage` path.
- Added the bilingual Shader Lab hub at `/ja/labs/shader/` and `/en/labs/shader/`, and the playable `shader-safe` WASM demo. It states the central contract: visual effects enhance rendering but may not block gameplay, collision, saving, or input.
- Registered the generator in `scripts/build.sh` so both language pages regenerate with the rest of the site.

## Commands and results

```text
gofmt -w internal/shaderlab/shaderlab.go games/tracks/shader-lab/shader-safe/main.go
go test ./internal/shaderlab
GOOS=js GOARCH=wasm go build ./games/tracks/shader-lab/shader-safe
node scripts/gen-shader-lab.mjs
bash scripts/build.sh --fast
success
```

## Manual review

- The local `shader-safe` WASM page created one canvas at phone width with no console errors.
- The demo visibly reports `KAGE ACTIVE` when its pulse shader compiles; its normal image branch remains present for the fallback path.
- Japanese and English hubs have their own localized title, explanation, navigation, and same playable demo.
